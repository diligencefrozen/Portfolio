## 한눈에 보는 핵심

| 모델 | 핵심 Point | Reason | Explain | 실전 Point |
| --- | --- | --- | --- | --- |
| SVM | 스케일에 민감한 분류 모델 | 거리/마진 기반으로 경계를 찾기 때문 | 변수 단위가 크면 해당 변수가 모델을 지배할 수 있음 | 정규화/표준화 후 학습 |
| Naive Bayes | 확률 기반 분류 모델 | 각 Feature가 서로 독립이라고 가정 | 스팸 필터, 텍스트 분류, 감정 분석 등에 자주 사용 | 문자 데이터는 Encoding 필요 |
| Decision Tree | 규칙을 나무 구조로 분류 | 조건을 기준으로 데이터를 반복 분할 | Entropy/Gini로 좋은 분기 기준 선택 | `max_depth`로 과적합 방지 |

## 핵심 공식

### 정규화 Normalization

```python
x_scaled = (x - x_min) / (x_max - x_min)
```

- 값을 **0~1 사이**로 맞춘다.
- SVM처럼 스케일에 민감한 모델에서 중요하다.

### 표준화 Standardization

```python
x_standardized = (x - x_mean) / x_std
```

- 평균을 0, 표준편차를 1에 가깝게 맞춘다.
- 모델이 특정 변수 크기에 끌려가지 않게 만든다.

SVM은 분류 경계와 마진을 기준으로 클래스를 나누는 모델이며, **Feature scale에 매우 민감**하다.

# Decision Tree - Iris

> 원본 파일: `Jun29_3_DecisionTree.ipynb`
> 

## 핵심정리

Decision Tree는 데이터를 조건별로 나누며 최종 Label에 도달하는 지도학습 분류 모델이다.

각 노드에서 Entropy 또는 Gini 같은 불순도 기준을 사용해 가장 잘 나뉘는 조건을 찾는다.

노트북에서는 Iris 데이터 중 꽃잎 길이/넓이만 사용해 `criterion='entropy'`, `max_depth=3` 조건으로 트리를 학습했다.

Tree는 해석이 쉽지만 깊어질수록 과적합되기 쉬우므로 `max_depth` 같은 가지치기 옵션이 중요하다.

---

## 원본 코드 전체

### Cell 1 · markdown

# Decision Tree (의사결정나무) = 가지치기

머신러닝 - 지도학습 - 분류

입력값에 대한 예측값을 나무 형태로 나타내주는 모델

데이터를 분류하거나 연속된 값을 예측하기 위해서 사용하는 트리 (Tree) 구조의 모델

트리구조는 사람이 이해하기 쉽고, 각 노드의 조건이 어떤 결정을 내리는지 명확하게 보여줌

이 모델은 계층적인 방식으로 의사결정을 내리며, 
각 단계에서 데이터를 특정 기준에 따라 나누는 과정을 통해 
최종 결과를 도출 => (이러한 최종결과는) 데이터들의 분포를 나눠줌

데이터를 어떤 기준으로 분류했을 때 동일한 객체들로 잘 모아지게 할 수 있을까?
를 고려해서 분류기준을 찾는 것이 중요!

- 특징
1. 알고리즘 해석이 쉬운 편
2. 수치형, 범주(카테고리)형 데이터도 적용이 가능함
3. 데이터 전처리 양이 적은 편

```python
#       학습 시작 단계에서는 분류할 것들이 많아서
#       엔트로피가 높은 상태
#     어떤 기준에 맞춰 하나씩 정리해가면서
#       엔트로피를 줄여 최종적으로
#       0에 가깝게 만드는 방식

# criterion = 'gini' (default) :
#     분류를 잘못할 확률을 최소화하기 위한 목적
#       > 분할이 가능한 모든 방법을 지니계수로 확인
#         > 가장 낮은 것을 선택하는 방식
#  지니계수 : (분류가) 얼마나 불확실한가에 대한 지표
#     0에 가까워질수록 불확실성이 0에 가까워짐
#     => 같은 특성을 가진 개체들끼리 잘 모여있다!

# max_depth = 3 : 깊이
#   (몇 단계로 나눠서 표현할 지)
# 노드 (Node) : 데이터의 분기나 결정이 이루어지는 지점
#       첫 시작점인 루트 노트(Root Node)
#       중간 지점인 내부 노드(Internal Node)
#       최종 지점인 리프 노드(Leaf Node)

# 나무(모델)의 깊이 :
#     루트노드에서 시작해서
#     리프노드까지 이르는 경로에서 거쳐가는
#     노드의 단계수의 수를 의미

# 깊이를 제한하는 옵션으로,
#   Tree가 지나치게 깊어지는 것을 방지(하기위한 하이퍼파라미터임)!!!
# 깊이를 제한하지 않으면 Tree에는
#     데이터가 과도하게 적합(overfitting)하게
#       될 가능성이 높음
# 반대로 깊이가 얕은 Tree 에는
#   학습이 덜 된 상태(underfitting)가
#     될 가능성이 높음
# 깊이를 적절히 제한하면
#     모델에 복잡도를 줄이고,
#     과접합을 방지해 더 나은 성을 기대할수 있음.

# 어떤 기준으로 분류를 하는가에 따라서
#   tree 모양도 변하고, 깊이도 달라짐

# 독립변수 : 원인 / (시험지)
# 종속변수 : 결과 / (답안지)
# 종속변수는 독립변수에 의해 영향을 받는다
iris_tree.fit(p_feature, labels)
# .fit(독립변수, 종속변수)
```

## 핵심정리

Decision Tree는 Breast Cancer 데이터처럼 Feature가 많은 분류 문제에서도 규칙 기반으로 예측 과정을 시각화할 수 있다.

각 분기점이 특정 Feature와 기준값으로 구성되기 때문에 모델의 판단 과정을 사람이 비교적 쉽게 해석할 수 있다.

노트북에서는 클래스 불균형을 확인한 뒤 `train_test_split(..., stratify=cancer.target)`로 label 비율을 유지했고, `max_depth=4` 트리를 만들었다.

불균형 데이터에서는 `stratify`를 넣어 train/test의 클래스 비율을 원본과 비슷하게 유지하는 것이 중요하다.

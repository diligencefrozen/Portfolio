

## 1. 정수를 입력받고, 두 수의 합이 짝수일 때까지 다시 입력받는 코드

```java
import java.util.Scanner;

public class YMain1 {

    static Scanner s = new Scanner(System.in);

    // 정수를 입력받기 위한 함수 생성
    // => 정수를 2개 생성(x, y)
    // 2개의 정수를 더했을 때 짝수면 결과 리턴,
    // 아니면 값을 재입력하는 함수

    public static int num1() {
        System.out.printf("x를 입력: ");
        int x = s.nextInt();
        return x;
    }

    public static int num2() {
        System.out.printf("y를 입력: ");
        int y = s.nextInt();
        return y;
    }

    // 덧셈을 계산해서 결과를 리턴할 함수
    public static int calc(int x, int y) {
        int sum = x + y;

        // 합이 짝수면 sum 리턴
        // 아니면 x, y를 다시 입력받아서 calc 재호출
        return (sum % 2 == 0) ? sum : calc(num1(), num2());
    }

    public static void start() {
        int x = num1();
        int y = num2();

        System.out.printf("x의 값: %d\n", x);
        System.out.printf("y의 값: %d\n", y);

        int sum = calc(x, y);
        System.out.printf("2개의 정수를 더한 후: %d\n", sum);
    }

    public static void main(String[] args) {
        start();
    }
}
```

### 필기

- 함수가 자기 자신을 다시 호출하는 것을 **재귀 호출**이라고 한다.
- 이 코드에서는 합이 짝수가 아니면 `calc(num1(), num2())`를 다시 호출해서 재입력받는다.

## 2. 국어 시험 점수를 입력해서 받아오는 함수 (0~100)

```java
import java.util.Scanner;

public class CMain1 {

    static Scanner s = new Scanner(System.in);

    // 국어 시험 점수를 입력해서 받아오는 함수
    // (0~100)
    public static int korean() {
        System.out.printf("국어 시험 점수를 입력(0~100): ");
        int k = s.nextInt();
        return ((k >= 0) && (k <= 100)) ? k : korean();
        // 참 : 거짓
    }

    public static void main(String[] args) {
        int k = korean();
        System.out.println(k);

        // 이렇게 불러올 수도 있고
        System.out.println(korean());
        // 이렇게 바로 불러올 수도 있다.
    }
}
```

### 필기

- 조건식을 만족하면 그 값을 리턴
- 조건식을 만족하지 않으면 `korean()` 함수를 다시 호출
- 즉, 올바른 값이 들어올 때까지 반복 입력받는 구조
- 이 방식도 재귀 호출이다

# 3. 조건문

## 조건문이란?

작성한 코드를 **조건에 따라 실행 흐름이 다르게 동작하도록 제어하는 문법**이다.

---

## 3-1. if문

조건식의 결과가 `true`, `false`로 나뉘고,

그 결과에 따라 실행문이 결정된다.

```java
if (조건 A) {
    조건 A가 만족했을 때(참일 때) 이 부분이 실행
} else if (조건 B) {
    조건 A는 거짓, 조건 B는 참일 때 이 부분이 실행
} else if (조건 C) {
    조건 A/B는 거짓, 조건 C는 참일 때 이 부분이 실행
    // else if 문은 필요한 만큼 생성 가능
} else {
    맞는 조건이 하나도 없으면 이 부분이 실행
}
```

### 정리

- `if + else`
- `if + else if`
- `if + else if + else`

형식은 자유롭게 작성 가능하다.

### 핵심

- 위에서부터 순서대로 검사한다.
- **처음으로 참이 되는 조건 하나만 실행된다.**
- 그래서 범위 조건은 보통 **큰 값부터 내려오는 식으로 작성**한다.

---

## 3-2. 국어 점수 등급 출력

```java
import java.util.Scanner;

public class CMain1 {

    static Scanner s = new Scanner(System.in);

    // 국어 시험 점수를 입력해서 받아오는 함수
    // (0~100)
    public static int korean() {
        System.out.printf("국어 시험 점수를 입력(0~100): ");
        int k = s.nextInt();
        return ((k >= 0) && (k <= 100)) ? k : korean();
    }

    // 국어 점수가 60점 미만이면 '가'
    // 60점 이상이면 '양'
    // 70점 이상이면 '미'
    // 80점 이상이면 '우'
    // 90점 이상이면 '수' 출력
    public static void korean02() {
        System.out.printf("국어 시험 점수를 입력:\t");
        int k02 = s.nextInt();

        if (k02 >= 90) {
            System.out.println("수");
        } else if (k02 >= 80) {
            System.out.println("우");
        } else if (k02 >= 70) {
            System.out.println("미");
        } else if (k02 >= 60) {
            System.out.println("양");
        } else {
            System.out.println("가");
        }
    }

    // 국어 점수가 50점이 안되면 욕, 아니면 칭찬
    public static void korean03() {
        System.out.printf("국어 시험 점수를 입력:\t");
        int k03 = s.nextInt();

        if (k03 < 50) {
            System.out.println("ㅗ");
        } else {
            System.out.println("대단합니다.");
        }
    }

    public static void main(String[] args) {
        korean03();
    }
}
```

### 필기

- 점수 구간별 출력처럼 **범위 비교**가 필요한 경우에는 `if ~ else if ~ else`가 적합하다.
- `90 이상`, `80 이상`, `70 이상`처럼 **큰 범위부터 검사해야 함**
- 그렇지 않으면 위 조건에서 먼저 걸려서 아래 조건은 실행되지 않는다.

## 3-4. if - if 문과 if - else 문의 차이

### if - if

- 각각 다른 조건문으로 해석되어 수행하게 됨
- 즉, 조건을 각각 따로 검사함
- 서로 독립적인 조건일 때 사용 가능

### if - else

- 하나의 조건이 만족되면, 해당 블록을 수행하고 나머지 부분은 수행하지 않음
- 서로 반대되는 조건일 때 더 효율적이고 자연스럽다

### 정리

- `if - if` : 독립적인 두 조건
- `if - else` : 둘 중 하나만 실행되는 조건
- `if문 안에서 if문을 또 사용하는 것`도 가능하다
    
    → 이것을 **중첩 if문**이라고 한다.
    

---

## 3-5. 중첩 if 예제

```java
import java.util.Scanner;

public class CMain1 {

    static Scanner s = new Scanner(System.in);

    // 점수가 50점 이상 ~ 60점 미만이면 아쉬운 소리를 해주세요.
    // 점수가 30점 미만이면 퇴학 처리.
    public static void korean04() {
        System.out.printf("국어 시험 점수를 입력:\t");
        int k04 = s.nextInt();

        if ((k04 >= 50) && (k04 < 60)) {
            System.out.println("더 노력하세요.");
        } else if (k04 < 30) {
            System.out.println("ㅗ");
            System.out.println("해당 학생은 퇴학 처리하였습니다.");
        } else {
            System.out.println("잘하시네요");

            if (k04 < 60) {
                System.out.println("굿");
            }
        }
    }

    public static void main(String[] args) {
        korean04();
    }
}
```

### 필기

- `if`문 안에 또 다른 `if`문을 넣을 수 있다.
- 큰 조건을 먼저 검사하고, 그 안에서 세부 조건을 다시 검사할 때 사용한다.

# 4. if문 안에서 변수 생성

```java
import java.util.Scanner;

public class CMain2 {
    public static void main(String[] args) {

        // 정수 하나를 입력
        // 입력받은 정수 값이 5보다 크면
        // 새로운 변수 y에 정수 10을 담아서 출력
        // 입력받은 정수 값이 3보다 크거나 같으면
        // 새로운 변수 y에 정수 20을 담아서 출력

        Scanner s = new Scanner(System.in);
        System.out.printf("정수 입력: ");
        int x = s.nextInt();

        if (x > 5) {
            int y = 10;
            System.out.println(y);
        } else if (x >= 3) {
            int y = 20;
            System.out.println(y);
        }
    }
}
```

### 필기

- if문 안에 변수를 만들면 그 블록 안에서만 사용 가능하다.
- 이런 변수는 **지역변수(local variable)** 라고 한다.
- 작은 예제에서는 문제 없지만, 같은 패턴이 반복되면 바깥에서 한 번 만들고 값을 바꾸는 방식이 더 깔끔할 수 있다.

---

## 4-1. if문 바깥에서 변수 생성 후 값 변경

```java
int z = 0; // 기본값 설정

if (x > 5) {
    z = 10;
    System.out.println(z);
} else if (x >= 3) {
    z = 20;
    System.out.println(z);
}
```

### 필기

- 변수를 블록 밖에서 미리 만들고
- 조건에 따라 값을 변경(할당)해서 쓰는 방식
- 코드 구조가 더 깔끔해질 수 있다
- 상황에 따라 안쪽에서 선언하는 게 더 좋은 경우도 있고, 바깥에서 선언하는 게 더 좋은 경우도 있다
- 즉, 무조건 어느 한쪽이 정답이라기보다 **코드 구조와 사용 범위에 따라 판단**하면 된다

## 5-1. switch - case

`switch - case`는

**비교 대상이 특정한 값이나 문자열일 때** 사용하기 좋다.

보통 `sw` + 자동완성으로 작성 가능하다.

```java
switch (변수 or 식) {
    case 값A:
        변수 or 식의 값이 A일 때 이 부분이 실행됨
        break;

    case 값B:
        변수 or 식의 값이 B일 때 이 부분이 실행됨
        break;

    default:
        위 case에 해당하는 값이 하나도 없을 때 실행
        break;
}
```

### 핵심

- `break;`를 이용해서 각 case 실행 후 switch문을 빠져나온다.
- `break;`가 없으면 다음 case까지 계속 실행된다.
- 이 현상을 **fall-through**라고 한다.

---

## 5-2. switch 예제

```java
import java.util.Scanner;

public class CMain3 {
    public static void main(String[] args) {
        Scanner k = new Scanner(System.in);
        System.out.printf("x : ");
        int x = k.nextInt();
        System.out.println(x);

        // x가 1일 때 'ㅋ' 출력
        // x가 2일 때 'ㅎ' 출력
        // x가 3일 때 'ㅠㅠ' 출력
        // 위의 값이 아닌 경우에는 '배고파...' 출력

        switch (x) {
            case 1:
                System.out.println("ㅋ");
                break;
            case 2:
                System.out.println("ㅎ");
                break;
            case 3:
                System.out.println("ㅠㅠ");
                break;
            default:
                System.out.println("배고파...");
                break;
        }
    }
}
```

### 필기

- 케이스별로 `break`를 넣어주지 않으면 여러 case가 연속으로 실행될 수 있다.
- 예를 들어 `case 1`, `case 2`에 `break;`가 없으면
    
    `1` 입력 시 `ㅋ`, `ㅎ`, `ㅠㅠ`가 한 번에 출력될 수 있다.

# 7. BMI 검사 프로그램 (함수.ver)

---

## 7-1. BMI란?

BMI(체질량지수) 공식:

```java
BMI = 몸무게(kg) / (키(m) * 키(m))
```

### 주의

- 키를 **cm로 입력받았으면 m로 바꿔야 한다**
- BMI는 소수점 값이 나오므로 `double` 타입을 사용하는 것이 적절하다

---

## 7-2. BMI 판정 기준

- 18.5 미만이면 저체중
- 18.5 이상이면 정상
- 25 이상이면 과체중
- 30 이상이면 경도 비만
- 35 이상이면 중증도 비만
- 40 이상이면 고도 비만

### 중요

이런 범위 비교는 **큰 값부터 검사해야 함**

예:

```java
if (BMI >= 40) {
    ...
} else if (BMI >= 35) {
    ...
}
```

---

## 7-3. BMI 프로그램 최종 수정본

```java
import java.util.Scanner;

public class CMain5 {

    static Scanner s = new Scanner(System.in);

    public static String info1() {
        System.out.printf("이름을 입력하세요:\n");
        String n = s.next();
        return n;
    }

    public static double info2() {
        System.out.printf("키를 입력하세요(cm):\n");
        double h = s.nextDouble();
        return h / 100.0;   // cm -> m 변환
    }

    public static double info3() {
        System.out.printf("몸무게를 입력하세요(kg):\n");
        double w = s.nextDouble();
        return w;
    }

    public static double calc(double h, double w) {
        double BMI = w / (h * h);
        return BMI;
    }

    public static String calc2(double BMI) {
        String result = "저체중";

        if (BMI >= 40) {
            result = "고도비만";
        } else if (BMI >= 35) {
            result = "중증도 비만";
        } else if (BMI >= 30) {
            result = "경도비만";
        } else if (BMI >= 25) {
            result = "과체중";
        } else if (BMI >= 18.5) {
            result = "정상";
        }

        return result;
    }

    /*
     * 결과를 판단해주는 함수
     * 불필요한 변수나 중복 코드를 줄이면
     * 코드가 더 읽기 쉬워지고 관리하기 편해진다.
     */

    public static void start(String n, double h, double w, double BMI, String result) {
        System.out.println("BMI 검사를 시작합니다.");
        System.out.printf("당신의 이름은 %s이고\n당신의 키는 %.1fcm이며\n당신의 몸무게는 %.1fkg입니다.\n", n, h * 100, w);
        System.out.printf("BMI : %.1f입니다\n", BMI);

        try {
            Thread.sleep(1000);
            System.out.println(".");
            Thread.sleep(1000);
            System.out.println(".");
            Thread.sleep(1000);
            System.out.println(".");
            Thread.sleep(1000);
            System.out.println("곧, 결과가 나옵니다.");
            Thread.sleep(3000);
        } catch (InterruptedException e) {

        }

        System.out.printf("\n%s님, 당신의 BMI는 현재 %s입니다\n", n, result);

        switch (result) {

            case "정상":

            case "과체중":

                break;
            case "경도비만":
                System.out.println("\t병원 ㄱ");
                break;
            case "중증도 비만":

                break;
            case "고도비만":
                System.out.println("\t집부터 나와라");

    }

    public static void test() {
```

# 8. 용어 정리 (수정 + 보완)

## 8-1. 조건문

조건에 따라 프로그램의 실행 흐름을 다르게 만드는 문법

### 대표 종류

- `if`
- `switch`

---

## 8-2. 조건식

결과가 `true` 또는 `false`로 나오는 식

예:

```java
x > 5
score >= 60
k >= 0 && k <= 100
```

---

## 8-3. 재귀 호출

함수가 자기 자신을 다시 호출하는 것

예:

```java
return (k >= 0 && k <= 100) ? k : korean();
```

---

## 8-4. 지역변수

특정 블록 `{ }` 안에서 선언된 변수

그 블록 안에서만 사용 가능

예:

```java
if (x > 5) {
    int y = 10;
}
```

---

## 8-5. switch문

하나의 변수값을 여러 `case`와 비교하여 실행 흐름을 나누는 조건문

### 특징

- 값 비교에 적합
- `break`가 중요
- 같은 case는 묶을 수 있음

---

## 8-6. break

현재 블록을 빠져나가는 명령어

switch문에서는

해당 case 실행 후 switch를 종료시키는 역할을 한다.

---

## 8-7. fall-through

switch문에서 `break`가 없어서

다음 case까지 계속 실행되는 현상

---

## 8-8. 형변환 / 자료형 선택

BMI처럼 소수점 계산이 필요한 경우 `int`보다 `double`이 적절하다.

---

## 8-9. 범위 조건 작성 순서

다음처럼 범위가 겹치는 조건은 **큰 값부터 써야 함**

```java
if (score >= 90) {
    ...
} else if (score >= 80) {
    ...
}
```

이유:

- 위에서 먼저 참이 되면 아래는 검사하지 않기 때문

---

# 9. 필기

## 9-1. Scanner 사용

원본 필기에서는 함수마다 `Scanner`를 새로 만들었는데,

보통은 `Scanner`를 하나만 만들어서 같이 쓰는 방식이 더 일반적이다.

예:

```java
static Scanner s = new Scanner(System.in);
```

---

## 9-2. 재귀 vs 반복문

원본 필기처럼 재귀 호출로 재입력을 처리할 수 있다.

하지만 사용자가 계속 틀린 값을 넣으면 함수 호출이 계속 쌓일 수 있으므로,

실제로는 `while`문을 더 자주 사용한다.

즉,

- 재귀 : 구조 이해용, 학습용으로 좋음
- 반복문 : 실전에서 더 안정적인 경우가 많음

---

## 9-3. if문과 switch문의 사용 기준

- 범위 비교 → `if문`
- 정확한 값 비교 → `switch문`

예:

- 점수가 90 이상인가? → `if`
- 월이 1월인가 2월인가? → `switch`

---

## 9-4. BMI 계산 시 주의점

- 키는 반드시 `m` 단위로 변환
- 계산식은 `몸무게 / (키 * 키)`
- 소수점 계산이므로 `double` 사용
- 판정 조건은 큰 값부터 비교

# 10. 암기용

## if문

```java
if (조건식) {
    실행문;
} else if (조건식) {
    실행문;
} else {
    실행문;
}
```

- 위에서부터 순서대로 검사
- 처음 참인 블록 하나만 실행
- 범위 비교에 적합

---

## if - if / if - else 차이

- `if - if` : 각각 따로 검사
- `if - else` : 하나가 실행되면 나머지는 실행 안 함

---

## 중첩 if

- if문 안에 if문을 넣는 것
- 큰 조건 안에서 세부 조건을 검사할 때 사용

---

## switch문

```java
switch (변수) {
    case 값1:
        실행문;
        break;
    case 값2:
        실행문;
        break;
    default:
        실행문;
        break;
}
```

- 특정 값 비교에 적합
- `break` 없으면 다음 case까지 실행됨
- 같은 case는 묶을 수 있음

---

## 재귀 호출

- 함수가 자기 자신을 다시 부르는 것
- 잘못된 입력 시 다시 입력받는 구조로 사용 가능

---

## BMI 공식

```java
BMI = 몸무게(kg) / (키(m) * 키(m))
```

### 기준

- 18.5 미만 : 저체중
- 18.5 이상 : 정상
- 25 이상 : 과체중
- 30 이상 : 경도비만
- 35 이상 : 중증도 비만
- 40 이상 : 고도비만

- **값 비교는 switch문**
- **break 없으면 switch는 계속 내려감**
- **재귀 호출은 함수가 자기 자신을 호출하는 것**
- **BMI는 kg / m²**
- **범위 조건은 큰 값부터 검사**

# 1. Word2Vec

## 핵심 한 줄

**Word2Vec은 단어를 숫자 벡터로 바꿔서, 단어 간 의미적 유사도를 계산할 수 있게 만드는 임베딩 기법이다.**

## 핵심 개념

### Word2Vec

- `Word → Vector`
- 단어를 고정 길이 숫자 벡터로 표현한다.
- 비슷한 문맥에서 등장하는 단어는 비슷한 의미를 가진다고 가정한다.
- 이 가정이 **분포 가설(distributional hypothesis**)이다.

### Embedding

- 사람이 쓰는 자연어를 모델이 계산 가능한 숫자 벡터로 바꾸는 과정 또는 결과.
- 예: `"최민식"` → `[0.12, -0.44, ...]`

### Distributed Representation

- 단어 의미를 하나의 숫자가 아니라 여러 차원의 벡터에 분산해 표현한다.
- 그래서 단어 간 거리, 유사도, 방향성을 계산할 수 있다.

## 주요 파라미터

| 파라미터 | 의미 |
| --- | --- |
| `sentences` | 학습할 토큰화 문장 리스트 |
| `vector_size=100` | 단어 벡터 차원 수 |
| `window=5` | 중심 단어 기준 주변 단어 범위 |
| `min_count=5` | 5회 미만 등장 단어 제거 |
| `workers=4` | 병렬 학습 프로세스 수 |
| `sg=0` | CBOW 방식 사용 |
| `sg=1` | Skip-gram 방식 사용 |

```python
# 불용어 제거

# 불용어 정의
stopwords = ['의' '가', '이', '은', '들', '는', '좀', '걍', '과', '도', '를', '으로', '자', '에', '와', '한', '하다']
okt = Okt()
tokenized_data = []
for sentence in train_data['document']:
  # 형태소 분석 (stem = True : 동사의 원형)
  tokenized_sentence = okt.morphs(sentence,
                                  stem=True)
  # 불용어 제거
  stopwords_removed_sentence = [
      word for word in tokenized_sentence
      if not word in stopwords

  ]
  tokenized_data.append(stopwords_removed_sentence)
```

# 2. Regex

## 핵심 한 줄

**정규표현식은 특정 규칙을 가진 문자열 패턴을 정의하고, 검색·검증·추출할 때 쓰는 도구다.**

## 핵심 개념

### Python `re` 모듈

- Python에서 정규표현식을 사용할 때 기본적으로 `re` 모듈을 사용한다.
- 문자열이 특정 패턴과 일치하는지 확인하거나, 일치하는 부분을 추출할 수 있다.

## 주요 함수

| 함수 | 역할 |
| --- | --- |
| `re.match(pattern, string)` | 문자열 시작부터 패턴이 맞는지 확인 |
| `re.search(pattern, string)` | 문자열 전체에서 패턴이 있는지 검색 |
| `re.findall(pattern, string)` | 일치하는 모든 문자열을 리스트로 반환 |
| `re.finditer(pattern, string)` | 일치하는 모든 위치를 iterator로 반환 |
| `re.fullmatch(pattern, string)` | 문자열 전체가 패턴과 완전히 일치하는지 확인 |
| `re.compile(pattern)` | 패턴을 미리 컴파일해서 재사용 |

## Match 객체 메서드

| 메서드 | 의미 |
| --- | --- |
| `.group()` | 매칭된 문자열 반환 |
| `.start()` | 매칭 시작 인덱스 |
| `.end()` | 매칭 끝 다음 인덱스 |
| `.span()` | `(start, end)` 튜플 반환 |

## 메타문자 정리

| 패턴 | 의미 |
| --- | --- |
| `.` | 임의의 문자 1개 |
| `?` | 앞 문자가 0개 또는 1개 |
| `*` | 앞 문자가 0개 이상 |
| `+` | 앞 문자가 1개 이상 |
| `^` | 문자열 시작 |
| `$` | 문자열 끝 |
| `{n}` | 앞 문자가 n번 반복 |
| `{n,m}` | 앞 문자가 n번 이상 m번 이하 반복 |
| `{n,}` | 앞 문자가 n번 이상 반복 |
| `[abc]` | a, b, c 중 하나 |
| `[^abc]` | a, b, c가 아닌 문자 |
| `[a-z]` | 소문자 a부터 z |
| `[A-Z]` | 대문자 A부터 Z |
| `[0-9]` | 숫자 |
| `[가-힣]` | 한글 완성형 문자 |

# 3. SVM BMI

## 핵심 한 줄

**SVM은 데이터의 feature를 기준으로 class를 나누는 최적의 경계면을 찾는 지도학습 분류 알고리즘이다.**

## 머신러닝 개념 정리

### 군집 Clustering

- 비지도 학습.
- 정답 label 없이 비슷한 데이터끼리 묶는다.
- 목적은 예측보다 **패턴 발견**에 가깝다.
- 예: Word2Vec, K-means, Hierarchical clustering.

### 분류 Classification

- 지도 학습.
- label이 있는 데이터를 학습한 뒤, 새로운 데이터의 category를 예측한다.
- 예: SVM, Logistic Regression, Random Forest.

### 군집 vs 분류

| 구분 | 군집 | 분류 |
| --- | --- | --- |
| 학습 방식 | 비지도 학습 | 지도 학습 |
| label | 없음 | 있음 |
| 목적 | 데이터 구조 발견 | 새 데이터 예측 |
| 예시 | K-means, Word2Vec | SVM, Decision Tree |

## SVM

- Support Vector Machine.
- 두 클래스 또는 여러 클래스를 구분하는 결정 경계(decision boundary)를 찾는다.
- 경계와 가까운 데이터 포인트를 support vector라고 한다.
- 분류 성능이 높은 편이고, 작은/중간 규모 feature 데이터에서 강력하다.

## 정규화 vs 표준화

### 정규화 Normalization

- 데이터를 특정 범위로 압축한다.
- 보통 `0~1` 범위로 만든다.
- 올바른 공식:`x_normalized = (x - min) / (max - min)`

### 표준화 Standardization

- 평균이 0, 표준편차가 1이 되도록 변환한다.
- 공식:`z = (x - mean) / std`

## BMI 분류 기준

| BMI | status |
| --- | --- |
| `< 18.5` | `thin` |
| `< 25` | `normal` |
| `>= 25` | `fat` |

## 평가 지표

| 지표 | 의미 |
| --- | --- |
| Accuracy | 전체 예측 중 맞춘 비율 |
| Precision | 모델이 특정 class라고 예측한 것 중 실제로 맞은 비율 |
| Recall | 실제 특정 class 중 모델이 찾아낸 비율 |
| F1-score | Precision과 Recall의 조화평균 |

## 주의 포인트

- `for i in range(20001)`은 20,001개 데이터를 생성한다. 정확히 20,000개면 `range(20000)`을 쓰면 된다.
- `train_test_split(..., test_size=0.3)`은 전체 데이터 중 30%를 테스트 데이터로 분리한다.
- 마지막 예측에서 `data = [[weight / 100, height / 200]]`를 넣으면 feature name warning이 뜬다. 학습 때 DataFrame column name을 사용했기 때문이다.
- 경고를 줄이려면 아래처럼 입력 데이터를 DataFrame으로 만들면 된다.

```python
#  => 이럴때 사용하는 게 바로 조화 평균 !
```

# 4. 요약

## Word2Vec

- 목적: 단어를 벡터로 바꿔 유사도 계산.
- 핵심 가정: 비슷한 문맥의 단어는 비슷한 의미.
- 전처리: 결측값 제거 → 정규표현식 → 형태소 분석 → 불용어 제거.
- 모델: `Word2Vec(sentences, vector_size, window, min_count, workers, sg)`.
- 결과: `model.wv.most_similar()`로 유사 단어 확인.

## Regex

- 목적: 문자열 패턴 검색/추출/검증.
- `match`: 시작부터 검사.
- `search`: 전체에서 검색.
- `findall`: 모든 매칭 문자열 리스트.
- `finditer`: 모든 매칭 위치.
- `fullmatch`: 전체 완전 일치.
- `compile`: 패턴 재사용.

## SVM BMI

- 목적: 키/몸무게 feature로 BMI status 분류.
- label: `thin`, `normal`, `fat`.
- 모델: `svm.SVC()`.
- 평가: `accuracy_score`, `classification_report`.
- 핵심: 지도학습 분류 문제.

---

전처리 품질이 모델 성능을 좌우한다.

Word2Vec도 SVM도 결국 입력 데이터의 품질과 feature 형태에 따라 결과가 달라진다.

Word2Vec에서는 정규표현식, 형태소 분석, 불용어 제거가 단어 벡터 품질을 만든다.

SVM에서는 `height`, `weight`를 정규화해서 모델이 안정적으로 경계를 찾게 만든다.

코드 흐름을 외우는 것보다 **데이터 → 전처리 → 모델 → 평가** 구조를 잡는 게 더 중요하다.

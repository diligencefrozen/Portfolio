```java

		// 이름

			int math = Integer.parseInt(sData[3]);
			int eng = Integer.parseInt(sData[4]);

			int sum = kor + math + eng;
			System.out.println("총점: "+ sum);

			double avg = (double) sum / 3;
			System.out.println("평균: "+ avg);

		} catch (Exception e) {
			// TODO: handle exception
			e.printStackTrace();
		}
	}
}
```

## 3. 인코딩 / 디코딩

### 개념

- **Encoding**: 정보의 형태나 형식을 변환하는 처리 방식
- **Decoding**: 인코딩된 데이터 및 파일을 원래 상태로 변환하는 것

---

### 핵심

- 컴퓨터는 전자기계라서 결국 **전기가 흐르거나 안 흐르는 상태**로 데이터를 표현한다.
- 문자도 결국 전기 신호 형태로 바뀌어 저장/전송된다.
- 같은 데이터를 인코딩 방식이 다르면 다른 결과가 나올 수 있다.
- 따라서 **보내는 쪽과 받는 쪽의 인코딩 방식이 같아야 한다.**

---

```
Encoding : 정보의 형태나 형식을 변환하는 처리 방식

컴퓨터 : 전자기계 (전기가 흐르거나, 안 흐르거나)

‘ㅋ’
⇒
전기의 흐름으로 표현 (예시: 모스 부호) →
                                                                         encoding

Decoding : 인코딩 된 데이터 및 파일을 원래의 상태로 변환

전기의 흐름
⇒
‘ㅋ’ ⇒
           decoding

encoding 방식은 여러 종류가 있음 !

추석 -A → 1010 -A → 추석
추석 -B → 0101 -C → 설날

(인코딩 방식은 일단 서로 같아야 같은 결과값이 나옴)

전세계적으로 주력 : UTF-8(Unicode Transform Format)
한국만 타겟 : EUC-KR
MS전용 : MS949
```

## 5. Stream / 입출력 구조

### 개념

- 프로그램은 외부와 데이터를 주고받는다.
- 입력은 `Input`
- 출력은 `Output`
- `Stream`은 데이터를 전달하는 **빨대** 같은 개념이다.

---

```java
// CPU : 연산장치
// RAM : 저장장치(임시) - 재부팅하면 날아감
//       static / stack - 프로그램을 끄면(종료하면) 날아감
//       heap - 자동으로 날아감 x (Garbage collector 가 있으면 자동으로 날아감)
// HDD(SSD) : 저장장치(영구) - 파일
// GPU : 그래픽처리용 ( CPU + RAM + HDD )

// 프로그램에서는 데이터를 외부에서 읽고,
// 다시 외부로 출력하는 작업이 자주 일어나는데
// 데이터는 사용자로부터 키보드를 통해서 입력될 수도 있고,
//        (Scanner의 System.in)
//파일.네트워크에서도 입력될 수 있다.
//        (Input)

// 반대로 데이터는 모니터를 통해서 출력될 수도 있고,
//      (System.out.println)
// 파일.네트워크에서도 출력될 수 있다.
//      (Output)

// System.in => InputStream
// System.out => PrintStream

// 'Stream' : '빨대'
//      기본형
//                   InputStream :
//                   장치(키보드, 파일, 네트워크) => 프로그램으로 입력
//                   1byte씩

//                   InputStreamReader :
//                                        장치 => 프로그램
//                                        2byte씩 (빨대 크기 Up!) + 인코딩방식!

//                   BufferedReader :
//                                        장치 => 프로그램
//                                        (1 String - line) (빨대 크기 Up!)

//                   OutputStream :
//                                         프로그램의 결과를 => 장치로 출력 (1byte씩)

//                   OutputStreamWriter :
//                                         2byte씩 + 인코딩방식 !

//                   BufferedWriter :
//                                         1 String - line

//      개조형(특수목적에 맞게)
//                   printStram (OutputStream 개조)
//                   : 쓰기 편하게 유용한 메소드들 추가]

//                   FileReader (InputStreamReader 개조)
//                   : 파일에서 데이터를 읽어오기 편하게

//                   FileWriter (OutputStreamWriter 개조)
//                   : 데이터를 출력하기 편하게
```

			// 기존에 있던 파일의 내용은 사라지게 됨.
			// 기존의 내용 뒤에 덧붙이고 싶다면(데이터를 추가하고 싶다면)
			// 두번쨰 파라미터 값으로 true를 주시면 됨.
			//Encoding 방식지정 X => ANSI로 인코딩 됨
			FileWriter fw = new FileWriter("C:\\Users\\sdedu01\\Desktop\\Test\\beaver1.txt", true);

			// 한 바이트씩 입출력하기보다는
			// '버퍼'를 이용해서 한번에 여러 바이트를 입출력하는 것이 빠름
			//    => 대부분의 입출력 작업에서 사용
			// 버퍼 : CPU와 HDD 사이에서 사용되는 임시 저장 공간
			// 버퍼링 : <데이터>를 버퍼로 저장하는 동작
			//       => 실행성능을 향상 시킬 수 있음 !!
			bw = new BufferedWriter(fw);

			bw.write(cmt + "\r\n");
			bw.write("-----------------------------\r\n");

			bw.flush(); // 버퍼 용량이 다 안채워져도
			            // 강제적으로 내보내기
			            // (모든 데이터를 다 내보내기)
		} catch (Exception e) {
			// TODO: handle exception
			e.printStackTrace();
		}

		try {
			bw.close();
		} catch (IOException e) {
			// TODO Auto-generated catch block
			e.printStackTrace();
		}
	}
}

package com.beaver.mar313.main;

import java.io.BufferedReader;
import java.io.InputStream;
import java.io.InputStreamReader;

import org.apache.http.HttpEntity;

import org.apache.http.client.methods.HttpGet;

// apache.org -downloads -Releases -httpcomponents 검색
// -httpcomponents 4.5.14 선택 - Download -Older releases(archives)
// - httpclient/ 폴더 선택 - binary/ 폴더 선택 -  httpcomponents-client-4.5.9-bin.zip 선택

public class HMain1 {

			while ((line = br.readLine()) != null) {

			}

		}

## 11. 직접 만든 HTTP 유틸 클래스

### JeesungHttpClient

```java
package com.jeesung.http.client;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStream;
import java.io.InputStreamReader;
import java.net.HttpURLConnection;
import java.net.URL;
import javax.net.ssl.HttpsURLConnection;

public class JeesungHttpClient {
	// 서버에 요청해서 다운 받기
	public static InputStream download(String address) throws IOException {
		// java.net.URL은 URL의 형식이 유효한지 자동으로 검증
		URL u = new URL(address);

		if (address.startsWith("https")) {
			HttpsURLConnection huc = (HttpsURLConnection) u.openConnection();
			return huc.getInputStream();
		} else {
			HttpURLConnection huc = (HttpURLConnection) u.openConnection();
			return huc.getInputStream();
		}
	}

	// 다운받은걸 String 으로 바꾸기
	public static String convert(InputStream is, String charset) throws IOException {
		InputStreamReader isr = new InputStreamReader(is, charset);
		BufferedReader br = new BufferedReader(isr);

		String line = null;
		StringBuffer sb = new StringBuffer();

		while ((line = br.readLine()) != null) {
			sb.append(line);
		}
		return sb.toString();
	}
}
```

# 암기용

## 1) 문자열 분리

- `split("구분자")` : 문자열을 잘라 배열로 받음
- `StringTokenizer(문자열, "구분자")` : 토큰 단위로 꺼냄
- 학생 정보 입력 예시:
    - `이름/생년월일/국어/수학/영어`

---

## 2) 날짜 처리

- 문자열 날짜 → `SimpleDateFormat`으로 파싱
- 예:
    - `"yyyy.MM.dd"` 로 입력값 해석
    - `"yyyy/MM/dd E"` 로 출력 형식 변경
- 나이 계산:
    - `now.getYear() - birthday.getYear() + 1`

---

## 3) 인코딩

- **Encoding**: 문자를 저장/전송 가능한 형태로 바꾸는 것
- **Decoding**: 다시 원래 문자로 되돌리는 것
- 대표 인코딩:
    - `UTF-8` : 전세계 표준
    - `EUC-KR` : 한국 중심
    - `MS949` : MS 계열

---

## 4) Stream 구조

- 입력: `InputStream`
- 출력: `OutputStream`
- 문자 처리:
    - `InputStreamReader`
    - `OutputStreamWriter`
- 줄 단위 처리 / 속도 향상:
    - `BufferedReader`
    - `BufferedWriter`

---

## 5) 파일 입출력

- 파일 쓰기:
    - `FileWriter`
    - `BufferedWriter`
- 파일 읽기:
    - `FileReader`
    - `BufferedReader`
- `flush()` : 버퍼 강제 출력
- `close()` : 스트림 닫기
- `FileWriter(path, true)` : 이어쓰기
- `FileWriter(path)` : 덮어쓰기

---

## 6) Scanner 입력 차이

- `next()` : 띄어쓰기 전까지만 입력
- `nextLine()` : 한 줄 전체 입력
- 문장 입력 받을 때는 `nextLine()`이 안전

---

## 7) 통신

- **HTTP 통신**: 요청 → 응답
- **Socket 통신**: 실시간 양방향 통신
- 예전:
    - Apache `HttpClient` 사용
- 현재:
    - Java 내장 `URL`, `HttpURLConnection`, `HttpsURLConnection` 사용 가능

---

## 8) 요약

- 문자열 자르기: `split`, `StringTokenizer`
- 날짜 변환: `SimpleDateFormat`
- 파일 쓰기: `FileWriter + BufferedWriter`
- 파일 읽기: `FileReader + BufferedReader`
- 인코딩 지정: `InputStreamReader(..., "UTF-8")`
- HTTP 접속: `URL`, `HttpsURLConnection`

# 1. TensorFlow Variable / Placeholder

## 1-1. 핵심 정리

### 

TensorFlow의 기본 단위는 **Tensor**이고, 학습 중 값이 바뀌는 데이터는 **Variable**로 관리합니다.

### 

딥러닝은 가중치와 편향을 계속 업데이트하면서 손실을 줄이는 과정입니다. 따라서 바뀌지 않는 값과 바뀌는 값을 구분해야 합니다.

- `tf.constant()`
    - 상수 텐서 생성
    - 값을 한 번 정하면 변경 불가
- `tf.Variable()`
    - 변수 텐서 생성
    - `assign()`으로 값 변경 가능
    - 단, 기존 shape과 dtype은 맞아야 함
- `tf.convert_to_tensor()`
    - Variable을 Tensor로 변환
    - 변환 후에는 상수처럼 다루는 개념
- `tf.placeholder()`
    - TF1.x에서 외부 데이터를 주입받는 입력 자리
    - `feed_dict`로 실제 값을 넣음
- `tf.Session()`
    - TF1.x 그래프를 실행하는 객체

### 

`Variable`은 모델의 “상태”, `placeholder`는 모델에 들어가는 “입력 통로”입니다.

## 1-2. 실행 결과 메모

- `tf.Variable` 생성 후 `assign()`으로 값이 `[[100,200,300],[400,500,600]]`으로 변경됨
- `var1 + tensor1` 결과는 같은 값끼리 더해져 `[[200,400,600],[800,1000,1200]]`
- TF1 방식 그래프 실행 결과: `10 + 20 + 30 = 60`
- placeholder에 `feed_dict`로 `10, 20, 30`을 넣은 결과: `60.0`

# 2. TensorFlow BMI Softmax Regression

## 2-1. 핵심 정리

### 

BMI 예제는 **키와 몸무게를 입력(feature)** 으로 받고, `thin / normal / fat` 중 하나를 예측하는 **다중분류 문제**입니다.

분류 모델을 만들려면 입력 데이터, 정답 라벨, 손실함수, 최적화 방식이 명확히 연결되어야 합니다.

1. 랜덤으로 키,몸무게 데이터 20,000개 생성
2. BMI 공식으로 라벨 생성
3. `bmi.csv`로 저장
4. CSV 읽기
5. 키·몸무게 정규화
6. 라벨을 One-Hot Encoding으로 변환
7. 학습 데이터와 테스트 데이터 분리
8. TF1.x placeholder와 Variable로 softmax regression 모델 구성
9. Cross Entropy 손실함수 정의
10. Gradient Descent로 최적화
11. 정확도 계산
12. 테스트 데이터 및 임의 입력값 예측

이 예제의 핵심은 “데이터 전처리 → One-Hot Encoding → Softmax → Cross Entropy → Optimizer” 연결 구조입니다.

## 2-2. 핵심 개념

| 개념 | 의미 |
| --- | --- |
| Feature | 모델 입력값. 여기서는 `h`, `w` |
| Label | 정답값. 여기서는 `thin`, `normal`, `fat` |
| One-Hot Encoding | 문자열 라벨을 `[1,0,0]` 같은 숫자 배열로 변환 |
| Softmax | 여러 클래스에 대한 확률값을 출력 |
| Cross Entropy | 예측 확률과 실제 정답의 차이를 손실로 계산 |
| Gradient Descent | 손실을 줄이는 방향으로 W, b를 업데이트 |
| Accuracy | 예측 클래스와 정답 클래스가 같은 비율 |

## 2-3. 실행 결과 메모

- 생성 데이터 수: `20,000`
- CSV shape: `(20000, 3)`
- 테스트 데이터: 뒤쪽 `5,000`개 사용
- 학습 반복: `3,500` step
- 최종 정확도 출력 예시: `0.983`
- 임의 예측 입력: `183cm`, `79kg`
- 출력 예시: `thin`
- 단, `183cm / 79kg`는 BMI 기준 약 `23.6`이므로 `normal`이어야 자연스럽습니다. 예측 파트의 정규화 기준을 다시 점검해야 합니다.

  elif bmi < 25 : return 'normal'
  return 'fat'

#### Code Cell 12

```python
# 신경망 구성

# 데이터 플로우 그래프

# 키와 몸무게 데이터 넣기
# None : 데이터 갯수(행 수)는 얼마든지 가능
# 2: 각 데이터의 특성(Feature) 갯수
#             (키, 몸무게)
x = tf.placeholder(tf.float32, [None, 2])

# 정답 label
# None : label 샘플 데이터 갯수
# 3 : 각 샘플의 클래스 수 (thin, normal, fat)
y_ = tf.placeholder(tf.float32, [None, 3])

# 변수 선언
#W = tf.Variable(tf.zeros([2,3]])
W = tf.Variable(tf.zeros([2, 3]))
# 가중치 행렬 W를 0으로 초기화된
#     2행, 3열의 행렬 생성
# 입력 차원 : 2 (키, 몸무게)
# 출력 차원 : 3 (thin, normal, fat)

#b = tf.Variable(tf.zeros[[3]])
b = tf.Variable(tf.zeros([3]))
# 바이어스(bias) : 뉴런의 활성화를 조절
#           (결과에 대한 출력을 조정하는 역할)
# 바이어스를 0으로 초기화된 3개의 요소를 가진 후
#     벡터로 정의
#    각 출력 노드(thin, normal, fat)에 대해
#     하나씩의 바이어스 값을 가짐
# *** 출력 클래스 수와 똑같아야 함 !!!

# 소프트맥스 회귀(Softmax Regression)
#  다중 클래스 분류를 위한 방법
#   여러개의 클래스(결과)를 보류
# onehot encoding이랑 연결하기 위해서
#     0~1 사이의 숫자로 바꾸는 역할

# 어떤 가게에서 사고 3, 바나나 2

y = tf.nn.softmax(tf.matmul(x,W)+b)

# 손실 함수의 cross entropy
#   모델이 예측한 값과 실제 값 사이에
#   차이를 측정하기 함수
#   cross entropy :
#     모델에서 예측한 값이 틀릴 수 있다 라는 정보량
#   y_ * tf.log(y)
# 2차원 텐서가 들어있음
# reduce_sum() 로 지정된 차원에 따라서 합계를 구하는 함수
# 크로스엔트로피(데이터의 불확실성0)에서는 실제 값고, 예측 값이 맞는 경우 0에 수렴

# 값이 틀릴 경우 cs 값이 커지기 떄문애
#   => 실제값과 예측값차이를 즐이기 위한노력을 해야겠음|

#

# y_ :정답

#  y : softax() # 결과 (예측 확률)
#  tf.log(y) 예측 확률애 대한 로그
# 확률 그대로써버리면 학습이 잘안됨]\#
# 정답확률이 높을 수요감
# softMax결과  = y는 항상 0~1사이 => 확률 값
# log() 음수값을 처리해버리면
#   c최적화를 진행할 때 오류가 생길 여지가 큼!
#   => 손실(loss, 오차의 크기는) ** 값이 작을수록 좋은것**

ce = -tf.reduce_sum(y_ * tf.log(y))

# 최적화 : 경사화강법
# 함수의 절댁값이 낮은 쪽으로 이용동시켜서 극값에 이를 때까지 반복하는 방법
# 손실함수의 값을 최소화하는
# 최적의 가중치(W)와 바이어스(b)를 찾기 위함!

# 0.01 : 학습 계수(하강 간격)
zer = tf.train.GradientDescentOptimizer(0.01)

# gradient : 경사 / descent : 내려감
#     경사를 따라 내려가는 알고리즘
# optimizer : 각각의 변수에 기대되는 값과
#             손실의 도함수의 정도에 따라서
#             변수의 값을 변경해줌
#  도함수 : 원래 함수를 미분해서 얻어낸 함수
#           >> 어떤 점에서의 기울기
# 텐서플로우 : 도함수를 자동으로 만들어줌!!

# 가중치, 편차를 보정
#optimizer1 = optimizer.minimize(ce)
optimizer1 = zer.minimize(ce)

# 정답률 구하기
predict = tf.equal(tf.argmax(y,1),
                   tf.argmax(y_, 1))

# 평균 구하기
score = tf.reduce_mean(tf.cast(predict, tf.float32))
```

# 3. TensorFlow + Keras MNIST

## 3-1. 핵심 정리

Keras는 TensorFlow 위에서 동작하는 고수준 딥러닝 API이며, 복잡한 신경망 구성 과정을 훨씬 단순하게 만듭니다.

TensorFlow만으로 모델을 만들면 그래프, 세션, 변수 관리가 복잡해집니다. Keras는 `Sequential`, `compile`, `fit`, `evaluate`, `predict` 흐름으로 개발자가 빠르게 실험할 수 있게 해줍니다.

MNIST 실습 흐름은 다음과 같습니다.

1. MNIST 데이터 로드
2. 학습 데이터와 테스트 데이터 분리
3. 이미지 픽셀값 범위 확인: `0 ~ 255`
4. 픽셀값을 `0 ~ 1`로 정규화
5. `Sequential` 모델 생성
6. `Flatten`으로 28x28 이미지를 1차원으로 변환
7. `Dense(128, relu)` 은닉층 구성
8. `Dropout(0.2)`로 과대적합 완화
9. `Dense(10, softmax)`로 0~9 숫자 확률 출력
10. `compile()`로 optimizer, loss, metrics 설정
11. `fit()`으로 학습
12. `evaluate()`로 테스트 성능 확인
13. 직접 만든 28x28 이미지 업로드 후 예측

Keras MNIST 모델의 본질은 “28x28 이미지 → 1차원 벡터 → 은닉층 → 10개 숫자 확률”입니다.

## 3-2. 모델 구조

| Layer | 역할 |
| --- | --- |
| `Flatten(input_shape=(28, 28))` | 2D 이미지를 1D 벡터로 변환 |
| `Dense(128, activation='relu')` | 숫자 패턴을 학습하는 은닉층 |
| `Dropout(0.2)` | 뉴런 일부를 비활성화해 과대적합 완화 |
| `Dense(10, activation='softmax')` | 0~9 각 숫자일 확률 출력 |

## 3-3. 손실함수 선택

### 

정답 라벨이 정수라면 `sparse_categorical_crossentropy`를 사용합니다.

### 

MNIST 라벨은 `[0,1,2,...,9]` 같은 정수 형태입니다. One-Hot Encoding이 되어 있지 않으므로 sparse 버전이 맞습니다.

### 

- `categorical_crossentropy`: 정답이 One-Hot Encoding일 때
- `sparse_categorical_crossentropy`: 정답이 정수 클래스일 때
- `binary_crossentropy`: 이진분류일 때

라벨 형태와 손실함수는 반드시 맞아야 합니다.

## 3-4. 실행 결과 메모

- MNIST train shape: `(60000, 28, 28)`
- MNIST test shape: `(10000, 28, 28)`
- 픽셀값 범위: `0 ~ 255`
- 정규화 후 범위: `0 ~ 1`
- 기본 모델 평가 예시: loss 약 `0.0708`, accuracy 약 `0.978`
- 직접 만든 이미지 예측 결과 예시: `7`
- 후반부 유럽식 7 재학습 실험은 `load_mnist_style_image()` 함수 정의가 필요합니다.

#### Code Cell 10

```python
# 모델
# Sequential() :
#   순차적 구조의 모델
#     (하나의 입력 => 하나의 출력)
# >> 구조를 설정하고, 층(layer)을 설정

model = tf.keras.models.Sequential([
    # 1차원(vector)으로 평평하게 만들기
    tf.keras.layers.Flatten(input_shape=(28, 28)),

    # 중간층(은닉층)에 신경망을 만드는 단계
    # (128개짜리 퍼셉트론(인공 뉴런)으로 구성
    #       으로 구성되는 layer 1개)
    # activation='relu'
    #   : 은닉층에서 활성화 함수
    tf.keras.layers.Dense(128, activation='relu'),

    # 과대적합을 막아주는 역할
    # 128개의 뉴런 중에서
    #   무작위로 20%는 0으로 만들어줌
    #     (뉴런의 기능을 끔(비활성))

    tf.keras.layers.Dropout(0.2),

    tf.keras.layers.Dense(10, activation='softmax')
    # 10개의 퍼셉트론으로 만든 layer
    # 10개로 구성한 이유 ?
    # : 0 ~ 9까지 총 10종류의 이미지가
    # 어떤 숫자를 의미하는지 파악하기 위해
    # activation='softmax'
    # : 출력층의 결과 값을
    #   다중분류를 위한 확률 값으로
    #   계산할 수 있도록 !

    # 활성화함수 (activation) :
    #   결과값을 변환해서 다른 층으로 보내주는 함수

    # linear : 기본값, 입력뉴런과 가중치로 계산된 결과값이 그대로 출력
    # relu : 은닉(중간)층에서 주로 사용
    #     입력값이 0보다 크면 그대로 출력하고,
    #     0 이하면 0으로 만든다
    # sigmoid : 출력층에서 주로 사용 - 이진분류
    # softmax : 출력층에서 주로 사용 - 다중분류

])

print(model.summary())
```

#### Code Cell 11

```python
# 모델 컴파일
#   .compile() : 학습 방법에 대한 설정
# Sequential() 에서 정해진 모델을
#   컴퓨터가 알아볼 수 있게
#   말로(언어로) 컴파일하는 부분
model.compile(
    # 최적화하기
    # 옵치마이저는 손실함수를 통해서
    # 얻어낸 손실값으로부터
    # 모델을 업데이트 하는 방식임.
    # SGD, Adam, Nadam, Lion, ...
    optimizer = 'adam',

    loss ='sparse_categorical_crossentropy',
    # 손실함수(loss function) :
    # 모델을 최적화시킬 때 사용하는 함수
    # 손실함수는 신경망의 예측이 얼마나
    # 잘 맞는지 측정하는 역할

    # categorical_crossentropy :
    #   (클래스가) 다중분류일 때 사용하는 다중분류 손실함수
    #     (one-hot encoding 클래스)
    #     출력값이 one-hot encoding된 결과

    # sparse_categorical_crossentropy:
    # 다중분류 손실함수
    #   (one-hot encoding이 아닌 (클래스들에 대해서))
    #     출력값이 integer type 클래스로 반환

    # binary_crossentropy :
    # 클래스가 2개인 이진분류 손실함수
    #    label(정답지)이 0 or 1을 값으로 가질 때 사용

    metrics=['accuracy'] #평가지표 : 정확도

)
```

# 4. Keras Callback / ModelCheckpoint

## 4-1. 핵심 정리

### 

Callback은 모델 학습 도중 특정 조건이 발생했을 때 자동으로 실행되는 기능입니다.

딥러닝은 epoch가 늘어난다고 항상 좋아지지 않습니다. 어느 순간부터 검증 성능이 나빠질 수 있으므로, 가장 좋은 시점의 모델을 저장하는 전략이 필요합니다.

### 

`ModelCheckpoint`는 epoch마다 모델 상태를 확인하고, 지정한 지표가 좋아졌을 때 모델을 저장합니다.

주요 파라미터는 다음과 같습니다.

| 파라미터 | 의미 |
| --- | --- |
| `filepath` | 저장 파일 경로 |
| `monitor` | 저장 기준 지표 |
| `save_best_only` | 가장 좋은 모델만 저장할지 여부 |
| `verbose` | 저장 로그 출력 여부 |

### 

실무에서는 “마지막 epoch 모델”보다 “검증 성능이 가장 좋았던 모델”을 쓰는 것이 더 안전합니다.

## 4-2. 실습 흐름

1. MNIST 데이터 로드
2. 정규화
3. Keras Sequential 모델 생성
4. 모델 컴파일
5. `ModelCheckpoint` 설정
6. `fit(..., callbacks=[checkPoint])`로 학습 중 저장
7. 현재 모델 평가
8. 체크포인트 가중치 로드
9. 다시 평가해서 성능 비교

## 4-3. 실행 결과 메모

- 체크포인트 적용 전 테스트 정확도 예시: 약 `0.9786`
- 체크포인트 로드 후 테스트 정확도 예시: 약 `0.9797`
- 검증 손실 기준으로 저장했기 때문에 마지막 모델보다 더 좋은 시점의 모델을 복원할 수 있습니다.

#### Code Cell 8

```python
# 체크포인트 설정
checkPoint = keras.callbacks.ModelCheckpoint(
    filepath='checkPoint.keras',
    # 이 파일을 통해서 재학습이 가능, 불필요한 정보도 있어서
    #   파일의 크기가 크고 무거움

    save_best_only = True,
    # True : monitor 되고 있는 값 기준으로
    #       가장 뛰어난 epoch 모델이 저장
    # False : epoch마다 모델이 filepath{epoch} 형태로 저장

    monitor = 'val_loss', # model을 저장할 때 기준이 되는 값 지정
    # 테스트 데이터셋을 기준으로 loss가 가장 적을 때 저장하려면
    #   : val_loss
    # 학습 데이터셋을 기준으로 loss가 가장 적을 때 저장하려면
    #   : loss

    verbose = 1 # 1이면 모델 저장할 때 문구가 뜸
                # 0이면 문구 없이 저장만
)
```

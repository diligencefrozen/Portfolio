

## 1. 비만도 검사 프로그램

```java
public static void main(String[] args) throws InterruptedException {

    // 키, 몸무게를 입력받아서 => 비만도 검사 프로그램

    Scanner test = new Scanner(System.in);

    System.out.printf("키를 입력하세요:");
    double h = test.nextDouble();

    // 정수형으로 입력을 받았을 때,
    // nextInt() 사용 중 입력 흐름이 꼬이는 경우가 있어
    // 키는 double로 받는 것이 좀 더 유연하다.

    System.out.printf("몸무게를 입력하세요:");
    int w = test.nextInt();

    // 표준체중 = (키 - 100) x 0.9
    double calc1 = (h - 100) * 0.9;

    // 비만도 = (실제 체중 / 표준체중) x 100
    double calc2 = w / calc1 * 100;

    System.out.printf("키:");
    System.out.println(h);

    System.out.printf("몸무게:");
    System.out.println(w);

    // 전체 정보(키, 몸무게, 비만도) 출력
    System.out.println(h);
    System.out.println(w);
    System.out.println(calc2);

    // 비만도가 120이 넘으면 '비만'
    boolean d1 = calc2 > 120;
    String res1 = d1 ? "비만" : "안비만";

    System.out.println(res1);

    // 아무 키나 입력했을 때 프로그램 종료
    Thread.sleep(500);
    System.out.print(".");
    Thread.sleep(500);
    System.out.print(".");
    Thread.sleep(500);
    System.out.println(".");
    Thread.sleep(500);

    System.out.println("키가");
    System.out.println(h);
    System.out.println("몸무게가");
    System.out.println(w);
    System.out.println("비만도가");
    System.out.println(res1);

    System.out.println("아무 키나 입력");
    String bye = test.next();

    System.out.println("프로그램을 종료합니다.");
}
```

### 메모

- 키는 `double`로 받으면 더 유연하게 처리 가능하다.
- 비만도 계산식에서 **괄호가 중요**하다.
    - `h - 100 * 0.9` 가 아니라
    - **`(h - 100) * 0.9`** 로 써야 한다.
- `Thread.sleep()` 을 쓰려면 `throws InterruptedException` 처리가 필요하다.

## 2. Random 기능

Random 기능은 **int 범위 안에 있는 무작위 정수**를 불러온다.

```java
int i2 = r.nextInt(5);
```

- 괄호 안에 5를 넣으면
    
    **0 ~ 4 중 무작위 정수**를 가져온다는 의미이다.
    

```java
int i3 = r.nextInt(5) + 1;
```

- 이렇게 하면
    
    **1 ~ 5 중 무작위 정수**를 가져오게 된다.

## 3. 홀짝 게임 만들기

```java
public static void main(String[] args) {

    Random r = new Random();

    int i = r.nextInt();
    int i2 = r.nextInt(5);
    int i3 = r.nextInt(5) + 1;

    // System.out.println(i);
    // System.out.println(i2);
    // System.out.println(i3);

    // 홀짝 게임 만들기
    // 동전 10개를 쥐고 있다가,
    // 내가 '홀수' 또는 '짝수'라고 하면
    // 정답인지 아닌지 판정해서 결과를 출력하는 프로그램

    Scanner test = new Scanner(System.in);

    int coin = r.nextInt(10) + 1;
    String answer = coin % 2 == 1 ? "홀수" : "짝수";

    System.out.println("홀수 또는 짝수를 입력:");
    String useranswer = test.next();

    String result = answer.equals(useranswer) ? "정답" : "땡";

    System.out.println(coin);
    System.out.println(result);
}
```

### 메모

- `r.nextInt(10) + 1`
    
    → 1 ~ 10 사이 정수 생성
    
- 정답 비교는 문자열이므로 `==` 보다 **`.equals()`** 를 쓰는 것이 맞다.
- 오류가 났던 이유는 **Scanner 함수명, Random 함수명 오타** 때문이었다.

# 4. 함수(Function)

## 함수란?

함수를 이용하면 **관련 있는 기능(작업)들을 묶어서**,

필요할 때마다 **불러서(호출해서)** 사용할 수 있다.

---

## 자바 실행 구조

- **JDK (Java Development Kit)** : 자바 개발 도구
- **JRE (Java Runtime Environment)** : 자바 실행 환경
- 코드를 실행하면 JRE가 OS(운영체제)에 맞춰서 JVM을 실행시킨다.
- **JVM (Java Virtual Machine)** : 자바 가상머신

이 구조 덕분에 자바는

**WORA (Write Once, Run Anywhere)**

라는 장점을 가진다.

즉, **한 번 작성하면 OS별로 따로 만들지 않아도 실행 가능**하다는 뜻이다.

---

## main도 함수의 일종이다

자동 완성해서 사용하는 `main`도 사실 함수이다.

```java
public static void printMyThink() {
    // 지금 당장 생각하고 있는 내용을 출력
    System.out.printf("%s\n", "소세지");
}

public static void main(String[] args) {

}
```

이 프로그램을 실행하면 **JVM이 main을 자동으로 호출**한다.

즉,

- 코드를 실행하려면 `main` 함수 안에 작성해야 하고
- 다른 함수에 기능을 만들어두었다면
- `main`에서 그 함수를 **호출**해야 한다.

예:

```java
printMyThink();
```

---

## 함수의 기본 형태

```java
public static 리턴타입 함수명(파라미터, 파라미터, ...) {
    내용
    return 결과값;
}
```

### 구성 요소 정리

### 1) 리턴 타입

- 함수가 최종적으로 **반환할 값의 자료형**
- 결과값이 없으면 `void`

### 2) 함수명

- 함수의 이름
- 이름을 보고 무슨 기능인지 알 수 있게 짓는 것이 좋다.

### 3) 파라미터(parameter)

- 함수를 실행하는 데 필요한 재료
- 필요한 개수만큼 사용 가능
- 반드시 자료형에 맞게 사용해야 함

### 4) return

- 함수 안에서 만든 최종 결과를 호출한 곳으로 되돌려줌

---

## 함수명 작성 규칙

1. 숫자로 시작하면 오류 발생
2. 특수문자, 띄어쓰기 사용 시 오류 발생
3. 자바 예약어 사용 시 오류 발생
    - 필요하면 `_` 등을 활용하기도 함
4. 함수명을 보고 기능을 알 수 있게 짓는 것이 합리적
5. 보통 **소문자로 시작**
6. 가독성을 위해 이름 규칙 사용
    - 스네이크 케이스: `abcd_efg_hijk`
    - 카멜 케이스: `abcDefg`
7. 한글은 사용하지 않는 것이 일반적

---

## 변수명 vs 함수명

- **변수명**은 최대한 **명사**
- **함수명**은 최대한 **동사**

으로 짓는 것이 합리적이다.

**반환할 값이 없으면 `void`를 작성**해야 한다.

# 6. 파라미터(parameter)와 지역변수(local variable)

`main`에서 함수를 호출할 때 값을 넣으면,

그 값은 **복사되어 함수의 파라미터 변수에 들어간다.**

예를 들어:

```java
public static void test(int b) {
    int a = 123;
    System.out.println(b);
    System.out.println(a + b);
}

public static void main(String[] args) {
    int a = 10;
    int b = 20;
    int c = 30;

    test(c);
    System.out.println(b);
}
```

### 흐름 설명

- `main`이 먼저 실행됨
- `c`의 값은 `30`
- `test(c)` 호출 시
- `c`의 값 **30이 복사되어** `test`의 파라미터 `b`에 들어감
- 따라서 `test` 안의 `b`는 30

출력 결과 해석:

- `System.out.println(b);` → 30
- `System.out.println(a + b);` → 123 + 30 = 153
- `main`으로 돌아와 `System.out.println(b);` → 20

---

## 지역변수(local variable)

함수 안에서 만든 변수는 **지역변수**이다.

```java
int a = 10;
int b = 20;
int c = 30;
```

이 변수들은 `main` 함수 안에서 만든 것이므로 `main`의 지역변수이다.

---

## 같은 공간 안에서는 같은 이름 불가

```java
int a = 100;
```

이미 같은 함수 안에 `a`가 있으면

**Duplicate local variable a**

오류가 발생한다.

즉,

- **같은 블록(같은 공간)** 안에서는 같은 이름 사용 불가
- **서로 다른 함수** 안에서는 같은 이름 사용 가능

# 7. 값에 의한 호출(Call by Value)

```java
public static void test(int q) {
    System.out.println(q);
    q = 20;
    System.out.println(q);
}

public static void main(String[] args) {
    int q = 10;

    System.out.println(q);
    test(q);
    System.out.println(q);
}
```

### 출력 흐름

1. `main`의 `q` 출력 → 10
2. `test(q)` 호출
3. `main`의 `q` 값 10이 **복사**되어 `test`의 `q`로 들어감
4. `test`에서 `q` 출력 → 10
5. `test`에서 `q = 20`으로 변경
6. 다시 `test`에서 `q` 출력 → 20
7. `test` 종료 후 `main`으로 복귀
8. `main`의 원래 `q` 출력 → 10

### 핵심 해석

- `test` 함수는 **원본을 직접 바꾸는 것이 아니라**
- **복사본을 받아서 작업**한다.

즉 자바는 기본적으로 **값에 의한 호출(Call by Value)** 이다.

### 한 줄 요약

- `main`의 `q`와 `test`의 `q`는 **서로 다른 변수**
- 값만 복사되어 전달될 뿐이다.

# 8. 오버로딩(Overloading)

## 개념

오버로딩은

**함수명은 같게, 파라미터를 다르게 하는 기술**

이다.

호출할 때 어떤 값을 넣었느냐에 따라

알맞은 함수가 선택되어 실행된다.

---

## 오버로딩 성립 조건

1. **파라미터 개수가 다르면 가능**
2. 파라미터 개수가 같아도 **자료형이 다르면 가능**

대표적인 예시:

```java
System.out.println();
```

---

## 예시 문제

- 정수 1개를 넣으면 → `"윽!"`
- 문자열 1개를 넣으면 → `"악!"`
- 실수 1개를 넣으면 → `"으악!"`
- 아무것도 안 넣으면 → `"..."`
- 정수 2개를 넣으면 합 출력
- 정수 3개를 넣으면 합 출력
- 실수 3개를 넣으면 합 출력

---

## 오버로딩 형태로 정리한 코드

```java
public static void test(int x) {
    System.out.println("윽!");
}

public static void test(String x) {
    System.out.println("악!");
}

public static void test(double x) {
    System.out.println("으악!");
}

public static void test() {
    System.out.println("...");
}

public static void printCalc(int a, int b) {
    System.out.println(a + b);
}

public static void printCalc(int a, int b, int c) {
    System.out.println(a + b + c);
}

public static void printCalc(double a, double b, double c) {
    System.out.println(a + b + c);
}

public static void main(String[] args) {
    test(1);
    test("안");
    test(0.5);
    test();
}
```

### 메모

원래 함수명이 `test1`, `test2`, `test3`, `test4`로 나뉘어 있었는데,

**오버로딩의 핵심은 “함수명은 같고, 파라미터가 다른 것”** 이라서

위처럼 묶어두는 편이 더 이해가 잘 된다.

# 10. 함수 예제 2

문제:

- 랜덤한 정수를 **출력**하는 함수
- 랜덤한 정수를 **생성**하는 함수
- 정수를 하나 넣으면 홀수인지 짝수인지 **출력**하는 함수
- 정수를 두 개 넣었을 때 앞 숫자가 크면 `"앞"`, 뒷 숫자가 크거나 같으면 `"뒤"`를 **생성**하는 함수

```java
public static void red() {
    // 랜덤한 정수를 출력하는 함수
    Random r = new Random();
    int i = r.nextInt(5) + 1;

    System.out.printf("%s", "랜덤한 정수 출력:");
    System.out.println(i);
}

public static int blue() {
    // 랜덤한 정수를 생성하는 함수
    Random r2 = new Random();
    int i2 = r2.nextInt(5) + 1;

    System.out.printf("%s", "랜덤한 정수 생성:");
    System.out.println(i2);

    return i2;
}

public static void green(int x) {
    // 정수 하나를 넣으면 홀수/짝수 출력
    String result = (x % 2 == 0) ? "짝수" : "홀수";
    System.out.println(result);
}

public static String purple(int a, int b) {
    // 앞 숫자가 크면 '앞', 뒤 숫자가 크거나 같으면 '뒤' 생성
    String result = (a > b) ? "앞" : "뒤";
    System.out.println(result);

    return result;
}

public static void main(String[] args) {

    // 랜덤한 정수를 '출력'하는 함수
    red();

    // 랜덤한 정수를 '생성'하는 함수
    blue();

    // 정수 하나를 넣으면 홀수인지 짝수인지 출력하는 함수
    green(4);

    // 정수 두 개를 넣었을 때
    // 앞 숫자가 크면 '앞', 뒷 숫자가 크거나 같으면 '뒤' 생성하는 함수
    purple(50, 50);
}
```

# 12. 암기 핵심 정리

## 1) `main`

- 자바 프로그램 실행 시 **JVM이 자동 호출하는 함수**
- 실행 시작점

## 2) 함수 기본형

```java
public static 리턴타입 함수명(파라미터) {

    return 결과값;
}
```

## 3) `void`

- 반환할 값이 없을 때 사용

## 4) 파라미터

- 함수 실행에 필요한 재료

## 5) return

- 함수에서 만든 최종 결과를 호출한 곳으로 돌려준다

- 함수 안에서 만든 변수
- 그 함수 안에서만 사용 가능

## 7) 자바의 값 전달 방식

- **값에 의한 호출(Call by Value)**
- 원본이 아니라 **복사본**이 전달된다

## 8) 오버로딩(Overloading)

- **함수명은 같고**
- **파라미터의 개수 또는 자료형이 다르면 가능**

## 9) Random

```java
r.nextInt(5)      // 0 ~ 4
r.nextInt(5) + 1  // 1 ~ 5
```

## 10) 문자열 비교

- 문자열은 `==` 보다 **`.equals()`** 사용

## 11) 삼항연산자

```java
조건식 ? 참일때값 : 거짓일때값
```

예:

```java
String result = (x % 2 == 0) ? "짝수" : "홀수";
```

## 12) 계산식 괄호 조심

```java
(h - 100) * 0.9
```

- 괄호 없으면 계산 순서가 달라질 수 있음

## 1. Keras Tokenizer

Keras는 텍스트 전처리를 위한 다양한 도구를 제공한다. `Tokenizer`는 단어 빈도수를 기준으로 단어 집합을 만들고, 각 단어에 정수 인덱스를 부여한다.

```python
from tensorflow.keras.preprocessing.text import Tokenizer
```

```python
final_sentence = [['twinkle', 'twinkle', 'little', 'star'], ['wonder'], ['world', 'high'], ['like', 'diamond', 'sky'], ['twinkle', 'twinkle', 'little', 'star'], ['wonder'], ['blazing', 'sun', 'gone'], ['nothing', 'shines', 'upon'], ['show', 'little', 'light'], ['twinkle', 'twinkle', 'night'], ['twinkle', 'twinkle', 'little', 'star'], ['wonder'], ['making', 'circle'], ['making', 'circle'], ['turning', 'turning', 'circle'], ['dance', 'along'], ['let', 'clap', 'hands'], ['let', 'sing', 'along'], ['lalalala', 'fun', 'dance', 'along'], ['lingalingalinga', 'lingalingalinga'], ['hand', 'hand', 'together', 'everybody']]
print(final_sentence) # '''단어''', '단어' 가 아니라 [[단어]]로 해야함
```

```python
tokenizer = Tokenizer()

# fit_on_texts(코퍼스) :
#   빈도수를 기준으로 단어 집합을 만들어줌
# 높은 빈도수의 단어부터 낮은 정수값을 부여
tokenizer.fit_on_texts(final_sentence)
```

```python
# word_index : 각각 어떤 인덱스가 부여됐는지
print(tokenizer.word_index)
```

```python
# word_counts : 단어들의 빈도수 확인
print(tokenizer.word_counts)
```

```python
# texts_to_sequences(코퍼스) :
#   text를 부여받은 인덱스로 변환
print(tokenizer.texts_to_sequences(final_sentence))
```

`fit_on_texts()`는 단어 집합을 만들고, `word_index`는 단어별 인덱스, `word_counts`는 빈도수, `texts_to_sequences()`는 문장을 정수 시퀀스로 변환한다.

## 2. 빈도수 상위 단어만 사용하기

`num_words`를 사용하면 빈도수가 높은 상위 단어만 시퀀스 변환에 사용할 수 있다. 불필요한 단어를 줄이면 데이터 크기와 노이즈를 줄일 수 있다.

```python
# 빈도수가 높은 상위 5개만 사용하겠다 ~
#   라고 지정하는 방법
# tokenizer = Tokenizer(num_words=숫자)
word_size = 5

# num_words는 count를 0부터 새서 여기에 +1만큼 더해줄것임
tokenizer = Tokenizer(
    num_words = word_size +1

)

tokenizer.fit_on_texts(final_sentence)
```

```python
print(tokenizer.word_index)
```

```python
print(tokenizer.word_counts)
```

```python
print(tokenizer.texts_to_sequences(final_sentence))
```

상위 5개 단어를 사용하고 싶을 때 `word_size + 1`처럼 작성하는 이유는 인덱스 `0`을 패딩용으로 남겨두는 경우가 많기 때문이다.

## 3. OOV 처리

OOV는 `Out Of Vocabulary`의 약자로, 단어 집합에 없는 단어를 의미한다. 기본적으로 Keras Tokenizer는 OOV 단어를 제거하지만, `oov_token`을 설정하면 모르는 단어를 하나의 토큰으로 유지할 수 있다.

```python
# keras의 tokenizer에 단어 집합에 없는 단어의
# OOV (Out-Of-Vocabulary) 에 대해서는
#   단어 자체를 제거하는 특성

# OOV를 따로 설정해서 가지고 있고 싶다면
#   'oov_token' 이라는 옵션을 사용

freSize = 5
tokenizer = Tokenizer(num_words=freSize +2,
                      oov_token='OOV') # oov가 들어갈 자리까지 추가로 +1+!1 = +2

tokenizer.fit_on_texts(final_sentence)
```

```python
tokenizer.word_index['OOV']
```

```python
# oov_token 을 사용하면
#     oov의 인덱스가 1로 설정
# 나머지 빈도수 상위 5개 (단어 집합)
#   자동적으로 인덱스가 1씩 추가됨
print(tokenizer.texts_to_sequences(final_sentence))
```

`oov_token='OOV'`를 사용하면 OOV 토큰이 보통 인덱스 `1`을 차지하고, 기존 단어들의 인덱스는 뒤로 밀린다.

전체 흐름은 원문 정리 → 형태소 분석 → 불용어 제거 → 한 글자 제거 → 빈도수 계산 → 빈도수 내림차순 정렬 → 정수 인덱스 생성이다.

## 6. Keras의 pad_sequences

직접 반복문으로 패딩할 수도 있지만, Keras의 `pad_sequences()`를 쓰면 더 간단하게 처리할 수 있다.

```python
from tensorflow.keras.preprocessing.sequence import pad_sequences
```

```python
# 위에 있는 aa를 원상태로 복구
aa = tokenizer.texts_to_sequences(final_sentence)
print(aa)
```

```python
# 데이터의 앞에서부터 0을 채움

#kerasPadding = pad_sequences(aa) # 이렇게해도, 똑같고 (위랑)

#데이터의 뒤에 0을 채움
kerasPadding = pad_sequences(aa,
                             padding='post') # 이렇게해도 똑같음
kerasPadding
```

```python
# 기준을 최대길이로 할 필요가 없는 경우에
# 'maxlen' 을 이용해서 원하는 길이 조절 가능
# 원하는 길이보다 긴 문장이 있는 경우 ,
#   데이터가 해당하는 갯수만큼 손실
# 기본적으로 문장의 맨 앞에서부터 차례대로
#   데이터의 손실이 발생하고,
# 데이터 손실을 맨 뒤에서부터 진행하고 싶다면
#   truncating = 'post' 옵션을 추가
kerasPadding2 = pad_sequences(aa,
                              padding='post',
                              maxlen=3,
                              truncating='post')
kerasPadding2
```

`padding='post'`는 뒤쪽에 0을 채우고, `maxlen`은 최대 길이를 제한한다. `truncating='post'`는 길이가 초과되는 경우 뒤쪽 데이터를 자른다.

## 7. One-Hot Encoding 직접 구현

원-핫 인코딩은 단어 집합 크기만큼의 벡터를 만들고, 해당 단어의 인덱스 위치만 `1`, 나머지는 `0`으로 표시하는 방식이다.

```python
!pip install konlpy
```

```python
from konlpy.tag import Okt
```

```python
okt = Okt()
word_token2 = okt.morphs("오늘은 어째서 금요일이 아니죠?")
print(word_token2)
```

```python
# 나눠진 단어에 각각 index 부여하기
#   (0부터 시작하는 ...)
word_index2 = {
    w : i for i, w in enumerate(word_token2)

}

print(word_index2)
```

```python
# 토큰을 입력하면,
# 토큰에 대한 원-핫 벡터를 뽑아내는 함수
def ohe(w, word_index):
  # 위에서 만든 index의 길이만큼
  # 0을 채운 list 생성
  ohVector = [0] * (len(word_index))

  # 파라미터로 받아온 단어의 위치를 담는 변수
  index = word_index[w]

  # 해당 값이 있는 위치를 값을 1로 변경
  ohVector[index] = 1
  return ohVector
```

```python
# '금요일' 이라는 단어의 원-핫 벡터
ohe('금요일', word_index2)
```

예를 들어 `금요일`의 인덱스가 `3`이라면 원-핫 벡터는 `[0, 0, 0, 1, 0, 0, 0]` 형태가 된다.

## 8. Keras를 이용한 One-Hot Encoding

Keras의 `to_categorical()`을 사용하면 정수 시퀀스를 원-핫 벡터로 쉽게 변환할 수 있다.

```python
from tensorflow.keras.preprocessing.text import Tokenizer
from tensorflow.keras.utils import to_categorical
```

```python
text = "곰 세 마리가 한 집에 있어 아빠 곰 엄마 곰"
text

# 빈도수 기준의 단어 집합 만들기
tokenizer = Tokenizer()

# fit_on_texts 함수는
#     리스트 형태의 데이터입력을 요구
tokenizer.fit_on_texts([text])
print(tokenizer.word_index)
```

```python
# 생성된 단어 집합 내의 일부 단어들로만
#     구성된 서브 텍스트를 하나 생성
text2 = "아빠 곰 세 마리가"

# texts_to_sequences 는 리스트 형식의 입력을 요구
# 이 함수는 입력 리스트의 각 요소(텍스트)를 숫자로 변환해서 리스트 형태로 반환
# 이 함수 반환 값이 [[7, 1, 2, 3]] 인 이중 리스트
# => 단일 텍스트 변환 결과를 얻기 위해서
#   이중 리스트의 0번째 요소로 가져올 것
bear = tokenizer.texts_to_sequences([text2])
print(bear)
```

```python
# to_categorical()
#   : keras 의 원-핫 인코딩하기 위한 함수
cate = to_categorical(bear)
print(cate)

# [7, 1, 2, 3] 을 원-핫 인코딩으로 표현한 결과
# 1. 단어 집합의 인덱스 시작 숫자는 1
# 2. 컴퓨터는 0부터 시작
# 3. 각 list의 0번째 자리는
#     임의적으로 0으로 채워두고
# 4. 각각의 인덱스에 해당하는 자리의 값이
#     1값으로 채워지도록.
```

흐름은 텍스트 입력 → `Tokenizer`로 단어 인덱스 생성 → `texts_to_sequences()`로 정수 시퀀스 변환 → `to_categorical()`로 원-핫 벡터 변환이다.

## 9. One-Hot Encoding 장점과 단점

# 장점 :

문자 데이터의 경우에 컴퓨터가 처리하기 쉽게 숫자로 변환

# 단점 :

1. 단어의 갯수가 늘어날수록, 벡터를 저장하기 위한 공간이 늘어남
2. 원-핫 벡터는 단어들의 유사도 를 표현하지 못한다

(추천 시스템의 경우)

특정 검색어에 대해서 유사 단어에 대한 결과도 함께 보여줄 수 있어야 하는데,
단어간의 유사성을 계산할 수 없다면 연관 검색어를 보여줄 수 없을 것 !

이러한 단점을 해결하기 위해서 단어의 잠재 의미를 반영해서
다차원 공간에 벡터화 하는 기법을 사용할건데 =>

대표적으로 예측 기반으로 벡터화하는 <font color="red">Word2Vec</font>이 있음

## 10. 전체 복습

텍스트 전처리의 기본 흐름은 다음과 같다.

```
문장 데이터
→ 토큰화
→ 정수 인코딩
→ OOV 처리
→ 패딩
→ 원-핫 인코딩
→ 모델 입력
```

핵심 함수와 역할은 다음과 같다.

```
Tokenizer      = 단어를 정수 인덱스로 변환
fit_on_texts   = 단어 집합 생성
word_index     = 단어별 인덱스 확인
word_counts    = 단어별 빈도수 확인
texts_to_sequences = 문장을 정수 시퀀스로 변환
num_words      = 사용할 상위 단어 개수 제한
oov_token      = 모르는 단어 처리
Okt            = 한국어 형태소 분석
stop_word      = 불용어 제거
pad_sequences  = 문장 길이 맞추기
to_categorical = 원-핫 인코딩
Word2Vec       = 의미 기반 벡터화
```

정리하면, 텍스트 전처리는 문자를 숫자 시퀀스로 바꾸고, 길이를 맞춘 뒤, 모델이 처리 가능한 벡터 형태로 변환하는 과정이다.

# 1. com.beaver.may191

## 핵심

XML에 Bean을 등록하고, Spring 컨테이너에서 객체를 꺼내 사용하는 방식.

흐름:

```
may191beans.xml에 Bean 등록
→ ClassPathXmlApplicationContext로 XML 읽기
→ getBean()으로 객체 가져오기
→ 기본형 / 배열 / List / Map / 객체참조 / @Autowired 확인
```

## 1-1. HomeController.java

```java
package com.beaver.may191;

import java.text.DateFormat;
import java.util.Date;
import java.util.Locale;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.context.support.AbstractApplicationContext;
import org.springframework.context.support.ClassPathXmlApplicationContext;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;

import com.beaver.may191.human.Human;

@Controller
public class HomeController {

	private static final Logger logger = LoggerFactory.getLogger(HomeController.class);

	@RequestMapping(value = "/", method = RequestMethod.GET)
	public String home(Locale locale, Model model) {

		AbstractApplicationContext aac = new ClassPathXmlApplicationContext("may191beans.xml");
		aac.registerShutdownHook();

		Human h1 = aac.getBean("h1", Human.class);
		System.out.println(h1.getName());
		System.out.println(h1.getAge());

		for (String nn : h1.getNickname()) {
			System.out.println(nn);
		}

		for (String f : h1.getFriend()) {
			System.out.println(f);
		}

		System.out.println(h1.getFamily().get("아버지"));
		System.out.println(h1.getFamily().get("어머니"));
		System.out.println(h1.getPet().getName());
		System.out.println(h1.getPet().getAge());
		System.out.println(h1.getPet2().getName());
		System.out.println(h1.getPet2().getAge());
//
//		aac = new ClassPathXmlApplicationContext("may191beans.xml");
//		aac.registerShutdownHook();

		aac.close();

		return "home";
	}

}

// 우리 컨셉상 웹 화면에서 값들을 입력받아서 그것을가지고 활용을 할건데..
// 지금 이렇게 고정된 값들을 가지고는 힘들다.
// 그래서 지금과 같은 방식은 특정 기능이나 고정값으로 활용할 때 사용하면 좋다.
```

### 메모

- `ClassPathXmlApplicationContext`는 XML 설정 파일을 읽어서 Bean을 생성하는 Spring 컨테이너.
- 웹 프로젝트에서는 보통 Controller 안에서 매번 컨테이너를 직접 만들지 않는다.
- 현재 방식은 XML Bean 등록과 DI 구조를 이해하기 위한 예제.

## 2-1. HomeController.java

```java
package com.beaver.may192;

import java.text.DateFormat;
import java.text.SimpleDateFormat;
import java.util.Date;
import java.util.Locale;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.beans.factory.annotation.Qualifier;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;

import com.beaver.may192.fruit.Fruit;

// Fruit 클래스 => 이름 / 가격
@Controller
public class HomeController {

	private static final Logger logger = LoggerFactory.getLogger(HomeController.class);

	 //현재 @AUtowired 된 Fruit f, f2 필드는
		// xml의 어떤 bean을 사용할지 지정하지 안아서 오류 발생 가능성이 있음.
		// 따라서 @Qualifier 를 사용해서 명확하게 어떤 bean을 주입할지 지정할 수 있음.
	@Autowired
	@Qualifier("f")
	private Fruit f;

	@Autowired // 따로 명시해주지 않으면 보통 오류남.
	@Qualifier("f_new")
	private Fruit f_new;

	@Autowired
	private SimpleDateFormat sdf;

	@Autowired
	private MemberDAO mDAO;

	public void test() {
		Date now = new Date();
		System.out.println(sdf.format(now));
	}

	@RequestMapping(value = "/", method = RequestMethod.GET)
	public String home(Locale locale, Model model) {
		System.out.println(f.getFr_name());
		System.out.println(f.getFr_price());
//		System.out.println(f_new.getFr_name());
		System.out.println(f_new.getFr_price());

		return "home";
	}

}
```

### 메모

`Fruit` 타입 Bean이 `f`, `f_new` 두 개라서 `@Qualifier`로 명확히 지정한다.

```
@Autowired
→ 타입 기준 자동 주입

@Qualifier
→ Bean id 기준으로 선택
```

| 설정 | 의미 |
| --- | --- |

| `ViewResolver` | 문자열을 JSP 경로로 변환 |
| `<beans:bean>` | 객체 Bean 등록 |
| `<context:component-scan>` | Annotation 클래스 자동 등록 |

# 3. com.beaver.may193

## 핵심

Spring MVC의 Controller 구조와 요청 파라미터 처리 방식.

```
JSP Model 2
→ Servlet 중심

Spring MVC
→ 일반 클래스 + 일반 메소드 + Annotation 중심
```

## 3-1. @Controller

사이트의 첫 접속시 method : GET

```java
public class HomeController {
```

### 메모

`@Controller`는 요청을 받아 처리하고 View로 흐름을 넘기는 클래스에 붙인다.

---

## 3-2. JSP Model 2

http://IP:port/프로젝트 명/컨트롤러파일명(@webservlet)

C : Servlet

상황판단 => 흐름을 제어

doGet(), doPost()를 활용

주소에 컨트롤러 파일명, 컨트롤러 하나당 GET 하나, POST 하나

⇒ 컨트롤러의 수가 많아짐

---

## 3-3. SpringMVC

http://IP:port/top-level-package의 마지막자리/임의대로 지정한 값

C : 일반 클래스

일반 메소드

컨트롤러 하나에 여러개의 요청 처리가 가능 O

⇒ 컨트롤러 수가 많이 필요 없어짐

---

## 3-4. 비교

| 구분 | JSP Model 2 | Spring MVC |
| --- | --- | --- |
| Controller | Servlet | 일반 Java 클래스 |
| 메소드 | doGet/doPost | 일반 메소드 |
| URL 매핑 | @WebServlet | @RequestMapping |
| 특징 | 컨트롤러 많아짐 | 하나의 컨트롤러에서 여러 요청 처리 가능 |

## 3-5. HomeController.java

```java
package com.beaver.may193;

import java.text.DateFormat;
import java.util.Date;
import java.util.Locale;

import org.slf4j.Logger;
import org.slf4j.LoggerFactory;
import org.springframework.stereotype.Controller;
import org.springframework.ui.Model;
import org.springframework.web.bind.annotation.RequestMapping;
import org.springframework.web.bind.annotation.RequestMethod;

@Controller
// 사이트의 첫 접속시 method : GET
public class HomeController {

// JSP Model 2 ---------------------------
// <http://IP>:port/프로젝트 명/컨트롤러파일명(@webservlet)
// C : Servlet
//		상황판단 => 흐름을 제어
// 		doGet(), doPost()를 활용
//		주소에 컨트롤러 파일명, 컨트롤러 하나당 GET 하나, POST 하나
//				=> 컨트롤러의 수가 많아짐
//	--------------------------------------

// SpringMVC -----------------------------
// <http://IP>:port/top-level-package의 마지막자리/임의대로 지정한 값
// C : 일반 클래스
// 		일반 메소드
// 		컨트롤러 하나에 여러개의 요청 처리가 가능 O
//			=> 컨트롤러 수가 많이 필요 없어짐
	@RequestMapping(value= "/", method = RequestMethod.GET)
	public String goHome() { // String 자료형이 기본
		return "index"; // index.jsp 로 포워딩
	}
	// hehehe.test 라는 주소로 GET 방식 의 요청이 들어오면
	//index.jsp로 포워딩(이동)해주는 메소드
	@RequestMapping(value = "/hehehe.test", method = RequestMethod.GET)
	public String testHehehe() {

	@RequestMapping(value = "/lalala.test", method = RequestMethod.GET)

```

### 메모

```
@RequestMapping(value="/", method=GET)
→ 해당 주소로 GET 요청이 오면 메소드 실행
```

## 4-4. CalcResult.java

<aside>
💡

요청파라미터명과 JavaBean 멤버변수명 같게 해야함.

그래야, 자동으로 요청파라미터의 값을 Javabean 객체에 ... 채워넣어줌

```java
package com.beaver.may193.calc;

// 요청파라미터명과 JavaBean 멤버변수명 같게
// 자동으로 요청파라미터의 값을 Javabean 객체에 ... 채워넣어준다
public class CalcResult {
	private String n;
	private int x;
	private int y;

	public CalcResult() {

	}

	public CalcResult(String n, int x, int y) {
		super();
		this.n = n;
		this.x = x;
		this.y = y;
	}

	public String getN() {
		return n;
	}

	public void setN(String n) {
		this.n = n;
	}

	public int getX() {
		return x;
	}

	public void setX(int x) {
		this.x = x;
	}

	public int getY() {
		return y;
	}

	public void setY(int y) {
		this.y = y;
	}

}
```

### 메모

자동 바인딩 조건:

| form name | JavaBean 변수 |
| --- | --- |
| `n` | `n` |
| `x` | `x` |
| `y` | `y` |

```
사용자 요청

→ Controller
→ Service / DAO
```

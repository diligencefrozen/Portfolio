# GPT 란?

> OpenAI 가 개발한 대화형 인공지능 모델로
> 
> 
> GPT (Generative Pre-trained Transformer) 기술을 기반으로,
> 
> 사용자가 자연스러운 대화를 나눌 수 있는 AI chatbot 임.
> 

이 GPT는 자연어 처리를 위한 AI 모델로, 크게 3가지 특징을 가짐

## 1. Generative (생성형)

- 주어진 입력을 바탕으로 새로운 텍스트를 생성할 수 있음.
- (질문에 대한 답이나 이야기 등을 작성하는 것이 가능함)

## 2. Pre-trained (사전학습)

- 대규모의 텍스트 데이터가 미리 학습되어 있어,
    
    다양한 주제와 그 맥락을 이해하고 처리하는 것이 가능함.
    

## 3. Transformer (트랜스포머)

- 텍스트의 맥락을 효과적으로 이해하고 처리하는 신경망의 구조를 사용함으로 인해
    
    더 자연스럽고 정확한 언어의 처리가 가능함.
    

→ 이러한 특징을 바탕으로 GPT는 텍스트 생성, 번역, 요약, 질의응답 등

다양한 언어 관련한 작업들을 할 수 있음.

# 주요 기능

## 검색

- 인터넷에서 최신 정보를 찾거나 주제에 대한 관련 내용을 조사

## 이성

- ‘이성’이 명확하지 않고 추상적이지만, (여기선) 논리적, 합리적 사고를 의미함.
- GPT는 이성적인 답변을 제공하는데에 초점을 맞추고 있음.

## 심층 리서치

- 복잡한 주제나 문제에 대해 깊이 있는 정보나 분석을 제공
- 다양한 자료들을 참고하여 체계적으로 분석 결과를 제공

## 이미지 그리기

- GPT는 텍스트 기반의 모델이지만,
- 이미지 생성 도구 (ex, DALL-E) 와 통합되어있는 경우
- 텍스트로 설정한 장면이나 아이디어를 이미지로 변환할 수 있음.

## 캔버스

- 텍스트와 코드 작업을 위한 실시간 협업 도구
- 사용자가 프로젝트를 생성하고, 내용을 편집하며,
- 실시간으로 피드백을 받을 수 있는 환경을 제공
- 긴 글 작성이나, 복잡한 코드 작업에 유용함.

## 음성입력

- 음성 인식 기능을 통해 사용자가 음성으로 입력할 수 있는 기능
- 이를 통해 음성으로 명령하거나 질문할 수 있음.

## 음성모드

- GPT가 음성으로 응답을 제공하는 기능

# GPT 프롬프트 공식 및 효율적인 GPT 스크립트 작성 법

> 해당 모델에는 다양한 분야의 데이터들이 담겨져 있고,
> 
> 
> 우리가 활용할 때에 조금 더 정확한 결과를 얻어내고자 한다.
> 

## 1. 페르소나 (Persona)

- GPT가 맡을 역할이나 직무를 정의
- “너는 데이터베이스 설계 전문가야”
- “너는 경험많은 자바 개발자야”
- “너는 UI/UX 디자이너로 사용자의 경험을 최우선으로 고려해”

## 2. 작업 (Task)

- GPT가 수행해야 할 구체적인 작업 설명
- “이 코드를 최적화해줘”
- “이 문장을 간단하게 요약해줘”
- “메뉴 등록 시스템의 ERD을 그려줘”

## 3. 맥락 (Context)

- 작업을 수행하는데 필요한 배경 정보나 조건 제공
- “이 프로젝트는 웹 애플리케이션이고 ~~ 기능과 ~~ 메뉴가 필요해”
- “이 코드는 Java XX버전에서 실행되어야 해”
- “이 문서의 대상은 초보자에게 기술을 설명하기 위해서야”

## 4. 예시 (Example)

- 작업 결과나 원하는 형태를 보여주는 예시를 제공
- “다음과 같은 결과를 원해 : x=10, y=20”
- “다음과 같은 ERD를 원해: [예시로 제공할 링크 or 파일]”
- “다음과 같은 방식으로 요약해줘: ‘요약 예시’”

## 5. 형식 (Format)

- 결과를 어떤 형식으로 출력할지 정의
- “결과를 JSON 형식으로 출력해줘”
- “표 형식으로 요약해줘”

## 6. 톤 (Tone)

- GPT 대화 스타일이나 감정적인 톤 설정

> 프롬프트를 작성할 때 위의 사항을 잘 고려하여 구성하면
> 
> 
> 더 정확하고 원하는 결과를 얻어낼 수 있음.
> 

### ex)

“너는 경험 많은 자바 개발자야 다음 제공하는 코드를 최적화해줘.

이 코드는 자바 11버전에서 실행되었고,

최적화한 한 코드에 대해서만 코드블록으로 출력해줘.”

# [추가]

- 앞서 질문했던 내용들을 모델이 기억을 하고 있기 때문에
    
    큰 틀의 내용은 제외하고 추가할 상세 사항에 대하여 질문한다.
    
- 구체적, 세부적인 질문일수록 좋다.
- 하지만, 직접 만들어보고 난 후에 코드 정리를 요청해볼 것.
- 기능에 대한 정리를 해주었을 때 모르는 코드가 나타나면
    
    지나치지 말고 그 부분에 대한 설명을 요구하면 주석으로 설명해줌.
    

## 예시 질문

- 이 코드가 어떤 기능을 하는지 알려줘.
- 이 코드에 대한 자세한 설명이 필요해.

## 오류를 찾지 못할 경우

본인의 코드 오류에 대한 부분을 잘 찾아내지못할 경우

- 해당 문제의 텍스트 및 코드와 함께
    
    ‘왜 이런 오류가 발생하는지 알려줘’
    
- 또는,
    
    ‘이 오류에 대한 원인과 해결방법에 대해 알려줘’
    

같은 질문을 던져서 해결 후에 그에 대한 해법을 숙지하여

다음번에 같은 실수에 대한 재발을 방지.

## 질문 방법 팁

- 주제를 나눠서 질문하기
- 응답 길이에 제한이 있으므로 너무 길게 실명하지 않도록 중간에 끊어서 응답을 유도하기
    
    ex) ~까지 설명하고 ‘다음’ 입력하면 계속 설명해줘.
    
- GPT가 절대적으로 실행이 되게끔 만들어주지는 않으므로
    
    테스트도 해보고, 문제 발생 시 직접 해결할 수 있는 능력을 키워야한다. (교차 검증 능력)


### 의미

- `/` : 현재 로그인 한 OS의 계정을 쓰겠다.


- 아이디 : test123
- 비밀번호 : 123456

```sql
grant connect, resource, dba to [ID명];
```


→ 이렇게 하면 원격 관리가 가능해짐.

# 용어 정리 !!

## 테이블(=릴레이션)

- 데이터베이스에서 정보를 구분하여 저장하는 기본 단위
- 행과 열로 이루어진 데이터의 집합

## 행 (Row)

- 테이블을 구성하는 데이터들 중 가로 부분 (→)
- 튜플 (Tuple) 또는 레코드 (Record) 라고도 부름.

## 열 (Column)

- 테이블을 구성하는 데이터들 중 세로 부분
- 필드 (Field) 또는 속성 (Attribute) 라고도 부름

## 스키마 (Schema)

- 데이터베이스에서 기본 구조를 정의하는 것. (ex) 헤더 영역)
- 쉽게 말하자면, 제목 부분 (이름, 나이, 성별)
- 테이블 헤더를 나타내며, 속성과 자료타입을 갖고 있음.

## 인스턴스 (instance)

- 테이블에 실제로 저장된 데이터의 값

## 도메인 (Domain)

- 속성이 가질 수 있는 값의 집합
- 특정 속성에서 사용할 데이터의 범위를 사용자가 정의

예시:

- `p_age number(3)` 이면 정수를 3자리까지만 받겠다는 의미
    
    3자리를 넘어가는 값이라면 → 오류!!!
    
- `p_name varchar2(10 char)`면
    
    문자열을 최대 10글자까지 받겠다는 의미
    
    10글자를 넘어가면 → 오류!

# 테이블(릴레이션)의 특징

## 1. 속성은 단일 값만 가진다.

- 각 속성의 값은 도메인에 정의된 값만 가지며
- 그 값은 모두 단일한 값이어야 함.

예시:

- `p_age` : 최대 3자리 정수로 된 값만 가지며,
- 그 값은 모두 단일한 값이어야 함.

## 2. 속성은 서로 다른 이름을 가진다.

- 각 속성마다 `p_name`, `p_age`, `p_gender` 라는 이름을 가지고 있다.
- 이름을 중복해서 만들지 않기!

예시: 컬럼명 중복 오류!

```sql
create table test {
    t_no number(3),
    t_no varchar2(10 char)
);
```

## 3. 한 속성의 값은 모~~두 같은 도메인의 값을 가진다.

→ 1번이랑 같은 맥락으로 `p_age` : 최대 3자리 정수로 된 값을 가짐

## 4. 속성의 순서는 상관없다.

- 속성의 순서가 달라도 테이블 스키마는 같음.

예시:

```sql
select * from person;
select p_age, p_gender, p_name from person;
```

순서만 바뀌었지 내용에는 변화가 없다.

## 5. 테이블 내의 중복된 튜플은 허용되지 않는다.

- 홍길동에 대한 정보가 한 번 더 들어가면
- 동일인물인지 다른인물인지 알 수가 없음.
- 즉, 모든 튜플에는 차별성이 있어야함!
- (생일값이나, 핸드폰 번호값이나..)

# SQL 명령문의 종류 !

## 데이터 정의어 (DDL / Data Definition Language)

- 테이블이나 관계의 구조를 생성하는데 사용하며,
- `Create`, `Alter`, `Drop`, ... 등이 있음

## 데이터 조작어 (DML / Data Manipulation Language)

- 테이블에 데이터를 검색, 삽입, 수정, 삭제 하는데 사용하며,

```
Select, Insert, Update, Delete
```

- 줄여서 CRUD 라고 함

### CRUD

- Create
- Read
- Update
- Delete
- `Select` 문은 특별히 질의어 (Query) 라고 함

## 데이터 제어어 (DCL / Data Control Language)

- 데이터베이스에 접근하고 객체들을 사용하도록 권한을 주고 회수하는데 사용하며,

```
GRANT, Revoke
```

- 제어어는 DBA(DataBase administrator)의 업무.

## 트랜잭션 처리어 (TCL / Transaction Control Language)

- DML에 의해 조작된 결과를
- 작업 단위(트랜잭션) 별로 제어하는데 사용

```
Commit, Rollback, Savepoint
```

# 암기

## 1. GPT란?

- **GPT = Generative Pre-trained Transformer**
- OpenAI가 개발한 **대화형 인공지능 모델**
- 자연어를 이해하고, 텍스트를 생성하는 AI 챗봇

### 3가지 핵심 특징

1. **Generative(생성형)**
    
    입력을 바탕으로 새로운 텍스트 생성
    
2. **Pre-trained(사전학습)**
    
    대규모 텍스트를 미리 학습
    
3. **Transformer(트랜스포머)**
    
    문맥을 잘 이해하는 신경망 구조 사용
    

---

## 2. GPT 주요 기능

- **검색**: 최신 정보 조사
- **이성**: 논리적, 합리적 답변
- **심층 리서치**: 복잡한 주제 분석
- **이미지 그리기**: 텍스트를 이미지로 변환
- **캔버스**: 텍스트/코드 협업 공간
- **음성입력**: 말로 질문 가능
- **음성모드**: 음성으로 답변 가능

---

## 3. GPT 프롬프트 공식

정확한 결과를 얻기 위해 아래 6가지를 넣으면 좋음.

1. **페르소나(Persona)**
    
    GPT가 맡을 역할
    
    예: “너는 데이터베이스 설계 전문가야”
    
2. **작업(Task)**
    
    무엇을 할지
    
    예: “이 코드를 최적화해줘”
    
3. **맥락(Context)**
    
    배경/조건
    
    예: “이 프로젝트는 웹 애플리케이션이야”
    
4. **예시(Example)**
    
    원하는 결과 예시 제시
    
5. **형식(Format)**
    
    출력 형식 지정
    
    예: JSON, 표 형식
    
6. **톤(Tone)**
    
    답변 스타일 지정

## 4. GPT 질문 팁

- 큰 질문보다 **작게 나눠서 질문**

    **“원인과 해결방법 알려줘”** 식으로 질문
    
- GPT 답변은 무조건 정답이 아니므로 **테스트와 검증 필요**

# 16. 용어 정리

- **테이블(릴레이션)**: 정보를 저장하는 기본 단위, 행과 열의 집합
- **행(Row)**: **가로 한 줄**, 튜플/레코드
- **열(Column)**: **세로 한 칸**, 필드/속성
- **스키마(Schema)**: 테이블의 구조, 헤더 부분, 속성과 자료형 정의
- **인스턴스(Instance)**: 테이블에 실제 저장된 데이터 값
- **도메인(Domain)**: 속성이 가질 수 있는 값의 범위

# 20. SQL 명령문 종류

## 1) DDL (데이터 정의어)

- 구조 생성/수정/삭제
- `CREATE`, `ALTER`, `DROP`

## 2) DML (데이터 조작어)

- 데이터 검색/삽입/수정/삭제
- `SELECT`, `INSERT`, `UPDATE`, `DELETE`

→ **CRUD**

- Create
- Read
- Update
- Delete

## 3) DCL (데이터 제어어)

- 권한 부여/회수
- `GRANT`, `REVOKE`

## 4) TCL (트랜잭션 처리어)

- DML 결과를 작업 단위로 제어

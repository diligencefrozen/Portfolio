# XML 파싱 / Open API / 요청 헤더 / URL 인코딩

## 1. 웹서버와 Open API 기본 개념

### 웹서버

- 클라이언트가 웹사이트에 요청하면, 서버가 응답하는 구조
- DB에 있는 데이터를 사람들에게 보여줘야 할 때 사용
- 대부분의 사용자는 SQL을 직접 다루지 못하므로, 웹사이트가 DB를 쉽게 사용할 수 있게 중간 역할을 함

### 개발자 관점

- 개발자는 DB 원본 자체보다 **필요한 데이터만 특정 형식으로 받아오고 싶다**
- 이때 많이 사용하는 데이터 표현 형식:
    - XML
    - JSON

### 실시간 데이터 처리 흐름

1. 클라이언트가 데이터 요청
2. 서버가 XML 또는 JSON으로 응답
3. 필요한 데이터만 파싱
4. 나중에 화면에 시각화하여 구현

### Open API 예시 사이트

- data.go.kr
- data.seoul.go.kr
- dev.naver.com
- dev.kakao.com

## 2. XML 기본 개념

### XML

**XML (eXtensible Mark-up Language)**

- DB에 있는 데이터를 구조적으로 표현한 형식
- HTML처럼 태그를 사용해서 데이터를 감쌈

### 기본 구조

```xml
<xxx>
    yyy
</xxx>
```

- `<xxx>` : 시작 태그
- `yyy` : 텍스트
- `</xxx>` : 종료 태그

### 예시

```xml
<RealtimeCityAir>
    <RESULT>
        <CODE>INFO-000</CODE>
        <MESSAGE>정상 처리되었습니다</MESSAGE>
    </RESULT>
</RealtimeCityAir>
```

## 6. XmlPullParser 이벤트 종류

### 주요 이벤트

- `START_DOCUMENT` : 문서 시작
- `END_DOCUMENT` : 문서 끝
- `START_TAG` : 시작 태그
- `END_TAG` : 종료 태그
- `TEXT` : 태그 사이의 실제 텍스트

### 특징

- 위 상수들은 각각 고유한 정수값을 가짐

## 7. XML 파싱으로 특정 태그 값만 꺼내기

### 목표

미세먼지 XML에서 `MSRSTN_NM` 태그 값을 추출해서 출력

### 핵심 흐름

1. XML 문서를 읽음
2. 현재 이벤트 타입을 확인
3. 시작 태그면 태그 이름 저장
4. 텍스트면 현재 태그 이름과 비교
5. 원하는 태그면 값 출력

### 코드

```java

// 대부분의 사람들이 SQL을 할 줄 모르는데,,,

// 웹 사이트 : 보통 사람들이 DB 쓰기 편하게

// 개발자가 데이터만 가져오고 싶음

// 데이터를 특정한 형식으로 표현해줘야 하는데

// 실시간 데이터를 XML Parsing ( 클라이언트가 데이터를 요청 => XML / JSON 응답 )

// ( 나중에 ) 그 데이터를 가지고 웹에 이미지화를 시켜서 구현

// data.go.kr

// data.seoul.go.kr

// dev.naver.com

// dev.kakao.com

// http://openapi.seoul.go.kr:8088/4f626857416f6c6c3632586a416843/xml/RealtimeCityAir/1/25/

public class AirPollutionMain1 {

	public static void main(String[] args) {
		try {
			String address = "http://openapi.seoul.go.kr:8088/4f626857416f6c6c3632586a416843/"
					+ "xml/RealtimeCityAir/1/25/";
			InputStream is = JeesungHttpClient.download(address);

//		    String result = JeesungHttpClient.convert(is, "UTF-8");
//		    System.out.println(result);

			XmlPullParserFactory xppf = XmlPullParserFactory.newInstance();
			XmlPullParser xpp = xppf.newPullParser();
			xpp.setInput(is, "UTF-8");

			int type = xpp.getEventType();
			String tagName = null;

			while (type != XmlPullParser.END_DOCUMENT) {
				if (type == XmlPullParser.START_TAG) {
					// == 를 != 으로 잘못 입력해서, 오류가 남.
					tagName = xpp.getName();
				} else if (type == XmlPullParser.TEXT) {
					if (tagName.equals("MSRSTN_NM")) {
						System.out.printf("측정 구 : %s\n", xpp.getText());
					}
				} else if (type == XmlPullParser.END_TAG) {
					tagName = "";
					// 모든 과정에서 공백이나 줄바꿈이 일어나면서
					// 그 값을 tagName 으로 인식하는 경우가 있음.
				}

				xpp.next(); // 다음 항목으로 넘어감
				type = xpp.getEventType(); // 다음 태그의 값을 가짐
			}

		} catch (Exception e) {
			// TODO: handle exception
			e.printStackTrace();
		}
	}
}
```

### 체크 포인트

- `START_TAG` 비교에서 `==` 대신 `!=`를 잘못 쓰면 오류 발생
- `END_TAG`에서 `tagName = ""` 처리하는 이유:
    - 공백, 줄바꿈 같은 텍스트가 들어왔을 때 이전 태그명이 남아 오작동하는 것을 막기 위해

## 10. 공원 정보 XML 파싱

### 목표

서울 공원 정보 API에서 아래 항목 출력

- 공원명
- 공원 개요
- 개원일
- 공원 주소
- 전화번호

### 코드

```java
package com.beaver.apr011.main;

import java.io.IOException;

import java.io.InputStream;

import org.xmlpull.v1.XmlPullParser;

import org.xmlpull.v1.XmlPullParserFactory;

import com.jeesung.http.client.JeesungHttpClient;

public class ParkMain {

	public static void main(String[] args) {

		String address = "http://openAPI.seoul.go.kr:8088/4f626857416f6c6c3632586a416843/xml/SearchParkInfoService/1/133";

		// 공원명, 공원 개요, 개원일, 공원 주소, 전화번호 출력

		try {
			InputStream is = JeesungHttpClient.download(address);

			XmlPullParserFactory xppf = XmlPullParserFactory.newInstance();
			XmlPullParser xpp = xppf.newPullParser();
			xpp.setInput(is, "UTF-8");

			int type = xpp.getEventType();
			String tagName = null;

			while (type != XmlPullParser.END_DOCUMENT) {

				if (type == XmlPullParser.START_TAG) {
					tagName = xpp.getName();
				} else if (type == XmlPullParser.TEXT) {
					if (tagName.equals("PARK_NM")) {
						System.out.printf("공원 명: %s\n", xpp.getText());
```

## 11. 인터넷 주소 체계

### URL 구성 요소

- **프로토콜**
    - 통신 방식
    - 예: `http`, `https`
- **컴퓨터 주소**
    - 서버 주소
- **경로(Path)**
    - 폴더명 / 폴더명 / 파일명
- **요청 파라미터(Query String)**
    - `?` 뒤에 붙음
    - 형식:
        
        ```
        변수명=값&변수명=값&변수명=값...
        ```
        
    - 클라이언트가 서버에 공개적으로 전달할 정보

### 주의

- 인터넷 주소에는 한글, 특수문자가 그대로 들어가면 안 됨
- 반드시 **URL 인코딩** 필요

예:

- `ㅋ` → 인코딩된 형태로 변환됨

## 14. JeesungHttpClient 유틸 클래스

### 역할

- URL로 요청해서 데이터 다운로드
- 헤더가 필요한 요청도 처리
- InputStream을 String으로 변환

### 코드

```java
package com.jeesung.http.client;

import java.io.BufferedReader;
import java.io.IOException;
import java.io.InputStream;
import java.io.InputStreamReader;
import java.net.HttpURLConnection;
import java.net.URL;
import java.util.HashMap;
import java.util.Iterator;
import java.util.Set;

import javax.net.ssl.HttpsURLConnection;

public class JeesungHttpClient {

	// 서버에 요청해서 다운 받기
	public static InputStream download(String address) throws IOException {
		// java.net.URL은 URL의 형식이 유효한지 자동으로 검증
		URL u = new URL(address);
		if (address.startsWith("https")) {
			HttpsURLConnection huc = (HttpsURLConnection) u.openConnection();
			return huc.getInputStream();
		} else {
			HttpURLConnection huc = (HttpURLConnection) u.openConnection();
			return huc.getInputStream();
		}

	}

	// 서버에 요청해서 다운받기 (헤더 처리 O)
	public static InputStream download(String address, HashMap<String, String> headers) throws IOException {
		URL u = new URL(address);
		if (address.startsWith("https")) {
			HttpsURLConnection huc = (HttpsURLConnection) u.openConnection();
			Set<String> headerName = headers.keySet();
			for (String hn : headerName) {
				huc.addRequestProperty(hn, headers.get(hn));

			}
			return huc.getInputStream();
		} else {
			HttpURLConnection huc = (HttpURLConnection) u.openConnection();
			Set<String> headerName = headers.keySet();
			for (String hn : headerName) {
				huc.addRequestProperty(hn, headers.get(hn));

			}
			return huc.getInputStream();}
	}

	// 다운받은걸 String 으로 바꾸기
	public static String convert(InputStream is, String charset) throws IOException {
		InputStreamReader isr = new InputStreamReader(is, charset);
		BufferedReader br = new BufferedReader(isr);

		String line = null;
		StringBuffer sb = new StringBuffer();
		while ((line = br.readLine()) != null) {
			sb.append(line);
		}
		return sb.toString();
	}

}
```

## XML 파싱 흐름

1. API 주소 준비
2. `InputStream`으로 다운로드
3. `XmlPullParser` 생성
4. `setInput()`으로 XML 연결

6. `START_TAG`면 태그명 저장

# 암기용

### 1. 웹서버와 API

- 웹서버는 클라이언트 요청에 응답하는 시스템
- DB 데이터를 일반 사용자가 쓰기 쉽게 보여주기 위해 웹/서버 사용
- 개발자는 DB 원본보다 XML, JSON 형태의 응답 데이터를 많이 사용

### 2. XML

- 태그로 데이터를 감싼 구조형 데이터
- 시작태그 / 텍스트 / 종료태그로 구성

### 3. Parsing

- 필요한 데이터만 골라서 가공하는 작업
- XML 파싱 시 `XmlPullParser` 사용

### 4. XML 파싱 핵심 순서

- 주소 준비
- `download()`
- `XmlPullParser` 생성
- `setInput()`
- `getEventType()`
- 반복문으로 순회
- `START_TAG`면 태그명 저장
- `TEXT`면 태그명 비교 후 값 출력
- `END_TAG`면 태그명 초기화

### 5. 이벤트 종류

- `START_DOCUMENT`
- `END_DOCUMENT`
- `START_TAG`
- `END_TAG`
- `TEXT`

### 6. 주의사항

- `==` / `!=` 오타 주의
- `END_TAG`에서 `tagName` 초기화 필요
- 공백/줄바꿈이 텍스트로 인식될 수 있음

### 7. URL

- 인터넷 주소에는 한글/특수문자를 바로 넣으면 안 됨
- `URLEncoder.encode(..., "UTF-8")` 사용

### 8. 요청 헤더

- 클라이언트가 서버에 보내는 추가 정보
- API 인증 정보 전달할 때 사용

### 9. 파일 저장

- 파싱한 결과는 `BufferedWriter`로 txt 저장 가능

### 10. 실전 포인트

- 서울 열린데이터 광장: 미세먼지, 공원 정보
- 네이버 API: 뉴스 검색
- `channel`과 `item`에 같은 태그가 있을 수 있으므로 원하는 영역만 골라야 함

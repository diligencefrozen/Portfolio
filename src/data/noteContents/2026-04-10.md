# 1. DML 기본 정리

테이블에 데이터를 **검색, 삽입, 수정, 삭제**하는 데 사용하는 문장인

**데이터 조작어(DML)** 

---

## INSERT

### 기본 형태

```sql
insert into [테이블명] values(값, 값, ...);
```

### 핵심 내용

- `insert`를 하면서 `primary key`를 줄 수 없는 상황이 있음
- 이럴 때 **강제적으로 고유 번호값**을 만들어서 PK를 부여함
- 그 고유한 번호값을 만드는 데 사용한 것이 **SEQUENCE**

### SEQUENCE 정리

- 번호값들을 자동으로 찍어내는 공장 같은 개념
- 테이블과 직접적인 관계는 없음
- 단점:
    - `INSERT`에 실패하더라도 `SEQUENCE` 값은 무조건 증가함
    - 번호값을 고의적으로 낮춰서 사용할 수 없음

# 4. WHERE절과 조건식

전체 데이터 중 **조건에 맞는 데이터**를 찾아주는 절

---

## 4-1. 비교연산자

```sql
=, !=, <, >, >=, <=
```

---

## 4-2. 관계연산자

```sql
AND, BETWEEN, OR, IN, NOT
```

### 핵심 포인트

- `AND`와 `BETWEEN`의 차이 구분
- 연속되지 않은 값들을 받아올 때: `OR`, `IN`
- 반대되는 값을 받아올 때: `NOT`

### 주의

- `NOT`의 위치는 `IN` 앞

예:

```sql
where 컬럼명 not in ('값1', '값2');
```

---

## 4-3. 패턴연산자

문자열 포함 검색할 때 사용

```sql
LIKE '%패턴%'
```

### 왜 `=`를 쓰면 안 되는가?

- `=`는 패턴이 아니라 **문자 그대로** 인식해버리기 때문

---

## 4-4. 집합연산자

### UNION / UNION ALL

- 합집합
- `UNION`: 중복 제거
- `UNION ALL`: 중복 포함 전체 출력

### INTERSECT

- 교집합
- 공통되는 값 출력

### MINUS

- 차집합
- 앞의 `SELECT`에는 있고 뒤의 `SELECT`에는 없는 값 출력

---

## 4-5. 연산자 우선순위

왼쪽이 우선순위 높음

```
괄호 > 산술연산자 > 비교연산자 > NOT > AND > OR
```

# 6. 서브쿼리 (SubQuery)

## 정의

`SELECT`문 안에서 다시 `SELECT`문을 사용하는 기술

하나의 SQL 문 안에 다른 SQL 문이 중첩된 질의문을 말함.

다른 테이블에서 가져온 데이터로 현재 테이블의 정보를 찾거나 가공할 때 사용함.

---

## 예시: 평균가보다 가격이 낮은 과자 조회

### 오류 코드

```sql
select s_name, s_price from snack where s_price < round(avg(s_price),1); -- 오류 발생
```

### 정답 코드

```sql
select s_name, s_price from snack where s_price < (select avg(s_price) from snack); -- 정답코드
```

---

## 구성

- 주 질의(main query, 외부질의)
- 부속질의(subquery, 내부질의)

---

## 서브쿼리 주의사항

- **소괄호 필수**
- 소괄호 없으면 구문 오류 발생
- 독립된 명령문으로 해석될 수 있어 문법적으로 잘못된 구조가 됨
- `where` 절 이후 `select` 문에서는 **소괄호 넣고 시작**

### 중요

main query의 `where` 절에서 subquery 결과와 비교할 때는

**반드시 비교연산자 사용**

# 7. 과자(snack) 예제 문제

---

## 7-1. 과자들 중 최고가

### 내 코드

```sql
select s_price from snack where s_price >= (select max(s_price) from snack); -- 내 코드
```

### 정답 코드

```sql
select max(s_price) from snack; -- 정답 코드 ( 위랑 결과는 같음 )
```

### 보완

- 최고가 **가격값만** 필요하면 `max(s_price)`가 더 직접적
- 최고가인 **행 전체**를 찾고 싶으면 `where s_price = (...)` 형태가 مناسب

---

## 7-2. 제일 비싼 과자 이름, 제조사, 가격

### 내 코드

```sql
select s_name, s_compa, s_price from snack where s_price >= (select max(s_price) from snack); -- 내 코드
```

### 정답 코드

```sql
select s_name, s_compa, s_price from snack where s_price = (select max(s_price) from snack); -- 정답 코드 ( 위랑 결과는 같음 )
```

### 보완

- 논리적으로는 `=`가 더 정확함
- 같은 최고가가 여러 개면 여러 행이 나올 수 있음

---

## 7-3. 제일 싼 과자는 어디에서 만드나요?

### 내 코드

```sql
select s_name, s_compa, s_price from snack where s_price <= (select min(s_price) from snack); -- 내 코드
```

### 정답 코드

```sql
select distinct s_compa from snack where s_price = (select min(s_price) from snack); -- 정답 코드
```

### 보완

- 질문이 **회사(제조사)** 를 묻고 있으니 `s_compa`만 조회하는 게 더 정확
- 같은 최저가 과자가 여러 개일 수 있어서 `distinct`를 붙임

---

## 7-4. 평균가보다 비싼 과자는 몇 종류?

### 내 코드

```sql
select s_name, s_price from snack where s_price >= (select avg(s_price) from snack); -- 내 코드
```

### 정답 코드

```sql
select count(*) from snack where s_price > (select avg(s_price) from snack); -- 정답 코드
```

### 보완

- 문제에서 “몇 종류?”를 물었으므로 `count(*)`가 맞음
- “비싼”이므로 `>`가 더 적절

---

## 7-5. 유통기한이 가장 오래 남은 과자의 전체정보

### 내 코드

```sql
select * from snack where s_year >= (select max(s_year) from snack); -- 내 코드
```

### 정답 코드

```sql
select * from snack where s_year = (select max(s_year) from snack); -- 정답 코드 ( 위랑 결과는 같음 )
```

### 보완

- 최대값 비교는 `=`가 더 정확한 표현

# 8. 과자 회사 테이블 만들기

## 요구사항

회사 이름, 주소(도시 이름), 직원 수 값을 가질 수 있게 만들기

참조관계 고려 X

```sql
create table snack_compa(
	sna_name varchar2(10 char) primary key,
	sna_addr varchar2(10 char) not null,
	sna_empl number(3) not null

);
insert into snack_compa values('오리온', '서울', 100);
insert into snack_compa values('크라운', '부천', 70);
insert into snack_compa values('해태', '광주', 150);
insert into snack_compa values('롯데', '부산', 50);
select * from snack_compa;
```

---

## 8-1. 직원 수 제일 적은 회사에서 만드는 과자 이름, 가격

### 내 코드

```sql
select s_name, s_price from snack where s_price = (select min(sna_empl) from snack_compa); -- 내 코드
```

### 정답 코드

```sql
select s_name, s_price from snack where s_compa = (select sna_name from snack_compa where sna_empl = (select min(sna_empl) from snack_compa));
```

### 보완

- `s_price`와 `sna_empl`를 비교하면 의미가 안 맞음
- 먼저 **직원 수가 최소인 회사명**을 찾고
- 그 회사명이랑 `snack.s_compa`를 비교해야 맞음

---

## 8-2. 제일 비싼 과자를 만드는 회사는 어디에 있는지

```sql
select sna_addr from snack_compa where sna_name = (select s_compa from snack where s_price = (select max(s_price) from snack)); -- 정답 코드
```

---

## 8-3. 부천에 있는 회사에서 만드는 과자 평균가

```sql
select round(avg(s_price),1) from snack where s_compa = (select sna_name from snack_compa where sna_addr = '부천'); -- 정답 코드
```

---

## 8-4. 평균가 이상의 과자를 만드는 회사의 이름, 위치

### 내 코드

```sql
select s_compa from snack where s_compa = (select sna)addr from snack_compa where s_price >= (select avg(s_price) from snack)); -- 내 코드
```

### 구문 오류 코드

```sql
select sna_name, sna_addr from snack_compa where sna_name = (select s_compa from snack where s_price >= (select avg(s_price) from snack)); -- 구문 오류
```

### 설명

```sql
-- 회사명이 해태거나, 롯데거나, ...
-- = 를 쓰면 단일한 값 이상의 결과를 내보낼 수가 없어서
--       'single-row ~~' 발생
-- => 관계 연산자 < in > 사용
```

### 정답 코드

```sql
select sna_name, sna_addr from snack_compa where sna_name in (select s_compa from snack where s_price >= (select avg(s_price) from snack));
```

### 보완

- 서브쿼리 결과가 여러 행이면 `=` 사용 불가
- 이럴 때는 `IN` 사용

# 9. 음료 / 카페 테이블 실습

---

## 9-1. 테이블 만들기

### 조건

- 음료 이름, 카페 이름은 각 테이블에서 고유값
- 참조관계 고려 X

```sql
create table drink(
	d_name varchar2(10 char) primary key,
	d_price number(4) not null,
	d_cafename varchar2(10 char) not null

);
drop table cafe cascade constraint purge;
create table cafe(
	c_name varchar2(10 char) primary key,
	c_loca varchar2(10 char) not null,
	c_remainseats number(3) not null

);

select * from drink;
select * from cafe;
```

---

## 9-2. 데이터 삽입

### A카페

- 서울
- 좌석 100석

```sql
insert into cafe values('A카페', '서울', 100);
```

판매 음료:

- 아메리카노 2000원
- 라떼 3000원
- 녹차 2500원

```sql
insert into drink values('아메리카노', 2000, 'A카페');
insert into drink values('라떼', 3000, 'A카페');
insert into drink values('녹차', 2500, 'A카페');
```

---

### B카페

- 서울에 위치하고 좌석 80석, 이라고 적혀 있지만
    
    코드에는 `인천`으로 들어가 있음
    
    → **필기 내용과 코드가 불일치**하므로 시험/과제 제출 시 하나로 통일 필요
    

```sql
insert into cafe values('B카페', '인천', 80);
```

판매 음료:

- 홍차 2500원
- 스무디 3000원
- 에스프레소 4000원

```sql
insert into drink values('홍차', 2500, 'B카페');
insert into drink values('스무디', 3000, 'B카페');
insert into drink values('에스프레소', 4000, 'B카페');
```

---

## 9-3. 쿼리문 작성

### 전체 음료 기준 평균가보다 비싼 음료는 몇 종류?

```sql
select count(*) from drink where d_price > (select avg(d_price) from drink); -- 내 코드
```

---

### 제일 싼 음료를 파는 카페의 이름?

```sql
select c_name from cafe where c_name = (select d_cafename from drink where d_price = (select min(d_price) from drink)); -- 내 코드; 정답 코드 1;
select d_cafename from drink where d_price = (select min(d_price) from drink); -- 정답 코드 2
```

### 보완

- 둘 다 가능
- 단, 최저가 음료를 파는 카페가 여러 곳이면 `=`보다 `IN`이 더 안전할 수 있음

---

### 서울에 있는 카페에서 만드는 음료의 평균가 ?

```sql
select round(avg(d_price),1) from drink where d_cafename = (select c_name from cafe where c_loca = '서울'); -- 구문오류; 내 코드;
select round(avg(d_price),1) from drink where d_cafename in (select c_name from cafe where c_loca = '서울'); -- 내 코드;
select avg(d_price) from drink where d_cafename = (select c_name from cafe where c_loca = '서울'); -- 정답 코드 ( 다만, 위와 결과는 같음 )
```

### 보완

- 서울 카페가 여러 개일 수 있으므로 `IN`이 일반적으로 더 안전함
- `=`는 단일 행 서브쿼리일 때만 안정적

---

### 좌석이 90석 이상인 카페에서 파는 음료의 이름?

```sql
select d_name, d_cafename from drink where d_cafename = (select c_name from cafe where c_remainseats >= 90); -- 정답 코드
```

### 보완

- 좌석 90석 이상 카페가 여러 개면 `IN` 필요
- 현재 데이터 기준으로는 단일 결과면 동작 가능

---

### 전체 음료 기준 평균가 이하인 음료의 이름?

```sql
select d_name, d_cafename from drink where d_price < (select avg(d_price) from drink); -- 내 코드
select d_name, d_cafename from drink where d_price <= (select avg(d_price) from drink); -- 정답 코드
```

### 보완

- “이하”는 `<=`
- “미만”은 `<`

---

### 가장 비싼 음료를 파는 카페는 어디에 있는지?

```sql
select c_loca, c_name from cafe where c_name = (select d_cafename from drink where d_price = (select max(d_price) from drink)); -- 내 코드
```

# 13.  핵심

## 꼭 기억할 것

### 1) 서브쿼리 결과가 여러 행이면

- `=` 쓰면 안 됨
- `IN` 써야 함

### 2) 문제 문장을 정확히 해석

- “몇 종류?” → `count(*)`
- “이하” → `<=`
- “이상” → `>=`
- “초과” → `>`
- “미만” → `<`

### 3) 최대/최소 비교

- 보통 `>=`, `<=`보다 `= (select max(...))`, `= (select min(...))`가 더 정확함

### 4) WHERE와 HAVING 차이

- `WHERE`: 그룹화 전 필터
- `HAVING`: 그룹화 후 필터

### 5) LIKE와 =

- `LIKE '%문자%'` → 패턴 검색
- `=` → 완전히 같은 값 비교

## 1. 자바(Java)란

- 자바는 프로그램을 만들 때 사용되는 언어, 즉 프로그래밍 언어이다.
- 언어는 고급 언어와 저급 언어로 나뉜다.
- 고급 언어: 컴퓨터가 처리하기 쉽게 만든 것이 아니라, 사람이 이해하고 작성하기 쉽게 만든 언어. 예: 파이썬, 자바스크립트, 자바, Go, C
- 저급 언어: 컴퓨터와 가까운 언어. 개발자가 더 세세하게 다뤄야 한다.
- 원문: 자바는 비교적 저급 언어에 포함된다 / 파이썬과 비교해서 자바는 상대적으로 저급 언어에 포함된다 / 저급 언어일수록 사람이 처리해야 하는 부분이 많다(예: 자료형) / 파이썬과 다르게 자바는 개발자가 명시해 주어야 하는 부분이 꽤 있다.
- 자바는 **저급 언어가 아니라 고급 언어**이다. 다만 파이썬보다 개발자가 직접 명시할 것이 많아서 상대적으로 더 엄격하고 저수준처럼 느껴질 수 있다.

## 2. 프로그래밍이란

- 프로그래밍은 컴퓨터가 일을 할 수 있도록 프로그래밍 언어로 이루어진 명령어들의 집합(프로그램)을 만드는 것을 의미한다.
- 프로그램 = 명령어들의 집합
- 소스 코드 = 프로그램을 글자로 작성한 것

## 3. 자바가 탄생한 계기

- 과거에는 컴퓨터가 100대가 존재하면, 이 100대 각각에 맞춰 프로그래밍을 고쳐주어야 인식이 가능했었다.
- 그래서 제임스 고슬링이 썬 마이크로시스템즈라는 회사에서 오류 발생이 현저하게 적은 자바를 개발하였다.
- 자바의 핵심 목적은 특정 기계에 종속되지 않고 여러 환경에서 실행되도록 만드는 것이었다.

## 4. 자바가 쓰이는 환경

- 자바가 쓰여지는 환경은 웹 서버 구현이 대표적이다.
- 한국에서는 Spring Framework가 자바 기반으로 많이 알려져 있다.

## 5. 자바의 장점: WORA

- WORA = Write Once, Run Anywhere
- 어떤 PC에서 한번 프로그램을 작성하면, 다른 개발 환경에서도 프로그램이 정상적으로 실행 가능하다는 뜻이다.
- 따라서 자바는 호환성 문제 해결이 가능한 대표적인 프로그래밍 언어이다.

## 6. 자바 실행 구조

- 프로그램은 명령어들의 집합이며, 소스 코드라고 부른다.
- 이 소스 코드가 자바 컴파일러를 통해 자바 가상 머신으로 전달된다.
- JVM이 있으면 어느 운영체제에서도 동일한 동작이 가능해진다.
- 소스 코드(.java) → javac 컴파일 → 바이트코드(.class) 생성 → JVM이 실행

## 7. 컴파일러

- 컴파일러: 사람이 이해하기 쉬운 언어인 고급 프로그래밍 언어로 작성된 소스 코드 전체를 컴퓨터가 이해할 수 있는 형태로 번역해 실행 파일 형태로 바꿔주는 프로그램이다.
- 자바는 보통 바로 기계어 실행 파일이 되는 것이 아니라 **바이트코드(.class)** 로 변환되고, 이 바이트코드를 JVM이 실행한다.

## 8. JVM / JRE / JDK

- 원문: JVM, JRE, JDK를 줄여서 자바 가상 머신(Java Virtual Machine) = JVM이라고 부른다.
- 셋은 같은 말이 아니다.
- JVM(Java Virtual Machine): 자바 가상 머신, .class 파일을 실행하는 주체
- JRE(Java Runtime Environment): 자바 실행 환경, 자바 프로그램을 실행할 수 있게 해주는 환경
- JDK(Java Development Kit): 자바 개발 도구, 개발용 PC에서 자바 개발 환경을 구축하기 위해 설치해야 하는 것
- JDK는 JRE + 실행 키트를 합쳐놓은 개념이라고 이해하면 된다.
- 정확히는 JDK = JRE + 개발 도구(javac 등)

## 11. JRE 설명

- JRE(Java Runtime Environment): 자바로 만든 프로그램을 실행할 수 있게 하는 것
- 종류는 두 가지가 있다: Oracle JDK, OpenJDK
- 원문: 썬 마이크로시스템즈에서 만든 제품이 Java이고, Oracle이 썬을 인수하여 지금은 Java는 Oracle의 소유가 되었다.
- Oracle JDK = OpenJDK + 추가 기능(원문: 유료)
- OpenJDK = 무료
- 많은 회사들은 무료 버전인 OpenJDK를 사용한다.

## 15. 자바 파일 실행 방법

- CMD 창에서 `javac (파일이름).java` 입력: 클래스 파일 생성 과정
- 예: `javac Hi.java`
- 컴파일 후 이상한 문자/정보가 보일 수 있는데, 원문에서는 이를 “기계어 상태로 컴파일 된 상태”라고 표현
- 정확히는 **바이트코드(.class)** 상태이다.
- CMD 창에서 `java (파일이름)` 입력하면 실행된다.
- 예: `java Hi`
- [주의] `.class` 확장자는 실행 명령에 붙이지 않는다.

## 16. javac.exe / java.exe

- javac.exe: JDK에 포함된 자바 컴파일러
- 자바로 작성된 텍스트 파일을 ByteCode(바이트코드, 실행을 위한 이진 표현 형태)로 변환하는 역할, 이것을 Compile이라고 한다.
- javac를 실행하면 `.class` 파일 생성
- java.exe: 컴파일된 `.class` 파일을 실행하는 프로그램
- 결론:
    - `javac.exe` = 번역하여 바이트코드로 변환
    - `java.exe` = 바이트코드를 실행하여 결과 출력

## 25. Print의 형태

- `println` : Console창에 데이터 출력 + 줄 바꿈(제일 많이 사용), 괄호 안에 값을 넣지 않으면 줄 바꿈 역할 수행
- `print` : Console창에 데이터 출력, 줄 바꿈 없음
- `printf` : Console창에 형식을 지정해서 데이터 출력(예: 소수점 자릿수 지정)
- 글자인 경우 큰따옴표 `"` 로 감싸준다.

## 26. 자동완성

## 32. Escape Sequence(이스케이프 문자)

- 특수한 문자나 기능을 표현할 때 역슬래시 `\` 사용
- `\t` : Tab키(줄 맞추기 용도)
- `\n` : New Line(줄 맞추기 용도)
- `\r` : Carriage Return(커서를 맨 앞으로 이동)
- `\r\n` : Enter키
- `\b` : Backspace키(1byte만 지움)
- 설명: C 언어에서 한 글자는 1Byte, 자바에서 한 글자는 2byte를 차지하는데 자바에서 `\b` 는 약간의 오류가 있어 실질적으로 사용하지 않는다.
- `\0` : space키
- `\0` 은 일반 공백(space)이라기보다 **널 문자(null character)** 로 보는 것이 더 정확하다.
- `\\` : `\` 를 문자로 출력
- `\"` : `"` 를 글자로 출력
- 경로 설정 시 `\` 사용

## 33. Escape 예제

```java
System.out.println("전자제품\r스마트폰:\t애플의\0아이폰\n\n안드로이드\0파벌\r\n스마트폰:\t삼성의\0갤럭시\n스마트폰:\t샤오미와\0화웨이");
```

- 현재 콘솔창은 가볍게 확인 용도라서 `\r` 이나 `\n` 둘 중 하나만 있어도 엔터 처리처럼 보일 수 있다.
- 파일 처리나 통신의 경우에는 `\n` 이나 `\r` 하나만 쓰면 의도치 않게 결과가 뒤집혀 출력될 수도 있다.
- 따라서 상황에 따라 `\r\n` 을 함께 고려해야 한다.
- `println` 은 `\n`, 즉 줄바꿈 기능까지 포함된 출력 함수라고 해석할 수 있다.

## 35. printf(Formatting - 형식 지정)

- 형태: `System.out.printf("%?", 값);`
- `%?` 는 자료형 또는 출력 형식 지정자

### 35-1. 정수(decimal)

- `%d` : 정수 데이터 자리
- `%xd` : 빈자리가 띄어쓰기로 채워져서 출력(x = 공백 자리 수)
- `%0xd` : 빈자리를 0으로 채워 x자리 정수로 만들어 줌

예:

```java
System.out.printf("%d\n", 1);
System.out.printf("%3d\n", 1);
System.out.printf("%010d\n", 1);
System.out.printf("---\n");
```

### 35-2. 실수(float)

- `%f` : 실수 데이터 자리
- `%.xf` : 소수점 이하 x자리, 잘리는 부분은 반올림, 자릿수는 많은데 표현할 숫자가 적다면 빈자리는 0으로 채움

예:

```java
System.out.printf("%f\n", 123.456789);
System.out.printf("%3f\n", 123.456789);
System.out.printf("%10f\n", 123.456789);
System.out.printf("---\n");
```

### 35-3. 문자열(String)

- `%s` : 글자 데이터 자리
- 숫자: 그냥 입력
- 문자열: 그냥 입력하면 자바 문법(변수)으로 해석 → 오류 발생
- 문자열은 `"내용"` 처럼 큰따옴표 안에 넣어야 한다.

예:

```java
System.out.printf("%s\n", "오늘 하루가 끝나가네요");
```

## 36. printf로 날짜/날씨 출력 예시

```java
System.out.printf("%s\0", "날짜:");
System.out.printf("%d\0", 2026);
System.out.printf("%s\0", "년");
System.out.printf("%d\0", 3, "월");   //  예시
System.out.printf("%s\0", "월");
System.out.printf("%d\0", 4, "일");   //  예시
System.out.printf("%s\n", "일");

System.out.printf("%s\0", "날씨:");
System.out.printf("%s\n", "맑음");

System.out.printf("%s\0", "기온");
System.out.printf("%d\0", 12);
System.out.printf("%s\n", "도");

System.out.printf("%s\0", "습도");
System.out.printf("%s\0", "35.35%");
```

- [정정] `System.out.printf("%d\0", 3, "월");` 와 `System.out.printf("%d\0", 4, "일");` 는 형식 지정자와 전달 값 개수가 맞지 않아 부정확한 예시이다.

## 37. printf 더 합리적인 방식

```java
System.out.printf("%d년\0", 2026);
System.out.printf("%02d일\0", 4);
System.out.printf("날짜 : %4d-%02d-%02d\n", 2026, 3, 4);
System.out.printf("날짜 : %4d-%02d-%02d\n", 2026, 3, 4);
```

- 이렇게 작성하는 것이 좀 더 합리적이다.
- `%4d` : 4칸 확보(2026)
- `%02d` : 한 자리 숫자 앞에 0 붙여 가독성 향상(01, 02, 03, 04)
- 두 자리 숫자는 그대로 출력(11, 12, 13)

## 38. %%의 의미

- `%%` : `%` 를 글자로 표현하고 싶을 때 사용(printf에서만 사용)
    
    예:
    

```java
System.out.printf("습도 : %.2f%%\n", 35.35);
System.out.printf("습도 : %.2f%%\n", 35.3559595959599595959595);
```

- 소수점 둘째 자리까지만 출력
- `%` 를 글자로 출력하기 위해 `%%` 사용
- `f` 는 실수 출력 형식

# 암기 포인트

## 1. 자바 기본

- 자바는 **고급 언어**이다.
- 파이썬보다 엄격하지만 저급 언어는 아니다.
- 자바는 웹 서버 구현, 백엔드 개발에서 많이 쓰인다.
- 한국에서는 Spring Framework가 자바 기반으로 많이 알려져 있다.

## 2. 자바 탄생 이유

- 과거에는 컴퓨터마다 프로그램을 따로 수정해야 했다.
- 이를 해결하기 위해 자바가 등장했다.
- 핵심 키워드: **호환성**

## 3. WORA

- Write Once, Run Anywhere
- 한 번 작성하면 여러 환경에서 실행 가능

## 4. 실행 구조

- `.java` → `javac` → `.class` → `JVM 실행`
- `javac` = 컴파일
- `java` = 실행

## 5. JVM / JRE / JDK

- JVM: 자바 가상 머신, 실행 담당
- JRE: 자바 실행 환경
- JDK: 자바 개발 도구
- 관계: **JDK > JRE > JVM**

## 6. OpenJDK / Oracle JDK

- OpenJDK: 무료
- Oracle JDK: 추가 기능 포함, 라이선스 주의
- 회사에서는 OpenJDK 많이 사용

## 7. LTS

- Long Term Support
- 장기 지원 버전

## 8. IDE

- IDE = 통합 개발 환경
- Eclipse는 무료라 교육용으로 많이 사용
- IntelliJ, VS Code도 존재

## 9. 주석

- `//` : 한 줄 주석
- `/* */` : 여러 줄 주석
- `/** */` : 문서 주석
- `Ctrl + /` : 주석 처리

## 10. Eclipse 단축키

- `Ctrl + S` 저장
- `Ctrl + M` 화면 최대화/복귀
- `Ctrl + Alt + 방향키` 줄 복사
- `Alt + 방향키` 줄 이동
- `Ctrl + D` 줄 삭제
- `Ctrl + Space` 자동완성
- `Ctrl + Shift + F` 코드 정렬
- `Ctrl + F11` 실행

## 11. 출력 함수

- `println` : 출력 + 줄바꿈
- `print` : 출력만
- `printf` : 형식 지정 출력

## 12. 문자열과 계산

- 큰따옴표 없으면 계산 가능
- 큰따옴표 있으면 문자열 그대로 출력

## 13. Escape 문자

- `\t` 탭
- `\n` 줄바꿈
- `\r` 커서 맨 앞으로
- `\r\n` 엔터
- `\\` 역슬래시 출력
- `\"` 큰따옴표 출력
- `\0` 은 공백이라기보다 널 문자로 이해하는 것이 더 정확

## 14. printf

- `%d` 정수
- `%f` 실수
- `%s` 문자열
- `%3d` : 3칸 확보
- `%010d` : 10칸 확보, 앞을 0으로 채움
- `%.2f` : 소수 둘째 자리까지
- `%%` : `%` 문자 출력

## 15. 자주 틀리는 포인트

- 자바는 저급 언어가 아니다.
- JVM = JRE = JDK 가 아니다.
- 실행 시 `java Hi` 처럼 `.class` 를 쓰지 않는다.
- `printf` 는 자동 줄바꿈이 아니다. 필요하면 `\n` 붙인다.

# 1. 멤버 변수(Member Variable) / 필드(Field)

## 개념

- **객체의 데이터가 저장되는 공간**
- 멤버변수는 변수처럼 생겼지만, 정확히는 **객체의 속성(필드)** 이다.

## 변수와 필드의 차이

### 변수

- 생성자, 메소드 내부에서만 사용
- 생성자나 메소드 실행이 끝나면 **자동으로 사라짐**
- 즉, **일시적으로만 존재**

### 필드

- 생성자, 메소드 전체에서 사용 가능
- **객체가 소멸되지 않는 한 객체와 함께 공존**
- 즉, **객체의 상태를 유지하는 데이터**

예시:

```java
int studentId;
String studentName;
String studentAddress;
```

## 초기값

- 초기값이 지정되지 않은 멤버 변수는
    
    **객체 생성 시 자동으로 기본 초기값이 들어간다.**

# 2. 메소드(method)

## 개념

- **객체의 동작에 해당하는 실행 블록**
- 중괄호 `{ }` 로 이루어진 코드 영역
- 이름을 붙여놓고 필요할 때 호출해서 사용
- 보통 **동사스럽게 작명**하는 것이 좋다

## 특징

- 메소드를 호출하면 그 안의 코드를 **일괄 실행**
- 객체 간 **데이터 전달 수단**이 되기도 한다

# 3. 라이브러리용 클래스와 실행용 클래스

## 라이브러리용 Class

- 다른 클래스에서 이용할 목적의 클래스
- 프로그램에는 여러 클래스가 있지만,
    
    **대부분은 라이브러리용 클래스**
    
- 즉, 실행을 직접 담당하지 않고 **기능/데이터를 제공**

## 실행용(Main) Class

- 프로그램 실행을 위한 `main()` 메소드를 제공
- 실제 시작점 역할을 함

# 5. 자료형의 기본값 예제

## WhatIsValue 클래스

```java
public class WhatIsValue {

    // 기본형 - 정수
    byte byteField;
    short shortField;
    long longField;

    // 기본형 - 실수
    float floatField;
    double doubleField;

    // 기본형 - 논리, 문자
    boolean booleanField;
    char charField;

    // 참조형 - 문자열, 배열
    String stringField;
    int[] arrayField;

    public void printInfo() {
        System.out.println("byte : " + byteField);
        System.out.println("short : " + shortField);
        System.out.println("long : " + longField);
        System.out.println("float : " + floatField);
        System.out.println("double : " + doubleField);
        System.out.println("boolean : " + booleanField);
        System.out.println("char : " + charField);
        System.out.println("String : " + stringField);
        System.out.println("Array : " + arrayField);
    }

```

## 실행 클래스

```java
public class OMain2 {

    public static void main(String[] args) {
        System.out.println("### 각 자료형들의 기본값 ###");

        WhatIsValue wiv = new WhatIsValue();
        wiv.printInfo();
    }
}
```

## 기본값 정리

- `byte`, `short`, `int`, `long` → `0`
- `float`, `double` → `0.0`
- `boolean` → `false`
- `char` → 공백처럼 보이는 값 (`\u0000`)
- `String`, 배열 등 참조형 → `null`

## char 설명

- `char`의 기본값은 눈에 잘 안 보이는 문자
- 유니코드상 **`\u0000`**

# 6. Stack / Heap / Garbage Collection

## Stack

- 변수를 만들 때마다 **아래에서부터 차곡차곡 쌓임**
- 지역 변수, 매개변수 등이 주로 저장됨
- 프로그램이 끝나면 자동 정리

## Heap

- 객체가 생성되는 공간
- 컴퓨터가 적당하다고 판단한 위치에 생성
- 예전에는 자동 정리 기능이 없어서 개발자가 직접 관리해야 했음

## Garbage Collection

- Heap 영역의 **자동 메모리 정리 시스템**
- Java는 JVM의 **Garbage Collector**가 메모리 관리를 대행
- 개발자는 메모리 해제에 덜 신경 써도 되어 개발에 집중 가능

## 언제 발생?

- 그 객체를 **더 이상 참조할 방법이 없어졌을 때**
- 즉, **어느 변수도 그 객체를 가리키지 않을 때**

## 주의

```java
System.gc();
```

- 강제로 호출할 수는 있지만
- 시스템 성능에 영향을 줄 수 있으므로 **실무/학습에서 남발 금지**

# 7. Mask 예제와 참조 개념

## Mask 클래스

```java
public class Mask {

    String mask_info;
    String mask_name;
    String mask_market;
    int mask_price;

    public void showMaskInfo() {
        System.out.println("마스크: " + this.mask_info);
        System.out.println("마스크 명: " + mask_name);
        System.out.println("마스크 구매처: " + mask_market);
        System.out.println("마스크 가격: " + mask_price + "원");
    }
}
```

## 실행 예제

```java
public class OMain3 {

    public static void main(String[] args) {

        Mask mask = new Mask();
        mask.mask_info = "의료용 마스크";
        mask.mask_name = "test mask name";
        mask.mask_market = "test market";
        mask.mask_price = 1000;

        mask.showMaskInfo();

        System.out.println("######");

        Mask mask2 = mask; // 같은 객체를 가리키는 별명 하나 추가
        mask = null;

        System.out.println(mask);   // null
        System.out.println(mask2);  // 아직 객체 참조 가능

        System.out.println("#####");

        mask2 = null; // 이제 아무도 객체를 참조하지 않음
        // Garbage Collection 대상

        System.out.println(mask2);

        // mask.showMaskInfo();  // 오류 발생 (mask는 null이므로)
    }
}
```

## 핵심

- `Mask mask2 = mask;`
    - 객체가 복사된 게 아니라 **참조값(주소)이 복사**
- `mask = null;`
    - `mask`만 참조를 끊은 것
- `mask2 = null;`
    - 이제 그 객체를 가리키는 참조가 하나도 없으므로 GC 대상

# 8. OOD(Object Oriented Design)

## 구분

- **Coder** : 코드를 작성하는 사람
- **Programmer** : 코드 + 설계까지 하는 사람

## OOD

- **Object Oriented Design**
- 객체지향 설계
- 실생활의 개념을 프로그램 속 객체로 설계하는 방식

예:

- 의사
- 손님
- 병원
- 마스크
- 학생

이런 걸 각각 별도 객체로 나누어 표현한다.

# 9. 지역 변수 / 파라미터 / 리턴값 / 멤버변수

## 지역 변수(Local Variable)

- 메소드 안에서 만든 변수
- **그 행동을 하는 동안만 필요한 임시 변수**

## 메소드 파라미터(Parameter)

- 그 행동을 하는 데 필요한 **재료**

## 리턴값(Return Value)

- 메소드 실행 후 돌려주는 **결과물**

## 멤버변수(Member Variable)

- 그 객체가 계속 가지고 있어야 할 **속성**

이 파트는 객체지향 개념이 많이 섞여 있어서 중요하다.

## Hospital 클래스

```java
public class Hospital {

    String doctor_name;
    int doctor_age;

    public double askHeight(Guest guest) {
        System.out.println("키(cm) :");
        double h = guest.answerHeight();

        if (h >= 1 && h <= 300) {
            System.out.println("입력 확인 완료");
        } else {
            System.out.println("조건에 해당하지 않음");
        }

        return (h >= 1 && h <= 300) ? h / 100 : askHeight(guest);
    }

    public double askWeight(Guest guest) {
        System.out.println("몸무게(kg) : ");
        double w = guest.answerWeight();

        if (w >= 1) {
            System.out.println("입력 확인 완료");
        } else {
            System.out.println("위험함");
        }

        return (w >= 1) ? w : askWeight(guest);
    }

    public double calcBMI(Guest guest) {
        return guest.w / (guest.h * guest.h);
    }

    public String judgeBMI(Guest guest) {
        if (guest.BMI >= 40) {
            return "고도비만";
        } else if (guest.BMI >= 35) {
            return "중증도비만";
        } else if (guest.BMI >= 30) {
            return "경도비만";
        } else if (guest.BMI >= 25) {
            return "과체중";
        } else if (guest.BMI >= 18.5) {
            return "정상";
        } else {
            return "저체중";
        }
    }

    public void tellResult(Guest g) {
        System.out.printf("%s님의 키는 %.1fcm이고, 몸무게는 %.1fkg입니다.\n", g.guest_name, g.h * 100, g.w);
        System.out.printf("BMI: %.1f\n", g.BMI);
        System.out.printf("당신은 %s입니다.\n", g.bmiResult);
    }

    public void showDoctorInfo() {
        System.out.println("<Doctor>");
        System.out.println(this.doctor_name);
        System.out.println(doctor_age);
        System.out.println("######");
    }

    public void start(Guest g) {
        g.h = askHeight(g);
        g.w = askWeight(g);
        g.BMI = calcBMI(g);
        g.bmiResult = judgeBMI(g);
        tellResult(g);
        g.exit();
    }
}
```

## 이 파트의 핵심 설계 포인트

> hospital 객체에 의사와 손님 정보 전부 넣어주려 했는데, 손님 객체 정보가 날아감.
> 

이 말의 핵심은,

**한 객체에 너무 많은 역할과 정보를 몰아넣지 말자**는 것이다.

즉:

- 의사 정보는 `Hospital`
- 손님 정보는 `Guest`

이렇게 **역할에 맞게 분리**하는 게 객체지향 설계다.

## 원본 코드에서 보완해야 할 부분

### 1) 잘못된 리턴 타입

원본:

```java
public void calcBMI(Guest g) {
    return g.w / (g.h * g.h);
}
```

이건 `void`인데 `return 값`을 하고 있으니 오류.

수정:

```java
public double calcBMI(Guest g) {
    return g.w / (g.h * g.h);
}
```

### 2) 메소드 이름 오타

원본:

```java
g.bmiResult = judgeBMU(g);
```

수정:

```java
g.bmiResult = judgeBMI(g);
```

### 3) 존재하지 않는 메소드 호출

원본:

```java
g.enter();
callGuest(g);
```

그런데 `Guest` 클래스에 `enter()`가 없고, `callGuest()`도 정리 중 제거됨.

그래서 최종본에서는 빼는 것이 맞다.

### 4) 출력문 오류

원본:

```java
System.out.println("당신은 %s니까, 이렇다.$f", );
```

이건 `println`으로 포맷문자를 쓸 수 없고 문법도 깨져 있다.

수정:

```java
System.out.printf("당신은 %s입니다.\n", g.bmiResult);
```

### 5) 키 단위 혼동

원본에서 키는 `cm`로 입력받고 BMI는 `m` 단위로 계산해야 하므로,

`askHeight()`에서 `h / 100`으로 바꾸는 흐름은 맞다.

이건 아주 중요한 포인트다.

## Computer 클래스

```java
import java.util.Random;

public class Computer {

    Random cpu = new Random();

    int coin = 20;
    String useranswer;
    String answer;
    String result;
    String useranswer2;

    public int askHowManyCoin(OMain1 u) {
        System.out.println("몇 개로 할래? (2~20)");

        return cpu.nextInt(userCoin) + 1;
    }

    public String askHJ(OMain1 u) {
        System.out.print("홀? 짝? : ");

    public String judgeCoin(int shakeCoin) {
        return (shakeCoin % 2 == 0) ? "짝" : "홀";
    }

    public String getResult(String userAns, String comAns) {
        return userAns.equals(comAns) ? "정답" : "오답";
    }

    public void printResult(int shakeCoin, String comAns, String result) {
        System.out.printf("동전은 %d개 뽑았고, %s이야!\n", shakeCoin, comAns);
        System.out.printf("결과: %s\n", result);
    }

    public void start(OMain1 u) {
        int userCoin = askHowManyCoin(u);
        int shakeCoin = shakeCoin(userCoin);
        String userAns = askHJ(u);
        String comAns = judgeCoin(shakeCoin);
        String result = getResult(userAns, comAns);

        printResult(shakeCoin, comAns, result);
    }
}의도 정리
```

---

게임 흐름:

1. 컴퓨터가 사용자에게 **몇 개의 동전으로 할지** 묻는다
2. 사용자가 수를 입력한다
3. 사용자가 **홀/짝** 을 예측한다
4. 컴퓨터가 랜덤하게 동전 수를 정한다
5. 실제 결과가 홀인지 짝인지 판정한다
6. 사용자의 답과 비교해서 정답/오답 출력

---

## 이 파트의 진짜 핵심

> u.userans 라는 멤버변수를 만들지 않고 지역변수를 사용한 이유
> 
> 
> 이 답은 단순히 1회성이라서 굳이 객체의 속성으로 저장할 필요가 없다고 판단함.
> 

### 즉

- **계속 유지해야 하는 데이터** → 멤버변수
- **잠깐 쓰고 끝나는 값** → 지역변수

예:

```java
String userAns = u.answerHJ();
```

이 값은 한 번 판정하고 끝나므로 **지역변수**가 더 적절하다.

- 손님의 이름, 나이, 키, 몸무게 → 필드
- 손님이 키를 대답한다 → 메소드
- 병원이 BMI를 계산한다 → 메소드

## 3) 한 객체에 너무 많은 역할을 넣지 않는다

- 손님 정보는 Guest
- 의사/병원 동작은 Hospital
- 게임 진행은 Computer

## 4) 메소드 분리의 장점

- 읽기 쉬움
- 수정 쉬움
- 재사용 가능
- 역할이 분명해짐

# 13. 정리

## 필드

- 객체의 데이터를 저장하는 공간
- 객체가 살아있는 동안 유지됨
- 초기값 없으면 기본값 자동 대입

## 지역 변수

- 메소드 안에서만 사용
- 메소드 끝나면 사라짐

## 메소드

- 객체의 동작
- 호출하면 내부 코드가 실행됨

## 클래스 종류

- 실행용 클래스: `main()` 포함
- 라이브러리용 클래스: 기능 제공

## 메모리

- Stack: 지역변수, 매개변수
- Heap: 객체 저장
- GC: 참조 끊긴 객체 자동 정리

## 객체지향

- 현실의 대상별로 클래스를 나눠 설계
- 속성과 행동을 분리
- 역할별로 객체를 나누는 것이 중요

# 14. 암기용 파트

## 암기 1

**멤버변수(필드)**

= 객체의 속성 저장 공간

= 객체가 소멸되기 전까지 유지

## 암기 2

**지역변수**

= 메소드 안에서 만드는 임시 변수

= 메소드 종료 시 사라짐

## 암기 3

**메소드**

= 객체의 동작

= 호출하면 그 안의 코드 실행

## 암기 4

**초기값 없는 필드**

= 객체 생성 시 기본값 자동 저장

## 암기 5

**Stack**

= 지역변수, 매개변수 저장

= 메소드 종료/프로그램 종료 시 정리

## 암기 6

**Heap**

= 객체가 생성되는 공간

## 암기 7

**Garbage Collection**

= Heap에서 더 이상 참조되지 않는 객체를 자동 정리하는 시스템

## 암기 8

**객체지향 설계 핵심**

= 한 객체에 모든 정보를 몰아넣지 말고

**역할별로 클래스를 분리한다**

## 암기 9

**필드 vs 지역변수 판단 기준**

- 계속 기억해야 하면 → 필드
- 잠깐 쓰고 끝이면 → 지역변수

# 15. 핵심 요약

1. **필드는 객체의 속성이다**
2. **지역변수는 잠깐 쓰는 값이다**
3. **메소드는 객체의 행동이다**
4. **객체지향은 역할별로 클래스를 나누는 것이다**
5. **참조가 끊긴 객체는 Garbage Collection 대상이다**
6. **계속 필요한 값만 멤버변수로 두고, 일회성 값은 지역변수로 둔다**

# 1. AJAX / JSON

## 1-1. AJAX란?

### 정의

- JavaScript를 사용한 **비동기 통신**
- 클라이언트와 서버 간에 데이터를 주고받는 기술
- 예전에는 XML 데이터를 많이 사용했음

### 핵심 포인트

- 원래는 **JavaScript에서 XML 파싱**이 중요했음
- 그런데 XML은 점점 덜 쓰이게 됨
- 이유: JavaScript에 더 잘 맞는 데이터 형식이 등장했기 때문

### 그 대안

- **JSON (JavaScript Object Notation)**

---

## 1-2. JSON이란?

### 정의

- 데이터를 **자바스크립트 객체 형태로 표현**한 포맷
- XML보다 훨씬 간결하고 읽기 쉬움
- 실무에서는 XML보다 JSON이 훨씬 더 많이 쓰임

### 비교

### Java 배열

```java
{ 1, 2, 3 }
```

- 자바에서는 배열처럼 보일 수 있지만

### JavaScript 배열

```jsx
[1, 2, 3]
```

- 자바스크립트에서 배열은 `[]`

### Java 객체

```java
Dog d = new Dog();
d.setName("가나다");
d.setAge(3);
```

### JavaScript 객체

```jsx
var dog = { name: "가나다", age: 3 };
```

### 정리

- JavaScript에서 `{}` 는 주로 **객체**
- JavaScript에서 `[]` 는 **배열**

---

## 1-3. JSON 관련 도구

### 추천

- 크롬 스토어에서 **json viewer** 검색

### 라이브러리

- `json-simple-1.1.1.jar` 사용

# 2. JSON 파싱 예제 1 — 서울 지하철 데이터

## 2-1. 예제 목적

서울 열린데이터 API에서 JSON을 받아와서,

- 역 이름
- 노선명
- 기점 정보
    
    를 출력하는 예제
    

---

## 2-2. 핵심 흐름

1. API 주소 준비
2. `InputStream`으로 데이터 다운로드
3. 문자열로 변환
4. `JSONParser`로 파싱
5. 최상위 `JSONObject`에서 필요한 값 추출
6. `JSONArray row`를 반복문으로 순회
7. 각 역 정보 출력

---

## 2-3. 전체 코드

```java
package com.beaver.apr021.main;

import java.io.InputStream;

import org.json.simple.JSONArray;
import org.json.simple.JSONObject;
import org.json.simple.parser.JSONParser;

import com.jeesung.http.client.JeesungHttpClient;

public class SubwayMain {

    // http://openapi.seoul.go.kr:8088/4f626857416f6c6c3632586a416843/json/TbSeoulmetroStOrigin/1/274/

    public static void main(String[] args) {
        try {
            String address = "http://openapi.seoul.go.kr:8088/4f626857416f6c6c3632586a416843/json/TbSeoulmetroStOrigin/1/274/";

            InputStream is = JeesungHttpClient.download(address);
            String result = JeesungHttpClient.convert(is, "UTF-8");

            // System.out.printf(result);

            JSONParser jp = new JSONParser();

            JSONObject jo = (JSONObject) jp.parse(result);

            JSONObject metro = (JSONObject) jo.get("TbSeoulmetroStOrigin");

            JSONArray row = (JSONArray) metro.get("row");

            JSONObject data = null;

            for (int i = 0; i < row.size(); i++) {
                data = (JSONObject) row.get(i);

                System.out.println(data.get("STTN"));
                System.out.println(data.get("SBWY_ROUT_LN"));
                System.out.println(data.get("STTN_ORGN"));
                System.out.println("-------------------------");
            }

        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

---

## 2-4. 코드 해설

### `JSONParser`

```java
JSONParser jp = new JSONParser();
```

- JSON 문자열을 자바 객체 형태로 바꾸기 위한 파서

### 최상위 JSON 객체

```java
JSONObject jo = (JSONObject) jp.parse(result);
```

### `TbSeoulmetroStOrigin` 꺼내기

```java
JSONObject metro = (JSONObject) jo.get("TbSeoulmetroStOrigin");
```

### `row` 배열 꺼내기

```java
JSONArray row = (JSONArray) metro.get("row");
```

### 반복문으로 각 데이터 접근

```java
data = (JSONObject) row.get(i);
```

---

## 2-5. 기억 포인트

- JSON 객체 → `JSONObject`
- JSON 배열 → `JSONArray`
- `get("키이름")`으로 값 추출
- 배열은 `size()`만큼 반복

# 3. JSON 파싱 예제 2 — OpenWeatherMap 날씨 검색

## 3-1. 예제 목적

도시명을 입력받아 날씨 정보를 출력

### 출력 대상

- 나라 이름
- 도시 이름
- 날씨 설명
- 기온
- 체감온도
- 최저기온
- 최고기온
- 기압
- 습도

---

## 3-2. 전체 코드

```java
package com.beaver.apr021.main;

import java.io.InputStream;
import java.util.Scanner;

import org.json.simple.JSONArray;
import org.json.simple.JSONObject;
import org.json.simple.parser.JSONParser;

import com.jeesung.http.client.JeesungHttpClient;

public class WeatherMain {

    public static void main(String[] args) {
        try {
            Scanner k = new Scanner(System.in);

            System.out.println("검색어 입력(City): ");
            String cityName = k.next();

            System.out.println(cityName);

            String address = "https://api.openweathermap.org/data/2.5/weather";
            address += "?q=" + cityName;
            address += "&appid=42008a8c8e7402a3fc9d3b1a7df8fee9&units=metric&lang=kr";

            InputStream is = JeesungHttpClient.download(address);
            String str = JeesungHttpClient.convert(is, "UTF-8");

            JSONParser jp = new JSONParser();
            JSONObject jo = (JSONObject) jp.parse(str);

            JSONObject sys = (JSONObject) jo.get("sys");
            String country = (String) sys.get("country");
            System.out.println(country);

            String cityName2 = (String) jo.get("name");
            System.out.println(cityName2);

            JSONArray ja = (JSONArray) jo.get("weather");
            JSONObject jo2 = (JSONObject) ja.get(0);

            String description = (String) jo2.get("description");
            System.out.println(description);

            JSONObject main = (JSONObject) jo.get("main");

            double temp = (double) main.get("temp");
            System.out.println(temp);

            double feelsLike = (double) main.get("feels_like");
            System.out.println(feelsLike);

            double tempMin = (double) main.get("temp_min");
            System.out.println(tempMin);

            double tempMax = (double) main.get("temp_max");
            System.out.println(tempMax);

            long pressure = (long) main.get("pressure");
            System.out.println(pressure);

            long humidity = (long) main.get("humidity");
            System.out.println(humidity);

        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

---

## 3-3. 오타 체크

```java
//JSONArray jo2 = (JSONArray) ja.get(0);
```

이건 오타입니다.

### 이유

`ja.get(0)`의 결과는 배열이 아니라 **배열 안의 첫 번째 객체**이므로,

### 틀린 코드

```java
JSONArray jo2 = (JSONArray) ja.get(0);
```

### 맞는 코드

```java
JSONObject jo2 = (JSONObject) ja.get(0)
```

---

## 3-4. 구조

### `sys`

```java
JSONObject sys = (JSONObject) jo.get("sys");
```

- 국가 코드 등 시스템 정보 포함

### `weather`

```java
JSONArray ja = (JSONArray) jo.get("weather");
JSONObject jo2 = (JSONObject) ja.get(0);
```

- `weather`는 배열
- 첫 번째 요소는 객체
- 그래서 `JSONArray`가 아니라 `JSONObject`로 받아야 함

### `main`

```java
JSONObject main = (JSONObject) jo.get("main");
```

- 온도, 체감온도, 기압, 습도 등의 핵심 날씨 정보 포함

---

## 3-5. 기억 포인트

- JSON 안에 배열이 있으면 `JSONArray`
- 배열 안의 각 원소가 객체이면 `JSONObject`
- 숫자 타입도 형변환 필요
    - 실수형: `double`
    - 정수형: `long`

# 4. JSON 파싱 예제 3 — 카카오 로컬 검색

## 4-1. 예제 목적

중심 좌표를 기준으로,

입력한 키워드가 포함된 주변 장소를 거리순으로 검색

### 조건

- 반경 5km 이내
- 거리 기준 정렬
- 출력:
    - 주소
    - 전화번호
    - 상호명
    - 중심좌표와의 거리

---

## 4-2. 전체 코드

```java
package com.beaver.apr021.main;

import java.io.InputStream;
import java.net.URLEncoder;
import java.util.HashMap;
import java.util.Scanner;

import org.json.simple.JSONArray;
import org.json.simple.JSONObject;
import org.json.simple.parser.JSONParser;

import com.jeesung.http.client.JeesungHttpClient;

public class LocSearchMain {

    public static void main(String[] args) {
        try {
            Scanner k = new Scanner(System.in);

            System.out.println("입력: ");
            String search = k.next();

            search = URLEncoder.encode(search, "UTF-8");

            // 37.5201769 / 127.0285047
            String address = "https://dapi.kakao.com/v2/local/search/keyword.json";
            address += "?query=" + search;
            address += "&x=127.0285047&y=37.5201769";
            address += "&radius=5000";
            address += "&sort=distance";

            HashMap<String, String> headers = new HashMap<String, String>();
            headers.put("Authorization", "KakaoAK b835af25654ec288102a03fec0b9933e");

            InputStream is = JeesungHttpClient.download(address, headers);

            String str = JeesungHttpClient.convert(is, "UTF-8");

            JSONParser jp = new JSONParser();
            JSONObject jo = (JSONObject) jp.parse(str);

            // 총 검색 결과 수
            JSONObject jo2 = (JSONObject) jo.get("meta");
            System.out.println(jo2.get("total_count"));

            // 현재 페이지 결과 수
            System.out.println(jo2.get("pageable_count"));
            System.out.println("---------------------");

            // 주소 / 전화번호 / 상호명 / 거리
            JSONArray ja = (JSONArray) jo.get("documents");
            JSONObject jo3 = null;

            for (int i = 0; i < ja.size(); i++) {
                jo3 = (JSONObject) ja.get(i);

                System.out.println(jo3.get("road_address_name"));
                // System.out.println(jo3.get("address_name"));
                System.out.println(jo3.get("phone"));
                System.out.println(jo3.get("place_name"));
                System.out.println(jo3.get("distance"));
                System.out.println("------------------------");
            }

        } catch (Exception e) {
            e.printStackTrace();
        }
    }
}
```

---

## 4-3. 코드 해설

### 1) 검색어 인코딩

```java
search = URLEncoder.encode(search, "UTF-8");
```

- 한글 검색어를 URL에 넣기 위해 인코딩 필요

### 2) 쿼리스트링 설정

```java
address += "?query=" + search;
address += "&x=127.0285047&y=37.5201769";
address += "&radius=5000";
address += "&sort=distance";
```

### 3) 인증 헤더 추가

```java
HashMap<String, String> headers = new HashMap<String, String>();
headers.put("Authorization", "KakaoAK ...");
```

- 카카오 API는 인증키를 헤더에 넣어야 함

### 4) `meta`와 `documents`

- `meta` : 검색 결과 개수 정보
- `documents` : 실제 장소 데이터 목록

---

## 4-4. 기억 포인트

- 카카오 API는 헤더 인증이 필요
- 검색어는 URL 인코딩 필요
- JSON 구조에서
    - `meta`는 요약 정보
    - `documents`는 실제 데이터 배열

# 5. 배열과 컬렉션(Collection)

## 5-1. 배열

### 정의

- 변수 하나에 데이터를 여러 개 저장하는 구조

### 특징

- `[]` 사용
- 만들 때 **크기가 고정**
- 크기를 한 번 정하면 바꿀 수 없음
- 크기를 모르면 배열 만들기가 곤란함

### 한계

- 데이터 수가 늘어날지 줄어들지 모를 때 불편함

---

## 5-2. 컬렉션(Collection)

### 정의

- Java에서 자료구조를 표현한 클래스들의 묶음

### 자료구조란?

- 데이터를 저장하기 위한 구조

### 대표 계열

- List
- Set
- Map

---

# 6. List 계열

## 6-1. 특징

- 가변 크기 가능
- 순서(index) 있음
- 중복 데이터 허용

## 6-2. 대표 클래스

- `ArrayList`

---

## 6-3. 배열 vs List

### 배열

- 크기 고정
- 데이터가 배열 크기보다 많아지면 저장 불가

### List

- 데이터 수에 따라 크기 유동적
- 요소가 많아지면 늘어나고 적어지면 줄어듦

---

## 6-4. 제네릭(Generic)

### 왜 필요한가?

자료형이 섞이면 다루기 불편함

### 역할

- 자료형 고정

### 문법

```java
클래스명<자료형>
```

예:

```java
ArrayList<String>
ArrayList<Integer>
```

---

## 6-5. 전체 코드 — List 예제

```java
package com.beaver.apr022.main;

import java.util.ArrayList;
import java.util.Comparator;

public class ListMain {

    public static void main(String[] args) {

        ArrayList test = new ArrayList();

        test.add("4월");
        test.add(444);
        test.add("야호!");
        test.add(4.44);
        test.add(true);

        for (Object object : test) {
            System.out.println(object);
        }

        System.out.println("------------");

        // 제네릭 사용
        ArrayList<String> al = new ArrayList<String>();
        al.add("파인애플");

        // al.add(1); // 에러! String만 가능

        ArrayList<Integer> a12 = new ArrayList<Integer>();

        a12.add(100);   // Auto-Boxing
        a12.add(50);

        System.out.println(a12.get(0));   // 해당 인덱스 값
        System.out.println(a12.size());   // 요소 개수

        a12.add(30);
        a12.add(1, 90);   // 1번 자리에 90 삽입
        a12.remove(0);    // 0번 인덱스 값 삭제

        for (Integer ii : a12) {
            System.out.println(ii);
        }

        System.out.println("---------------");

        a12.add(84);
        a12.add(2122);
        a12.add(5646);
        a12.add(55);

        Comparator<Integer> c2 = new Comparator<Integer>() {
            @Override
            public int compare(Integer o1, Integer o2) {
                // return o1.compareTo(o2); // 오름차순
                return o2.compareTo(o1);    // 내림차순
            }
        };

        a12.sort(c2);

        for (Integer i : a12) {
            System.out.println(i);
        }
    }
}
```

---

## 6-6. List 핵심 메서드

### 추가

```java
a12.add(30);
```

### 특정 위치에 삽입

```java
a12.add(1, 90);
```

### 삭제

```java
a12.remove(0);
```

### 값 가져오기

```java
a12.get(0);
```

### 크기 확인

```java
a12.size();
```

### 정렬

```java
a12.sort(c2);
```

---

## 6-7. 기억 포인트

- `ArrayList`는 배열처럼 인덱스 사용 가능
- 배열과 달리 크기가 자동 조절됨
- 실무에서는 제네릭 거의 필수
- 정렬은 `Comparator` 활용

# 7. Set 계열

## 7-1. 특징

- 가변 사이즈
- 중복 데이터 자동 제거
- 순서가 일정하지 않음
- 대표 클래스: `HashSet`

---

## 7-2. 전체 코드 — Set 예제

```java
package com.beaver.apr022.main;

import java.util.HashSet;

public class SetMain {

    public static void main(String[] args) {

        HashSet<String> hs = new HashSet<String>();

        hs.add("aa");
        hs.add("bb");
        hs.add("cc");
        hs.add("dd");
        hs.add("ff");
        hs.add("ee");
        hs.add("gg");
        hs.add("hh");
        hs.add("jj");
        hs.add("jj"); // 중복

        System.out.println(hs.size()); // 중복 제외한 개수

        for (String s : hs) {
            System.out.println(s);
        }
    }
}
```

---

## 7-3. 핵심 해설

```java
hs.add("jj");
hs.add("jj");
```

- 같은 값 두 번 넣어도 하나만 저장됨

```java
System.out.println(hs.size());
```

- 실제 저장된 개수 확인 가능

### 출력 순서

- 입력한 순서대로 나오지 않을 수 있음
- 순서가 랜덤하게 보임

---

## 7-4. 기억 포인트

- **중복 제거가 목적**일 때 유용
- 순서가 중요하면 Set은 부적합

---

# 8. Map 계열

## 8-1. 특징

- 가변 사이즈
- `Key`와 `Value`가 한 쌍
- 인덱스 개념 없음
- 대표 클래스: `HashMap`

---

## 8-2. 핵심 개념

```java
hm.put("아메리카노", 4000);
hm.put("녹차라떼", 5000);
hm.put("레몬에이드", 6000);
hm.put("녹차라떼", 3000);
```

### 결과

- `녹차라떼`라는 key가 중복되었으므로
- 마지막 값인 `3000`으로 덮어쓰기 됨

---

## 8-3. 전체 코드 — Map 예제

```java
package com.beaver.apr022.main;

import java.util.HashMap;

public class MapMain {

    public static void main(String[] args) {

        HashMap<String, Integer> hm = new HashMap<String, Integer>();

        hm.put("아메리카노", 4000);
        hm.put("녹차라떼", 5000);
        hm.put("레몬에이드", 6000);
        hm.put("녹차라떼", 3000); // 같은 key -> 값 덮어쓰기

        System.out.println(hm.get("녹차라떼"));
    }
}
```

---

## 8-4. 핵심 메서드

### 저장

```java
hm.put("아메리카노", 4000);
```

### 조회

```java
hm.get("녹차라떼");
```

- key를 넣으면 그에 대응하는 value 반환

---

## 8-5. 기억 포인트

- Map은 `key → value`
- 같은 key를 다시 넣으면 기존 값이 수정됨
- 인덱스로 접근하지 않음
- 검색용 자료구조로 매우 강력함

---

# 9. 정리

## 9-1. 배열 / List / Set / Map 비교표

| 구분 | 크기 변경 | 순서(index) | 중복 허용 | 접근 방식 |
| --- | --- | --- | --- | --- |
| 배열 | X | O | O | 인덱스 |
| List | O | O | O | 인덱스 |

# 10. 포인트

## JSON 파싱에서 자주 나오는 흐름

```java
String str = JeesungHttpClient.convert(is, "UTF-8");
JSONParser jp = new JSONParser();
JSONObject jo = (JSONObject) jp.parse(str);
```

## JSON 안에서

- 객체면 `JSONObject`
- 배열이면 `JSONArray`

## Collection 선택 기준

- 순서 필요 + 중복 허용 → `List`
- 중복 제거 → `Set`
- Key-Value 구조 → `Map`

# 11. 최종 정리 노트

1. AJAX는 비동기 통신 기술
2. XML보다 JSON이 훨씬 많이 쓰임
3. JSON 파싱은 `JSONParser`, `JSONObject`, `JSONArray`가 핵심
4. Java 컬렉션은 배열의 단점을 보완
5. 컬렉션은 크게 `List`, `Set`, `Map`
6. `ArrayList`는 가장 대표적인 List
7. `HashSet`은 중복 제거
8. `HashMap`은 Key-Value 저장 구조

# 12. 요약정리

### AJAX

- JavaScript 비동기 통신 기술
- 예전엔 XML 많이 사용
- 지금은 JSON이 대세

### JSON

- JavaScript Object Notation
- 객체: `{}`
- 배열: `[]`

### JSON 파싱

- 문자열 → `JSONParser`
- 객체 → `JSONObject`
- 배열 → `JSONArray`
- 값 꺼내기 → `get("키")`

### 날씨 예제 포인트

- `weather`는 배열
- `weather[0]`은 객체
- 따라서:

```java
JSONObject jo2 = (JSONObject) ja.get(0);
```

### Collection

- 배열: 크기 고정
- 컬렉션: 크기 가변

### List

- 순서 O
- 중복 O
- 대표: `ArrayList`

### Set

- 순서 X
- 중복 X
- 대표: `HashSet`

### Map

- `Key : Value`
- Key 중복 X
- 대표: `HashMap`

### 제네릭

```java
ArrayList<String>
HashMap<String, Integer>
```

- 자료형 고정용

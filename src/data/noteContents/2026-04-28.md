# 함수 Function

서비스를 만들다보면 비슷한 기능이 자주 생김.

→ ex) 팝업창, 결제, 로그인 실패, 결제 실패 …

매번 새롭게 만들기보다

⇒ 자주 사용하는 기능들을 하나로 묶어서 불러다 쓰는 게 효율적임.

```jsx
function 함수명 () {

내용

}
```

→ `함수명(파라미터);` 로 호출하면 됨.

```jsx
showError();
```

# 파라미터 있는 함수

```jsx
function sayHello(name) {
	let msg = 'Hello, ${name}';
	alert(msg);

}
sayHello("Beaver");
```

`name`이 파라미터.

함수 호출 시 `"Beaver"`가 name으로 들어감.

### 만약에 이름을 입력하지 않았다면?

```jsx
sayHello(); // 파라미터에 값을 넣지 않고 실행하면 => undefined
```

```jsx
function sayHello(name) {
	let msg = "Hello";
	if (name) {
		msg += ', ${name}';
	}
	alert(msg);
}
sayHello();
sayHello("zzz");
```

값을 안 넣으면 `undefined`.

`if (name)`으로 값이 있을 때만 처리 가능.

# 지역변수 Local Variable

함수 내부에서 접근할 수 있는 변수 : 지역변수

```jsx
function sayHello(name) {
	let msg = "Hellooo";
		// 함수 내부에서 접근할 수 있는 변수 : 지역변수 (Local Variable)
	if (name) {
		msg += `, ${name}`;
	}
	console.log("함수 내부");
	console.log(msg);
}
sayHello("Beaver");
// 지역변수는 함수 내에서 전역변수와 같은 이름으로 지을 수 있고,
// 서로 간섭을 받지 않음.
console.log("함수 호출 후");
console.log(msg);
```

전역변수가 많아지면 관리하기 힘듦.

⇒ 가능하면 지역변수를 사용하는 게 효율적.

### OR를 활용해서 기본값 넣기

```jsx
function sayHello2(name) {
	let newName = name || 'Friend';
	let msg = `Hello, ${newName}`;
	console.log(msg);
}
sayHello2();
sayHello2("asdf");

// 기본값을 설정해서 (default value)
function sayHello3(name = 'Friend') {
	let msg = `Hello, ${name}`;
	console.log(msg);
}
sayHello3();
sayHello3("qwer");

// return으로 값 반환하기
function add(num1, num2) {
	return num1 + num2;
}
let result = add(2,3);
console.log(result);
// return은 반환하는 것과 동시에 함수를 종료하는 기능도 있음 !

// 정리
//	- 함수는 한번에 한 작업만 하는게 좋다.
//		=> 하나의 함수가 여러 작업을 진행한다면,
//			여러개 함수로 구현하기
// - 함수명은 읽기 쉽고, 어떤 기능인지 알 수 있게 네이밍 !!
```

`||`는 값이 없을 때 기본값을 넣을 수 있음.

`return`은 값을 반환하고 함수도 종료함.

# 함수 표현식

```jsx
/*
	let sayHello = function() {
		console.log("Hello");

	}
	sayHello();

*/
```

차이는 호출 타이밍.

함수 선언문은 어디서든 호출 가능.

JS가 실행 전에 함수 선언문을 먼저 찾아 저장해두기 때문.

이것을 호이스팅 Hoisting이라고 함.

```jsx
function sayHello() {
	console.log('Hello');
}
sayHello();

/*
let sayHello2 = function() {
	console.log("Hello");

}
sayHello2();
*/
```

함수 표현식은 호이스팅 적용 X.

코드에 도달해야 생성됨.

⇒ 뭐가 더 좋다기보다는 편하고 익숙한 걸 사용해도 무방.

# 화살표 함수

```jsx
/*
	let add = function(num1, num2) {
		return num1 + num2;
	}
*/

/*
	let add = (num1, num2) => {
		return num1 + num2;
	}
*/

/*
	return 생략 가능 O
	let add = (num1, num2) => {
		num1 + num2;
	}
*/

// 반환할 코드가 한 줄 이라면 => 중괄호 생략 가능 O
let add = (num1, num2) => num1+num2;
console.log(add(3,2));

// 파라미터가 하나라면 => 파라미터가 들어갈 소괄호 생략 가능O
let sayHello2 = name => `Hello, ${name}`;

// 파라미터가 없는 함수라면 소괄호는 생략 가능 X
let asdf = () => console.log('asdf');
asdf();
```

정리하면,

- 코드 한 줄 반환 → `{}` 생략 가능
- 파라미터 하나 → `()` 생략 가능
- 파라미터 없음 → `()` 생략 불

### shift(), unshift() : 배열의 제일 앞에 있는 요소 제거 / 추가


정리

- `pop()` : 끝에서 제거
- `unshift()` : 앞에 추가
- `shift()` : 앞에서 제거

# BeaverValidChecker.js

```jsx

 */
// 잘못됐으면 true값
//(뭔가 문제가 되면 true, 정상적이면 false)

function isEmpty(input) {

// <input>을 넣었을떄
// 한글/ 특수문자 적혀있다면 true, 없다면 false
function containsAnother(input) {
	let pass = "1234567890zxcvbnmasdfghjklqwertyuiopZXCVBNMASDFGHJKLQWERTYUIOP@_.";
	let iv = input.value;
	for (let i = 0; i < iv.length; i++) {
		if (pass.indexOf(iv[i]) == -1) { // pass 에 있는 내용중에서 내가 쓴 내용이 없으면

			return true;
			}
	}
	return false;
}

// <input>, 글자 수를 넣었을 때
// 그 글자 수 보다 적으면 true, 그 글자수 이상이면 false
function atLeastLetter(input, len) {
	return (input.value.length < len);
}

// <input> x 2 넣었을 때
// 내용이 다르면 true, 같으면 false (pw, pw 확인)
function notEqualPw(input1, input2) {
	return (input1.value != input2.value);
}

// <input>, 문자열세트 넣었을 때
// 없으면 true, 있으면 false, (입력한 pw에 지정한 문자열이 없는 경우)
function notContain(input, passSet) {
	let iv = input.value;
	for (let i = 0; i < passSet.length; i++) {
		if (iv.indexOf(passSet[i]) != -1) {
			return false;
		}
	}
	return true;
}

// <input> 넣었을 때
//숫자 아닌거 있으면 true, 숫자만 있으면 false
function isNotNumber(input) {
	return isNaN(input.value);
}

// <input>, 확장자 넣었을 때
// 확장자명이 아니면 true, 맞으면 false
function isNotType(input, type) {
	// .png, .gif, .bmp, ...
	type = "." + type;
	return (input.value.inderOf(type) == -1);
}
```

함수별 핵심

- `isEmpty()` : 비어있으면 true
- `containsAnother()` : 허용 문자 외 값이 있으면 true
- `atLeastLetter()` : 최소 글자 수보다 적으면 true
- `notEqualPw()` : 두 값이 다르면 true
- `notContain()` : 지정 문자열이 하나도 없으면 true
- `isNotNumber()` : 숫자가 아니면 true
- `isNotType()` : 지정 확장자가 아니면 true

# 최종정리

- 함수 = 자주 쓰는 코드 묶음
- 파라미터 = 함수 안으로 들어오는 값
- return = 값 반환 + 함수 종료
- 전역변수 = 어디서나 접근 가능
- 지역변수 = 함수 안에서만 접근 가능
- 함수 선언문 = 호이스팅 O
- 함수 표현식 = 호이스팅 X
- 화살표 함수 = 짧은 함수 문법
- 객체 = key/value 묶음
- property = key/value 한 쌍
- method = 객체 안의 함수
- 배열 = 순서 있는 값 묶음
- 배열 인덱스 = 0부터 시작
- push/pop = 끝 추가/제거
- unshift/shift = 앞 추가/제거
- document = 현재 HTML 문서
- input.value = 입력값
- return false = 전송 막기
- return true = 전송 허용
- isNaN() = 숫자 아닌지 검사
- indexOf() = 포함 여부 검사, 없으면 -1
- GET = 주소에 데이터 붙여 전송
- POST = body에 데이터 담아 전송
- 파일 업로드 form = `enctype="multipart/form-data"` 필요
- CSS id = `#`
- CSS class = `.`

### 제일 비싼 메뉴 이름, 가격

```sql
select m_name, m_price from menu where m_price = (select max(m_price) from menu); -- 내 코드
select m_name, m_price from menu where m_price in (select max(m_price) from menu); -- 또 다른 정답 코드
```

> 💡 `=` 와 `in` 모두 가능하지만,
> 
> 
> 서브쿼리 결과가 **하나만 나온다면 `=`**, 여러 개 가능성을 열어두면 `in`도 사용할 수 있음.

## 3. 테이블 여러 개를 JOIN 시킨다

### 의미

> 테이블 여러 개를 붙여서 RAM에 잠깐 넣어놓는 느낌
> 

이렇게 테이블을 붙여서 사용 가능하지만,

테이블들이 합쳐지면서 나타낼 수 있는 **모든 값의 조합**이 생김.

```sql
select * from franchise, ceo;
```

따라서 **조건식**을 활용해야 함.

```sql
select * from franchise, ceo where f_ceo = c_no;
```

---

### 전체 식당 이름, 지점명, 사장님 이름, 사장님 생일

### 내 코드

```sql
select f_name, f_location, c_name, c_birthday from franchise, ceo; -- 내 코드
```

### 정답 코드

```sql
select f_name, f_location, c_name, c_birthday from franchise, ceo where f_ceo = c_no; -- 정답 코드
select m_name, m_price, f_name, f_location from menu, franchise where m_f_no = f_no; -- 정답 코드
```

> 💡 조건 없이 여러 테이블을 조회하면 **Cross Join 형태**가 되어 경우의 수가 전부 나옴.
> 

---

### 좌석 수가 50개 이상인 식당의 메뉴명, 가격, 식당이름, 지점명, 좌석 수

### 내 코드

```sql
select m_name, m_price, f_name, f_location, f_seat from franchise, ceo, menu where -- 내 코드
```

### 정답 코드

```sql
select m_name, m_price, f_name, f_location, f_seat from menu, franchise where m_f_no = f_no and f_seat >= 50; -- 정답 코드
```

> 💡 여기서는 `ceo` 테이블이 필요 없음.
> 

---

### 규모가 제일 큰 식당의 메뉴명, 가격, 식당이름, 지점명, 좌석 수

### 내 코드

```sql
select m_name, m_price, f_name, f_location, f_seat from franchise, ceo, menu where c_no in (select f_ceo from franchise where f_seat in(select max(f_seat) from franchise)); -- 내 코드
```

### 정답 코드

```sql
select m_name, m_price, f_name, f_location, f_seat from franchise, menu where m_f_no = f_no and f_seat in (select max(f_seat) from franchise); -- 정답 코드
```

---

### 최연장자 사장님네 전체 메뉴명, 가격, 식당이름, 지점명, 사장님 이름, 생일을 메뉴명 가나다순 → 식당이름 가나다순 정렬

### 내 코드

```sql

```

### 정답 코드

```sql
select m_name, m_price, f_name, f_location, c_name, c_birthday from franchise, menu, ceo where m_f_no = f_no and f_ceo = c_no and c_birthday = (select min(c_birthday) from ceo) order by m_name, f_name;
```

> 💡 여기서도 **최연장자이므로 `min(c_birthday)`** 사용.

## 5. ROWNUM (가상필드) ********

> 💡
> 
> - `select` 할 때마다 자동으로 부여됨
> - `order by` 보다 먼저 부여됨
> - **무조건 1번부터 조회해야 함!**

---

### 메뉴 이름을 가나다 순으로 정렬해서 메뉴 전체 정보를 조회

### 구문오류

```sql
select rownum, * from menu order by m_name; -- 구문오류
```

### 내 코드

```sql
select rownum, m_no, m_name, m_price, m_f_no from menu order by m_name; -- 내 코드
```

### 정답 코드

```sql
select rownum, m_no, m_name, m_price, m_f_no from menu where rownum >= 1 and rownum <= 3 order by m_name; -- 정답 코드
```

위와 같이 작성하면 `order by` 하기 전에 `rownum`이 생성되어서

sequence를 먹인 `m_no`와 비슷하게 나와버림.

우리가 의도했던 바는:

1. 이름 순으로 정렬하고
2. 그 다음 번호값(`rownum`)을 부여하는 것

```sql
select rownum, m_no, m_name, m_price, m_f_no from menu where rownum >= 1 and rownum <= 3 order by m_name;
```

코드상 결과 같이 나오게 하려면 위와 같이 작성하고 싶은데, 문법/동작상 원하는 결과가 나오지 않음.

```sql
select rownum, m_no, m_name, m_price from (select m_no, m_name, m_price from menu order by m_name) where rownum >= 1 and rownum <= 3;
```

> 💡 핵심:
> 
> 
> **정렬 먼저 → 그 결과에 ROWNUM 부여**를 하려면
> 
> `FROM` 절 안에 정렬된 서브쿼리를 넣어야 함.
>

> - 쿼리문 종료 후 사라지는 **임시적인 뷰(View)**

## 7. VIEW(뷰)

> 💡
> 
> 
> 사용자에게 접근이 허용된 자료만 제한적으로 보여주기 위해 사용
> 

### 그럼 진짜 테이블을 쓰면 되는데 왜 굳이 뷰를 쓸까?

1. 숨기고 싶은 정보가 있다면
    
    뷰를 생성할 때 해당 컬럼을 빼고 생성함으로써 **정보 보안**에 좋다.
    
2. 기존 테이블의 구조가 변경되어도
    
    뷰의 구조를 변경하지 않아도 되는 경우가 있다.
    

---

### 메뉴 테이블 정보를 메뉴명 가나다 순으로 정렬 → rownum 2 ~ 4 정보 조회

```sql
select rownum, m_no, m_name, m_price from (select m_no, m_name, m_price from menu order by m_name) where rownum >= 2 and rownum <= 4;
```

하지만 `rownum`은 **무조건 1부터 조회해야 해서** 정보가 출력되지 않음.

`select`를 겉에 한 번 더 씌워주면

안에 있는 `rownum`이 숫자만 부여해놓은 상태가 되어 바깥에서 범위 조건을 걸 수 있음.

```sql
select * from (select rownum rn, m_no, m_name, m_price from (select m_no, m_name, m_price from menu order by m_name)) where rn >= 2 and rownum < 4;
```

> 💡 보통은 바깥쪽 조건도 `rownum`이 아니라 **별칭 `rn`** 으로 맞춰서 사용함.
> 
> 
> 예: `where rn between 2 and 4`
>

## 8. 실전 문제: 페이징 + 정렬 + 조건

### ~길동 이름 가진 사장님네

### 메뉴명, 가격, 식당명, 지점명, 사장님 이름 조회

### 가격 내림차순으로 조회 → 메뉴명 가나다순 정렬 후

### 3 ~ 8번째까지 조회

### 내 코드

```sql
select * from (select rownum rn, m_no, m_name, m_price, f_name, f_location, c_name from (select m_no, m_name, m_price, f_name, f_location, c_name from menu, franchise, ceo order by m_price)) where rn >= 3 and rn <= 8; -- 내 코드
```

### 정답 코드

```sql
select * from (select rownum rn, m_name, m_price, f_name, f_location, c_name from (select m_name, m_price, f_name, f_location, c_name from menu, franchise, ceo where m_f_no = f_no and f_ceo = c_no and c_name like '%길동' order by m_price desc, m_name)) where rn between 3 and 8; -- 정답 코드
```

> 💡 빠진 핵심
> 
> - 조인 조건: `m_f_no = f_no and f_ceo = c_no`
> - 이름 조건: `c_name like '%길동'`
> - 정렬 조건: `order by m_price desc, m_name`

---

### 김씨 성을 가진 사장님네 메뉴명, 가격, 사장님이름, 사장님 성별

### 가격 오름차순 정렬 → 2 ~ 3번만

### 내 코드

```sql
select * from (select rownum rn, m_no, m_name, m_price, c_name, c_gender from (select m_no, m_name, m_price, c_name, c_gender from menu, franchise, ceo order by m_name)) where rn >= 3 and rn <= 7; -- 내 코드
```

### 정답 코드

```sql
select * from (select rownum rn, m_name, m_price, c_name, c_gender from (select m_name, m_price, c_name, c_gender from menu, ceo, franchise where m_f_no = f_no and c_no = f_ceo and c_name like '김%' order by m_price)) where rn between 2 and 3; -- 정답 코드
```

> 💡 빠진 핵심
> 
> - 조인 조건 누락
> - `김씨` 조건 누락: `c_name like '김%'`
> - 문제는 2~3번인데, 내 코드는 3~7번 조회 중

## 9. JOIN

> 💡
> 
> 
> 하나의 명령문에 의해 여러 테이블에 저장된 데이터를 한번에 조회할 수 있는 기능
> 
> 마치 하나의 테이블인 것처럼 보여줌
> 

---

## 데이터

```sql
create table tb1 (name varchar2(10 char), age number(3));
create table tb2 (name varchar2(10 char), age number(3));

insert into tb1 values('김비버', 30);
insert into tb1 values('박비버', 35);
insert into tb1 values('최비버', 40);
insert into tb1 values('오비버', 40);

insert into tb2 values('김비버', 30);
insert into tb2 values('정비버', 50);
insert into tb2 values('이비버', 55);

select * from tb1, tb2;
```

---

## 10. Cross Join

> 💡
> 
> 
> 별도의 조건 없이 두 테이블 간의 가능한 모든 결과를 조회
> 
> (= 모든 경우의 수)
> 

```sql
select * from tb1, tb2;
select * from tb1 cross join tb2;
```

---

## 11. Inner Join

> 조건에 해당하는 값만 나옴
> 
> 
> 테이블에서 지정한 조건을 만족하는 행만 반환
> 

```sql
select * from tb1 inner join tb2 on tb1.name = tb2.name;
```

---

## 12. EQUI 조인 (동등 조인)

```sql
select * from tb1 A, tb2 B where A.name = B.name;
```

## 13. EQUI 조인 - JOIN ~ USING (조인대상컬럼명)

```sql
select * from tb1 join tb2 using (name);
```

> 💡 `using(name)` 은 양쪽 테이블에 **같은 이름의 컬럼**이 있을 때 사용.

## 14. 자연조인 (Natural Join)

> 조건절 없이 양쪽에 같은 이름을 가진 동일한 컬럼만을 기준으로 조회
> 

```sql
select * from tb1 natural join tb2;
```

> 💡 `natural join`은 편하지만,
> 
> 
> 같은 이름의 컬럼이 늘어나면 의도치 않게 조건이 추가될 수 있어 조심해야 함.
> 

---

## 15. Outer Join

> 💡
> 
> 
> 기준 테이블의 데이터가 **모두 조회(누락 없이)** 되고,
> 
> 대상 테이블의 데이터가 있는 경우 해당 컬럼 값을 가져오기 위해 사용
> 

`(+)` 를 이용한 outer join

→ **null이 출력되는 테이블의 컬럼에 `(+)` 추가**

---

## 16. Left Outer Join

> 💡
> 
> 
> 왼쪽 테이블에 값이 있으면 오른쪽 테이블이 조건에 맞지 않아도 나옴
> 
> (조건에 맞지 않으면 `NULL`로 채움)
> 

> Right보다는 Left를 더 많이 씀
> 
> 
> (왼쪽 테이블을 기준으로 두는 경향 O)
> 

```sql
select * from tb1 left outer join tb2 on tb1.name = tb2.name;
select * from tb1, tb2 where tb1.name = tb2.name(+);
```

---

## 17. Right Outer Join

> 💡
> 
> 
> 오른쪽 테이블에 값이 있으면 왼쪽 테이블이 조건에 맞지 않아도 나옴
> 
> (조건에 맞지 않으면 `NULL`로 채움)
> 

```sql
select * from tb1 right outer join tb2 on tb1.name = tb2.name;
```

---

## 18. Full Outer Join

> 💡
> 
> 
> 한쪽 테이블에 값이 있으면 다른 쪽 테이블이 조건에 맞지 않아도 나옴
> 

```sql
select * from tb1 full outer join tb2 on tb1.name = tb2.name;
```

## 19. Self Join **자주 헷갈림..!!**

> 💡
> 
> 
> 하나의 테이블 내에 있는 컬럼끼리 연결하는 join이 필요한 경우
> 
> 단어 뜻대로 **스스로 JOIN** 하는 것
> 

---

### 예시 상황

- 한 웹사이트에 회원이 5명이 있고, 관리자가 2명이 있음
- 회원의 이름은 각각 회원1~5, 관리자의 이름은 관리자1, 관리자2
- 회원의 아이디는 각각 `member1 ~ 5`, 관리자의 아이디는 `manager1, 2`
- 회원 1~3번은 관리자 1이 관리 / 나머지 회원은 관리자 2가 관리

```sql
create table member (
	m_id varchar2(10 char) primary key,
	m_name varchar2(10 char),
	m_manager varchar2(10 char)
); -- 관리자들은 관리를 안 받으니 제약조건 X

drop table member cascade constraint purge;

insert into member values('member1', '회원1', 'manager1');
insert into member values('member2', '회원2', 'manager1');
insert into member values('member3', '회원3', 'manager1');
insert into member values('member4', '회원4', 'manager1');
insert into member values('member5', '회원5', 'manager1');
insert into member values('manager1', '관리자1', null);
insert into member values('manager2', '관리자2', null);
```

---

### 각 관리자가 어떤 회원을 관리하는지 알고 싶음 (join ~ on)

### 관리자 ID, 관리자 이름, 회원 ID

```sql
select * from member join member B on A.m_id = B.m_id;

select A.m_id, A.m_name, B.m_id from member A join member B on A.m_id = B.m_manager;
```

### 같은 테이블이지만, 셀프조인을 하기 위해

### 테이블에 별칭을 넣어서 구분해줌

- 계정 ID(A) 와 회원의 관리자ID(B)이 같은 값들을 연결해서
- 관리하는 회원 ID값을 가져올 수 있게 함

> 💡 Self Join 핵심
> 
> - **같은 테이블을 서로 다른 역할로 두 번 쓰는 것**
> - 그래서 반드시 **별칭(A, B)** 으로 구분해야 함

---

### 한 줄 요약

> **select → from → where(join) → order by → rownum**
> 

## 최종 정리

- **최연장자** = `min(c_birthday)`
- **최연소** = `max(c_birthday)`
- **제일 비쌈** = `max(가격)`
- **제일 쌈** = `min(가격)`
- **join 조건 없으면** 모든 경우의 수 출력
- **inner join** = 조건 맞는 것만
- **outer join** = 기준 테이블은 다 나옴
- **self join** = 같은 테이블끼리, **별칭 필수**
- **rownum은 order by보다 먼저 붙음**
- **정렬 후 rownum 필요** → `inline view`
- **n ~ m 조회** → `rownum 별칭 rn` 준 뒤 바깥에서 조회

```sql
-- 최댓값 / 최솟값
where 컬럼 = (select max(컬럼) from 테이블);
where 컬럼 = (select min(컬럼) from 테이블);

-- join
select 컬럼
from A, B
where A.컬럼 = B.컬럼;

-- 정렬 후 상위 n개
select rownum, 컬럼
from (
	select 컬럼
	from 테이블
	order by 기준
)
where rownum <= n;

-- 정렬 후 n ~ m번째
select *
from (
	select rownum rn, 컬럼
	from (
		select 컬럼
		from 테이블
		order by 기준
	)
)
where rn between n and m;
```

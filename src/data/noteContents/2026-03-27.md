# 1. 추상화 (Abstraction)

공통의 속성이나 기능을 묶어서 이름을 붙이는 것.

즉, 여러 객체들이 공통적으로 가지는 특징을 뽑아내어

하나의 개념으로 정리하는 과정이다.

예를 들어 동물이라면 공통적으로

- 종
- 나이
- 울기

같은 요소를 가질 수 있다.

이런 공통점을 묶어서 만든 것이 추상화다.

---

## 추상 메소드

```java
public abstract void roar();
```

### 의미

`abstract` 메소드는 **미완성된 메소드**이다.

메소드의 이름과 형태만 있고, 내용(구현부)은 없다.

즉, 상위 클래스에서는 “이런 기능이 있어야 한다”까지만 정하고,

실제 동작은 하위 클래스에서 반드시 완성해야 한다.

### 핵심

하위 클래스에서 반드시 **오버라이딩(재정의)** 해야 한다.

---

## 추상 클래스

```java
public abstract class Animal {
```

### 의미

추상 클래스는 **실체 클래스들이 공통적으로 가지는 부분을 추출해서 만든 추상적인 클래스**이다.

즉,

- 공통 속성은 미리 만들어두고
- 공통 기능도 일부 구현해두고
- 꼭 필요한 기능은 추상 메소드로 남겨둘 수 있다

라는 특징이 있다.

### 핵심

- 객체 생성 불가능
- 상속을 통해서만 사용 가능
- 일반 메소드 + 추상 메소드 함께 사용 가능

---

## Animal 추상 클래스

```java
package com.beaver.mar271.animal;

// abstract class :

// 사용할 실체 클래스의 공통적인 부분들을 추출해

// 어느정도 규격을 잡아놓은 추상적인 클래스

public abstract class Animal {

// 종, 나이  -

private String sort;

private int age;

public Animal(String sort, int age) {

super();

this.sort = sort;

this.age = age;

}

public String getSort() {

return sort;

}

public void setSort(String sort) {

this.sort = sort;

}

public int getAge() {

return age;

}

public void setAge(int age) {

this.age = age;

}

public void printInfo() {

System.out.println(sort);

System.out.println(age);

}

// abstract 메소드 : 미완성된 메소드 (내용이 없는)

// 하위클래스에서 반드시!! 오버라이딩해서 이 메소드를 완성해야

public abstract void roar();

// abstract 메소드(추상 메소드)가 하나라도 있으면

// abstract 클래스 or interface 여야 함!

}
```

### 정리

`Animal`은 직접 사용할 동물 객체가 아니라,

동물들이 공통적으로 가져야 할 기준을 제시하는 클래스다.

- `sort`, `age`는 공통 속성
- `printInfo()`는 공통 기능
- `roar()`는 동물마다 다르므로 추상 메소드로 선언

---

## Bird 클래스

```java
package com.beaver.mar271.animal;

public class Bird extends Animal {

private int leg;

public Bird(String sort, int age, int leg) {

super(sort, age);

this.leg = leg;

}

public int getLeg() {

return leg;

}

public void setLeg(int leg) {

this.leg = leg;

}

@Override

public void printInfo() {

super.printInfo();

System.out.println(leg);

}

@Override

public void roar() {

System.out.println("짹짹!");

}

}
```

### 정리

`Bird`는 `Animal`을 상속받은 구체적인 클래스다.

- `Animal`의 공통 속성인 종, 나이를 물려받음
- 새만의 속성인 `leg` 추가
- `printInfo()` 재정의
- `roar()`를 반드시 구현

즉, 추상 클래스에서 정해둔 규격을 실제 클래스로 완성한 예제다.

# 2. 인터페이스 (Interface)

인터페이스는 **클래스들이 반드시 구현해야 하는 동작을 지정하는 추상 자료형**이다.

→

“이 기능은 무조건 만들어라” 라고 규칙을 정하는 역할.

---

## 클래스 / 추상 클래스 / 인터페이스

### 클래스

- 멤버변수 사용 가능
- 메소드 사용 가능
- 객체 생성 가능

### 추상 클래스

- 멤버변수 사용 가능
- 일반 메소드 사용 가능
- 추상 메소드 사용 가능
- 객체 생성 불가능

### 인터페이스

- 추상 메소드 사용 가능
- 멤버상수 사용 가능
- 객체 생성 불가능

---

## Bird가 인터페이스 구현

```java
public class Bird extends Animal implements Bite {
```

### 의미

- `Animal`은 상속
- `Bite`는 구현

즉, 새는 동물이면서 물 수 있는 기능도 가지게 된다.

---

## 메인 실행

```java
package com.beaver.mar271.main;

import com.beaver.mar271.animal.Bird;

public class AMain1 {

public static void main(String[] args) {

// 공통의 속성이나 기능을 묶어 이름을 붙이는 것 : 추상화

Bird b = new Bird("삼족오", 999, 3); {

b.printInfo();

b.roar();

b.bite();

}

};

}
```

### 정리

`Bird` 객체를 만들어서

- 정보 출력
- 울기
- 물기

기능을 실행한 예제다.

---

# 3. 다중 상속과 인터페이스

클래스끼리는 **다중 상속이 불가능**하다.

즉,

```java
class A extends B, C
```

이런 식은 안 된다.

하지만

- 클래스 1개 상속
- 인터페이스 여러 개 구현

은 가능하다.

즉,

```java
class Ironman extends Avengers implements Smoke1, Drive
```

이건 가능하다.

# 5. 익명 내부 클래스 (Anonymous Inner Class)

## 개념

익명 내부 클래스는 이름이 없는 클래스다.

특징:

- 클래스명을 따로 만들지 않음
- 단 한 번만 사용할 목적
- 한 개의 객체만 만들기 위한 일회용 클래스
- 다른 곳에서 재사용 불가

→

“지금 여기서 잠깐만 상속해서 쓸게” 같은 느낌.

---

## 예제

```java
package com.beaver.mar271.main;

import com.beaver.mar271.avengers.Avengers;

public class AMain2 {

public static void main(String[] args) {

// 익명 내부 클래스 (Anonymous inner class)

// Avengers 로부터 상속받은..

// 클래스명은 모름

// 내부에 만들어놓은 클래스

// 단 한번만 사용될 수 있고,

//오직 하나의 객체만을 사용할 수 있는 일회용 클래스

// ( = 다른 곳에서 불러올수 없음)

// 어벤져스 클래스

Avengers a = new Avengers("헐크", 55) { // 중괄호 안쪽부분이 클래스 부분.

private int hand;

@Override

public void attack() {

// TODO Auto-generated method stub

System.out.println("smash");

}

};

// 이름, 나이 속성

// 공격하기 기능

}

}
```

### 정리

`Avengers`는 추상 클래스라서 원래 객체 생성이 불가능하다.

하지만 익명 내부 클래스를 이용하면

그 자리에서 바로 `attack()`을 구현해서 객체를 만들 수 있다.

즉,

- 상속용 클래스를 따로 만들 필요 없이
- 한 번만 사용할 객체를 빠르게 생성 가능

# 7. 다형성 (Polymorphism)

## 개념

`Poly` = 많은

`Morph` = 형태

즉, **하나의 상위 타입으로 여러 하위 객체를 다룰 수 있는 성질**이다.

---

## 예시

### 일반 객체 생성

```java
Ironman i = new Ironman("a", 123, 123123);
```

### 다형성 사용

```java
Avengers a = new Ironman("b", 124, 234234);
```

### 의미

실제로 만들어진 객체는 `Ironman`이지만

그 객체를 담는 변수의 타입은 상위 클래스인 `Avengers`이다.

즉,

- 겉모습은 `Avengers`
- 실제 내용물은 `Ironman`

이다.

---

## 왜 중요한가

```java
a.attack();
```

여기서 `a`는 `Avengers` 타입이지만

실제로는 `Ironman` 객체를 참조하고 있기 때문에

`Ironman`에서 오버라이딩한 `attack()`이 실행된다.

이것이 바로

**다형성과 오버라이딩이 밀접하게 연결되는 이유**다.

### 장점

- 코드가 유연해짐
- 유지보수가 쉬워짐
- 확장성이 좋아짐

---

# 8. 싱글톤 패턴 (Singleton Pattern)

## 개념

클래스의 객체를 **딱 하나만 생성**해서 공유하는 디자인 패턴.

즉,

- 최초 한 번만 객체 생성
- 그 뒤에는 같은 객체를 계속 사용

한다.

### 장점

- 메모리 낭비 방지
- 하나의 공용 객체를 안정적으로 사용 가능

---

## 실행 예제

```java
package com.beaver.mar272.main;

import com.beaver.mar272.beaver.Profile;

public class SMain1 {

public static void main(String[] args) {

// 원하는 객체 하나 생성

// 이름, 나이, 별명

// 출력

Profile p = new Profile("홍길동", 20, "서자");

p.printInfo();

System.out.println(p);

Profile p2 = new Profile("홍길동2", 22, "서자2");

p2.printInfo();

System.out.println(p2);

//유일한 비버였으면 좋겠음!

→

디자인 패턴 프로그래밍 중에

- 싱글톤 패턴(Singleton Pattern)

클래스가 최초 한번만 메모리를 할당하고 (Static)

그 메모리에 객체를 만들어 사용하는 디자인 패턴

고정된 메모리 영역을 얻으면서

한번의 new로 객체를 사용하기 때문에 메모리 낭비를 방지할 수 있음.

/*

Profile p3 = new Profile("비버", 5, "날쌘돌이");

*/

Profile b1 = Profile.getBeaver();

b1.printInfo();

System.out.println(b1);

Profile b2 = Profile.getBeaver();

b2.printInfo();

b2.setAge(150);

System.out.println(b2);

System.out.println();

}

}
```

---

## Profile 클래스

```java

private int age;

private String Nick;

*/

// TODO Auto-generated constructor stub

}

public Profile() {

super();

}

return BEAVER;

}

public void printInfo() {

/*

System.out.println("이름: "+name);

System.err.println("나이: "+age);

System.out.println("별명: "+Nick);

*/

}

// TODO Auto-generated method stub

```

### 정리

`BEAVER`라는 객체를 `static final`로 하나만 생성해 둠.

```java
private static final Profile BEAVER = new Profile("비버", 3, "임포스터");
```

그리고 외부에서는

```java
Profile.getBeaver();
```

를 통해서만 접근한다.

즉, `b1`, `b2`는 서로 다른 객체가 아니라

**같은 객체를 가리키는 것**이다.

그래서 `b2.setAge(150);`를 하면

`b1`에도 영향이 간다.

---

## 싱글톤 핵심 포인트

- 객체를 하나만 만든다
- `static`으로 관리한다
- 생성자를 제한해서 외부에서 마음대로 `new`하지 못하게 하는 구조로 자주 설계한다

# 9. Factory 패턴

## 개념

객체를 직접 만들지 않고,

**전문적으로 객체를 만들어주는 클래스(공장)** 를 두는 방식이다.

즉,

- 객체 생성 로직을 따로 분리
- 대량 생산, 자동화에 적합

---

## Student 클래스

```java

/*

private static final Student STUDENT1 = new Student("홍길동", 21);

private static final Student STUDENT2 = new Student("아무개", 24);

*/

private int age;

private int num;

super();

this.name = name;

this.age = age;

this.num = num;

}

return name;

}

public void setName(String name) {

this.name = name;

}

public int getAge() {

return age;

}

public void setAge(int age) {

this.age = age;

}

public int getNum() {

return num;

}

public void setNum(int num) {

this.num = num;

}
```

# 11. 오류

## 1) 추상 클래스에는 객체 생성 불가

```java
new Animal(...)
```

불가능

```java
new Avengers(...)
```

도 원칙적으로 불가능

단, **익명 내부 클래스**로 구현부를 바로 작성하면 가능

---

## 2) 추상 메소드가 하나라도 있으면

그 클래스는 반드시

- `abstract class`
    
    또는
    
- `interface`

여야 한다.

---

## 3) 클래스는 다중 상속 불가

```java
class A extends B, C
```

불가능

하지만

```java
class A extends B implements C, D
```

가능

---

## 4) 오버라이딩은 다형성과 연결됨

상위 타입 변수로 하위 객체를 참조해도

실행되는 것은 실제 객체의 오버라이딩된 메소드다.

---

## 5) 실행문은 main 메소드 안에

중간에 나온 이 코드는

```java
Ironman i = new Ironman("스타크", 40, 234105851);

i.printInfo();

i.attack();

i.smoke1();

i.drive();
```

반드시 `main()` 같은 메소드 안에 있어야 정상 실행된다.

즉, 필기 마지막에 적힌 내용처럼

**main에서 실행시키지 않아서 오류난 것**이라고 이해하면 된다.

# 13. 암기용

## 1) 추상

공통 속성/기능을 묶어서 이름 붙이는 것

---

## 2) 추상 메소드

```java
public abstract void roar();
```

- 내용이 없는 미완성 메소드
- 하위 클래스가 반드시 오버라이딩해야 함

---

## 3) 추상 클래스

```java
public abstract class Animal
```

- 공통 속성과 기능을 미리 정의
- 추상 메소드 포함 가능
- 객체 생성 불가능

---

## 4) 인터페이스

- 클래스들이 구현해야 하는 동작 규칙
- 추상 메소드, 상수 사용 가능
- 객체 생성 불가능
- `implements`로 구현

---

## 5) 클래스 / 추상클래스 / 인터페이스 차이

### 클래스

- 변수 O
- 메소드 O
- 객체 생성 O

### 추상 클래스

- 변수 O
- 일반 메소드 O
- 추상 메소드 O
- 객체 생성 X

### 인터페이스

- 추상 메소드 O
- 상수 O
- 객체 생성 X

---

## 6) 오버라이딩

부모가 준 메소드를 자식이 자기 방식으로 다시 구현하는 것

---

## 7) 다형성

상위 타입 변수에 하위 타입 객체를 담을 수 있는 것

```java
Avengers a = new Ironman(...);
```

- 변수 타입은 부모
- 실제 객체는 자식
- 호출 시 자식이 오버라이딩한 메소드 실행

---

## 8) 익명 내부 클래스

- 이름 없는 클래스
- 1회용
- 한 번만 사용 가능
- 추상 클래스/인터페이스를 즉석에서 구현할 때 사용

---

## 9) 싱글톤 패턴

- 객체를 하나만 생성
- `static`으로 공유
- 메모리 절약
- 같은 객체를 계속 사용

---

## 10) 팩토리 패턴

- 객체를 직접 만들지 않고 공장 클래스가 대신 생성
- 대량생산, 자동화에 적합

---

## 11) 다중 상속

- 클래스끼리 다중 상속 X
- 클래스 + 인터페이스 여러 개 구현 O

```java
class Ironman extends Avengers implements Smoke1, Drive
```

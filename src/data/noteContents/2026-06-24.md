텍스트 전처리는 **사람의 언어를 컴퓨터가 처리 가능한 숫자 형태로 바꾸는 과정**이다.

자연어 처리 모델의 성능은 전처리 품질에 크게 좌우된다.

---

## 1. AI / ML / DL

### AI

AI는 인간 지능과 유사한 기능을 컴퓨터 시스템으로 구현하는 전체 분야다.

### ML

Machine Learning은 데이터를 기반으로 패턴을 학습하는 기술이다.

사람이 모든 규칙을 직접 작성하지 않아도 모델이 데이터 속 관계를 찾아낸다.

### DL

Deep Learning은 인공신경망을 이용해 더 복잡한 패턴을 학습하는 방식이다.

텍스트, 이미지, 음성처럼 복잡한 데이터 처리에 강하다.

---

## 2. 텍스트 전처리

### 정제 Cleaning

노이즈 데이터를 제거하는 작업이다.

예시:

- 특수문자 제거
- HTML 태그 제거
- 불필요한 공백 제거

### 정규화 Normalization

같은 의미지만 표현이 다른 단어를 하나로 통일하는 작업이다.

예시:

```
US, USA, U.S.A → USA
Apple, apple → apple
```

---

## 3. 토큰화 Tokenization

토큰화는 문장이나 문서를 작은 단위로 나누는 작업이다.

영어는 공백과 구두점을 기준으로 단어를 나누기 쉽다.

한국어는 조사와 어미가 단어에 붙기 때문에 형태소 분석이 중요하다.

### 영어 토큰화 도구

| 도구 | 특징 |
| --- | --- |
| `word_tokenize` | NLTK 기본 단어 토큰화 |
| `WordPunctTokenizer` | 구두점까지 강하게 분리 |
| `text_to_word_sequence` | 소문자화, 구두점 제거 자동 처리 |

---

## 4. 문장 토큰화

### 영어

```python
from nltk.tokenize import sent_tokenize

sent_tokenize(sentence)
```

`Mr.`, `Dr.` 같은 약어를 고려해 문장을 분리한다.

### 한국어

```python
import kss

kss.split_sentences(kor)
```

한국어 문장 분리에는 KSS 같은 한국어 전용 도구를 사용할 수 있다.

## 5. 품사 태깅 POS Tagging

품사 태깅은 토큰화된 단어에 품사를 붙이는 작업이다.

예시:

```python
from nltk.tag import pos_tag

pos_tag(tokenized_sentence)
```

중요 태그:

| 태그 | 의미 |
| --- | --- |
| PRP | 인칭대명사 |
| RB | 부사 |
| DT | 관사/한정사 |
| JJ | 형용사 |
| NN | 단수 명사 |
| NNS | 복수 명사 |
| MD | 조동사 |
| VBD | 과거형 동사 |
| VBG | 동명사/현재분사 |

## 6. 한국어 형태소 분석

한국어는 교착어라서 조사, 어미, 접사가 단어에 붙는다.

따라서 단순 공백 기준 토큰화로는 정확한 분석이 어렵다.

### KoNLPy 주요 분석기

- Okt
- Komoran
- Kkma
- Hannanum

### Okt 예시

```python
from konlpy.tag import Okt

okt = Okt()

okt.morphs("오늘은 수요일이고요. 내일은 목요일입니다!")
okt.pos("오늘은 수요일이고요. 내일은 목요일입니다!")
okt.nouns("오늘은 수요일이고요. 내일은 목요일입니다!")
```

| 메서드 | 역할 |
| --- | --- |
| `morphs()` | 형태소 분석 |
| `pos()` | 품사 태깅 |
| `nouns()` | 명사 추출 |

## 7. 코퍼스 Corpus

코퍼스는 분석 대상이 되는 문장, 문서, 데이터셋이다.

코퍼스를 분석하려면 먼저 토큰화, 정제, 정규화가 필요하다.

## 8. 표제어 추출 Lemmatization

표제어 추출은 단어를 사전에 등재된 기본형으로 바꾸는 작업이다.

예시:

```python
from nltk.stem import WordNetLemmatizer

lemmatizer = WordNetLemmatizer()

lemmatizer.lemmatize('dies', 'v')   # die
lemmatizer.lemmatize('has', 'v')    # have
lemmatizer.lemmatize('ate', 'v')    # eat
lemmatizer.lemmatize('better', 'a') # good
```

중요한 점은 품사를 지정해야 더 정확하다는 것이다.

| 옵션 | 의미 |
| --- | --- |
| `v` | 동사 |
| `a` | 형용사 |
| `n` | 명사 |
| `r` | 부사 |

---

어간 추출은 단어에서 핵심 어간만 남기는 작업이다.

규칙 기반이라 빠르지만, 실제 단어가 아닌 결과가 나올 수 있다.

예시:

```python
from nltk.stem import PorterStemmer

stemmer = PorterStemmer()

stemmer.stem('channelize') # channel
stemmer.stem('allowance')  # allow
stemmer.stem('typical')    # typic
```

---

## 9. Lemmatization vs Stemming

| 구분 | Lemmatization | Stemming |
| --- | --- | --- |
| 기준 | 사전 기반 | 규칙 기반 |
| 결과 | 실제 단어일 가능성 높음 | 실제 단어가 아닐 수 있음 |
| 속도 | 느릴 수 있음 | 빠름 |
| 정확도 | 높음 | 낮을 수 있음 |
| 예시 | ate → eat | typical → typic |

## 10. 불용어 Stopword

불용어는 분석에 큰 의미를 주지 않는 단어다.

영어 예시:

```
the, is, a, and, of
```

한국어 예시:

```
은, 는, 이, 가, 을, 를
```

### 영어 불용어 제거

```python
from nltk.corpus import stopwords

sw = stopwords.words('english')

result = []
for w in word:
    if w not in sw:
        result.append(w)
```

### 한국어 불용어 제거

```python
from konlpy.tag import Okt

okt = Okt()

ex = "오늘은 수요일이네요. 여러분은 이번 주말에 무엇을 하실건가요?"
sw = "은 에 을 는 를".split(" ")

token = okt.morphs(ex)
result = [w for w in token if w not in sw]
```

## 11. 정수 인코딩 Integer Encoding

정수 인코딩은 단어를 숫자로 바꾸는 작업이다.

컴퓨터는 텍스트보다 숫자를 더 쉽게 처리하기 때문이다.

### 기본 절차

```
문장 토큰화
→ 단어 토큰화
→ 소문자화
→ 불용어 제거
→ 단어 빈도수 계산
→ 빈도수 기준 정렬
→ 높은 빈도 단어부터 정수 부여
→ OOV 추가
→ 문장을 정수 시퀀스로 변환
```

### 예시

```python
{
    'twinkle': 1,
    'little': 2,
    'star': 3,
    'wonder': 4,
    'circle': 5,
    'OOV': 6
}
```

OOV는 Out-Of-Vocabulary의 약자다.

단어 집합에 없는 단어를 처리하기 위한 특수 토큰이다.

---

## 12. 최종

```
정제 → 정규화 → 토큰화 → 품사 태깅 → 표제어/어간 추출 → 불용어 제거 → 정수 인코딩
```

영어 NLP는 단어 토큰화와 품사 태깅이 중요하다.

한국어 NLP는 형태소 분석과 사용자 정의 불용어 처리가 중요하다.

정수 인코딩은 텍스트를 모델이 처리 가능한 숫자 입력으로 바꾸는 단계다.

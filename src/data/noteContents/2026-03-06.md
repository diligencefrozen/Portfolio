
## 3. 기본 자료형 인식과 형 변환(type casting)

내용

- 프로그래밍 언어들마다 각각의 기본 자료형이 존재한다.
- 아무런 처리를 안 하고 값만 있을 때, 어떤 자료형으로 인식할까?

정리

- 자바의 경우:
    - 정수 → `int`
    - 실수 → `double`
    - 문자열 → `String`
    - 논리 → `boolean`

설명

- `a` 라는 변수에 `3` 을 넣는다고 가정해보면, `a` 를 명시해주기 위해 앞에 자료형을 명시하는 것이다.
- 일부 오류가 출력 시 몇 가지 방법을 사용하면 된다.

### 3-1. 형 변환(type casting)

- 자료형을 바꾸는 행위
- 자바가 자체적으로 자동 형 변환을 해주는 경우도 있다.
- 사용자가 강제로 형 변환을 시도하고 싶을 때 사용
- 형태:

```java
(자료형) 값;
```

대표 예시

- `float`

설명

- 변환 시 오류가 나면 `1.234F` 로 수정
- 혹은 `(float) 1.234;` 로 시도

형태 정리

```java
datatype name = (datatype) value;
```

## 5. 사칙연산 (Scanner + 출력)

주제

- 정수 2개 입력받음 `(x, y)`

```java
Scanner test = new Scanner(System.in);

System.out.printf("무작위로 숫자를 아무거나 입력하시오:");
int x = test.nextInt();

System.out.printf("다시 한번 더 숫자를 아무거나 입력하시오: ");
int y = test.nextInt();

System.out.println(x);
System.out.println(y);
```

사칙연산 결과 저장

```java
int sum01 = x + y;
int sum02 = x - y;
int sum03 = x * y;
int sum04 = x / y;
```

출력

```java
System.out.printf("%s","덧셈 값은?\0");
System.out.printf("%d\n",sum01);

System.out.printf("%s","뺄셈 값은?\0");
System.out.printf("%d\n",sum02);

System.out.printf("%s","곱셈 값은?\0");
System.out.printf("%d\n",sum03);

System.out.printf("%s","나누기 값은?\0");
System.out.printf("%d\n",sum04);

System.out.printf("%d + %d = %d\n",x, y, sum01);
System.out.printf("%d - %d = %d\n",x, y, sum02);
System.out.printf("%d x %d = %d\n",x, y, sum03);
System.out.printf("%d / %d = %d\n",x, y, sum04);
```

설명

- 사칙연산 = 산술 연산자
- 더블을 사용하면 위 방식은 오류를 배출해낸다.
- 따라서 산술 연산자를 사용할 때는 정수형을 사용해야만 한다.

- 산술 연산자는 `double` 에도 사용할 수 있다.
- 다만 **출력 형식 `%d` 는 정수용** 이므로 `double` 값에는 맞지 않는다.
- 즉, 연산이 안 되는 것이 아니라 **자료형과 출력 서식이 맞아야 한다.**

정리

- 정수 / 정수 = 정수 (자바 한정)
- 더블 / 정수 = 더블
- 정수 / 더블 = 더블
- 더블 / 더블 = 더블

실수 나눗셈 예시

```java
System.out.printf("%d / %d = %.2f\n", x, y, (double) x / y);
```

→ 이런 경우 강제 형 변환이 필요

```java

```

- 이유: 컴퓨터 자원을 효율적으로 사용하기 위해

### 12-2. OR


- B가 만족하거나
- 둘 다 만족하면 성립

팁

- 확률이 높은 조건을 앞으로 배치
- 이유: 자원을 덜 소모할 수 있기 때문


- `h >= 190 || a <= 50`
- `((h / 100) > 1.9 || a < 50);`

- 보통 확률이 낮은 걸 맨 뒤로

```java

```

설명

- `a <= 40 && a >= 10`
- `(age > 10 && age < 40);`


설명

- `a >= 10 || h >= 150;`

[정리]

- 문제 조건이 “하나만 만족” 이므로 `||` 이 아니라 `^` 가 핵심

설명

- `a <= 10 || h <= 150`
- `a <= 10 ^ h <= 150`

## 17. 공을 상자에 담는 문제 (삼항 연산자)

문제

- 상자 하나에 축구공 6개 들어감
- 공 개수를 입력받아서 필요한 상자 개수 출력
- 예: 공 49개 입력 → 필요한 상자 9개

입력

```java
System.out.printf("공의 갯수를 입력하세요:");
int ball = test.nextInt();
System.out.println(ball);
```

계산

```java
int ball2 = ball % 6 == 0 ? ball / 6 : ball / 6 + 1;
```

설명

- 공이 6개씩 딱 맞아떨어지면 `ball / 6`
- 나머지가 남으면 상자 하나 더 필요하므로 `ball / 6 + 1`

출력

```java
System.out.println(ball2);
System.out.printf("개 담음");
```

마지막 설명

- 왜 나머지는 생략하고 몫만 사용할까?
- 코드에서 사용된 것은 `%` 가 아니라 `/` 이기 때문이다.

# 암기 포인트

## 1. Scanner + 실수/정수 입력

- `nextDouble()` : 실수 입력
- `nextInt()` : 정수 입력
- `Scanner` 는 참조형

## 2. 원의 면적

- 직접 계산:

```java
circle * circle * 3.14
```

- 더 정확한 방식:

```java
Math.PI * Math.pow(circle, 2)
```

## 3. 정수 나눗셈 / 실수 나눗셈

- `정수 / 정수 = 정수`
- 실수 결과 필요하면 한쪽을 실수로:

```java
sec / 60.0
(double)x / y
```

## 4. 형 변환

- 형 변환 = 자료형 바꾸기
- 형태:

```java
(자료형) 값
```

- 예:

```java
(float)1.234
(double)x / y
```

## 5. 산술 연산자

- `+ - * / %`
- `%` 는 나머지

## 6. 대입 연산자

- `=` 는 같다 아님
- 오른쪽 값을 왼쪽에 넣기

## 7. 복합 대입 / 증감

- `+= -= *= /= %=`
- `++ --`

## 8. 비교 연산자

- `> >= < <= == !=`
- 결과는 `boolean`

## 9. 논리 연산자

- `&&` : and
- `||` : or
- `!` : not
- `^` : xor(하나만 참)

## 10. String 비교

- `==` 로 값 비교하지 않음
- 반드시:

```java
name.equals("홍길동")
```

## 11. 삼항 연산자

- 형태:

```java
조건 ? 참일 때 값 : 거짓일 때 값
```

## 12. 자주 틀리는 포인트

- `printf("\n", plus2);` 는 값 출력 아님
- `double` 도 산술 연산 가능
- `String` 값 비교는 `.equals()`
- 나눗셈에서 실수 결과가 필요하면 강제 형 변환 또는 실수 리터럴 사용
- “하나만 만족” 은 보통 `^`

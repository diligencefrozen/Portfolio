## 숫자형

NUMBER(P, S) : 십진수 기준

P : 정밀도 (Precision) / S : 범위(Scale)

P는 소수점 기준 모든 유효숫자를 의미

만약 P에 명시힌 것보다 큰 숫자값을 입력하면 오류가 발생!

S가 양수면 소수점 이하, 음수면 소수점 이상 (정수부)

의 유효숫자 자릿수를 나타냄

S에 명시한 숫자 이상의 값을 입력하면

S에 명시한 숫자로 반올림 처리를 함

S이 음수면 소수점 기준 왼쪽 자릿수만큼 반올림처리

P가 S보다 크면 S는 소수점 잏 유효숫자 자릿수를 나타냄

(Precision) : 숫자의 전체 자릿 수를 나타냄.소수점 포함 모든 자릿수를 계산

(Scale) : 소수점 위치를 기준으로 유효숫자의 자릿수

양수 : 소수점 이하 자릿수

음수 : 소수점 이상(정수부)의 자릿수

값이 123.54 인데

number(3)

→

정답: 124 : (S쪽은 명시하지 않아서 0도 무방, p가 3이라서 소수점 첫자리에서 반올림)

number (3,2)

- >

정답 : (P가 3이고, 123.54는 유효숫자가 다섯자리여서 오류가 발생,

유효숫자가 p보다 크면 오류가 발생!

number(5,2)

→

123.54

number(7,1)

→

정답 123.5

number(7,-1)

→

정답 120

cast는 데이터타입을 강제로 변환하는 함수

2진수 부동소수점의 오류 !

float(5) 라고 하면, 5자리 10진수가 아닌,

2진수 기준 5비트 수준의 정밀도를 의미함

부동소수점 : 컴퓨터에서 실수를 포현하는 방식

⇒

실수를 정확히 2진수로 표현하기 어려운 문제를 해결하기 위한 방식

(부호) (가수) 2^(지수)의 형식 !!

부호(Sign) : 숫자가 양수 또는 음수인지 나타냄

지수(Exponent) : 숫자의 크기를 결정

가수(Mantissa) : 유효숫자를 저장

FLOAT(5)

이진 정밀도(binary precision) 가 5비트 (10진수)

이진 정밀도를 10진수로 바꾸려면 (사람들은 10진수로 보는 것이 편하기에)

⇒

10진수 자릿수 = 약 (binary precision x log10(2))

⇒

즉 5 x 0.381 = 약 1.5 자리

⇒

이때 이진정밀도로 반올림 => 약 1.5 자리 => 2자리로 반올림

ex)

12345라는 정수를 float(5)에 담고싶은데

12345(10진수) = 11000000111001(2진수) 로 표현됨.

저 이진수를 부동소수점으로 표현하면

⇒

1.1000000111001 x2^13

float자료형의 정밀도에 몇시된 5비트 정밀도로 표현

⇒

1.1000 x 2^13

⇒

12288 (10진수)

⇒

10진수로 변환하는 과정에서 나타낼수 없는 자리값이 2자리까지라

⇒

12000으로 그 결과가 도출됨!

## 제약조건 (Constraint) 이라고 함

데이터 무결성을 보장하기 위해 사용하는데

데이터 무결성 : DB에 저장된 데이터의 일관성과 정확성을 지키는 것

1. 도메인 무결성 제약조건
    
    도메인 제약 (Domain Constraint) 이라고도 하며,
    
    테이블 내의 튜플들이 각 속성의 도메인에 지정된 값만 가져야한다는 조건
    
    데이터 타입 (type), 널(null/ not null)
    
    기본(default), 체크(check) 등을 활용해서 지정
    
2. 개체 무결성 제약 조건
    
    기본키 제약 (Primary key Constaint) 이라고도 함
    
    개체(Entity) : 사람, 장소, 사물, 사건 등과 같이 독립적으로 존재하면서
    
    고유하게 식별이 가능한 단위 ( 자바의 객체와 유사 )
    
    아무리 많은 개체가 있어도,내가 원하는 '유일한' 개체를 발견할 수 있어야한다.
    
    테이블은 기본키를 지정하고, null값을 가져서는 안되며,
    
    테이블 내에 오직 하나의 값만 존재해야한다는 조건
    
3. 참조 무결성 제약조건
    
    외래키 제액 (Foreign Key Constraint)이라고도 하며,
    
    테이블 간의 참조 관계를 선언하는 제약 조건
    
    참조 받는 테이블 (Parent Table)의 값은 기본키(Primary key) 또는 고유키(Unique Key)여야 하며,
    
    참조하는 테이블 (Child Table)의 외래키(Foreign Key가 이에 연결
    
    Parent Table의 값 변경 시, 참조 무결성 제약 조건에 따라 Child Table 의 값이 영향을 받거나 제한됨
    

### 1. NOT NULL ***! : NULL은 값이 비어있다는 뜻

⇒

0이나 공란 또는 띄어쓰기도 아닌 값을 넣지 않은 상태

NULL을 허용하지 않는다

⇒

필수적으로 입력해야할 컬럼에 설정함

⇒

가장 많이 사용되는 제약조건;

테이블을 만들때 가급적 값이 들어가게 했으면 좋겠음 !!

```sql
create table example1(
col1 varchar2(10 char),
col2 varchar2(10 char)
);
```

constraint:  제약 조건명을 따로 부여하는 경우!

```sql
create table example1(
col1 varchar2(10 char),
col2 varchar2(10 char) constrain col2.notnull col2.notnull not null
);
```

### 2. Unique

: 단어 뜻 그대로 해당 컬럼에 들어가는 값이 전체 테이블 내에서 유일해야한다는 조건

Not Null과 함께 사용이 가능하지만, unique는 자주 사용되지는 않음

```sql
create table example3 (
col1 varchar2(10 chat) unique,
col2 varchar2(10 char) unique not null,
col3 varchar3(10 char),
constraint col3_unique unique(col3)
);
```

### 3. Primary Key (기본 키) ***!: NULL 을 허용하지않고, + 중복된 데이터를 허용하지 않음

(NOTNULL + UNIQUE => Primary Key)

데이터의 특정 조건을 검색하거나 수정 등의 작업을 할 떄

기본키로 구분한다

ID, 주민등록번호, 회원 번호, 글 번호 등이 기본키에 해당

일반적으로 테이블에는 1개의 기본키를 무조건 가졌으면 좋겠음 ! ***

```sql
create table example4(
col1 varchar2(10 char) primary key,
col2 varchar2(10 char) not null,
col3 varchar2(10 char) not null
);
```

여러 컬럼을 묶을 때 constraint 키워드를 사용

- 가급적이면 primary key는 하나였으면..!

```sql
create table example5 (
col1 varchar2(10 char),
col2 varchar2(10 char),
col3 varchar2(10 char),
constraint PK_example5 primary key(col1, col2)
);
```

### 4. Foreign key (외래키) ***!

: 다른 테이블의 특정 컬럼을 조회해서

동일한 데이터가 있는 경우에만 입력이 허용됨

참조하는 테이블은 참조받는 테이블의 Primary Key나 Unique로 지정된 컬럼만을

Foreign key 로 지정할 수 있음.

외래키 설정을 위해서는 참조받는 컬럼이 먼저 생성되어 있어야 하고,

외래키를 심을 테이블이 그 이후에 생성이 되어야 함

외래키를 심을 테이블 형식

```sql
create table 테이블명(
컬럼명 자료형;
constraint FK명 foreign key(컬럼명)
references 대상테이블명(대상테이블의 기본키(or 고유키) 컬럼명)
on delete set null || on delete cascade
```

### 5. Check

# DDL 에서 Create 문을 다루는 방식

alter, drop, truncate

데이터 타입 변경 / 컬럼의 크기(용량) 변경

```sql
alter table [테이블명] modify [컬럼명] [데이터타입(용량)];
```

⇒

값이 있는 상태에서 데이터타입 변경하면 에러!!

```sql
alter table coffee modify c_name number(3);
alter table coffee modify c_name varchar2(2 char);
```

⇒

주의사항 : 데이터타입을 변경할 때는 해당 컬럼의 값을 모두 지워야 변경이 가능함 !!

⇒ 그렇지 않으면 오류가 발생!

컬럼의 용량을 줄일 경우 해당 컬럼의 용량을 조회해서 변경할 길이보다

큰 값이 있는지를 확인하고 ⇒ 그 다음에 변경

⇒

```sql
컬럼명 변경
alter table [테이블명] rename column [기존 컬럼명] to [바꿀 컬럼명];
alter table coffee rename column c_name to c_name2;
```

⇒

컬럼 추가

```sql
alter table [테이블명] add [컬럼명] [데이터타입] [제약조건(생략가능)];
```

⇒

기존의 데이터가 없어야 not null 가능 !!

```sql
alter table coffee add c_taste varchar2(20 char) not null;
```

⇒

컬럼 삭제

```sql
alter table [테이블명] drop column [컬럼명];
alter table coffee drop column c_taste;
```

⇒

테이블 내의 내용 지우기

```sql
truncate table [테이블명];
truncate table coffee;
```

⇒

테이블에 있는 데이터를 삭제하지만, 테이블의 구조 자체는 삭제하지 않음.

테이블 삭제 시 (휴지통에 넣기)

```sql
drop table [테이블명] cascade constraint;
drop table coffee cascade constraint;
```

⇒

휴지통에 있는 테이블을 복원

```sql
flashback table [테이블명] to before drop;
flashback table coffee to before drop;
```

⇒

휴지통 비우기

```sql
purge recyclebin;
```

⇒

휴지통에 넣기 + 비우기 => 테이블 완전 삭제 (휴지통에 넣지 않음)

```sql
drop table [테이블명] cascade constraint purge;
drop table coffee cascade constraint purge;
```

⇒

```sql
select * from coffee;
```

DDL : 사실상 신입들한테 맡기지 않음.

경력있는 DBA들이 설계한 테이블을 만들어서 CRUD 정도만 하게끔 할 것.

# 실습 문제

## 1. 노트북에 대한 테이블을 생성

이름, 가격, CPU, RAM(MB 단위, 최대 64 MB가 최대라고 가정)

HDD (GB 단위), 보증 기간을 속성으로 갖게 하고 싶음.

조건에 부합하는 테이블을 생성하는 문장

[하나의 기본키PK를 가져야 하고, 모든 값은 채워넣어야 함]

⇒

```sql
create table lab(
l_name varchar2(10 char) primary key,
l_price number(7) not null,
l_cpu varchar2(20 char) not null,
l_ram number(2) not null check (l_ram between 0 and 64),
l_hdd number(4) not null,
l_year date not null
);
```

⇒

```sql
drop table lab;
select * from lab;
```

⇒

## 2. 노트북의 사양이 좋아져서 128GB 짜리 RAM을 사용하게 되었음.

이 때 RAM 컬럼의 용량을 변경하는 문장 작성

```sql
alter table lab modify l_ram number(3) check (l_ram between 0 and 128);
```

⇒

## 3. 노트북의 크기(인치)에 대한 컬럼을 추가하는 문장 작성

[단위 cm, 소수점 두번째 자리까지 나타낼 것]

[이 컬럼은 빈 값을 넣을 수 없음]

```sql
alter table lab add l_size number(4,2) not null;
```

⇒

## 4. 노트북에는 앞으로 HDD 대신 SSD가 장착됨.

HDD 컬럼이름을 SSD 컬럼명으로 변겅하는 문장 작성

```sql
alter table lab rename column l_hdd to l_sdd;
```

⇒

## 5. 보증 기간 컬럼을 삭제하는 문장 작성

```sql
alter table lab drop column l_year;
```

⇒

## 6. 노트북의 가격이 [800000 ~ 2300000] 사이의 값만 들어갈 수 있게끔

해당 컬럼을 변경하는 문장 작성

```sql
alter table lab modify l_price number(7) check(l_price between 800000 and 2300000);
```

- 800000 ~ 2300000 로 입력해서 오류남, 문법 오류

⇒

## 7. 만든 테이블을 완전히 삭제하는 문장 작성

```sql
drop table lab cascade constraint purge;
```

# 암기 요약

## 1. 테이블 생성 기본형

```sql
create table 테이블명(
	컬럼명 자료형 [옵션],
	컬럼명 자료형,
	...
);
```

- 컬럼명은 보통 **스네이크 표기** 사용
- 옵션 = 제약조건 등 필요할 때만 작성

---

## 2. 용어 정리

- **테이블(Table)**: 데이터를 **표 형태**로 저장하는 공간
- **행(Row)**: 표의 **가로 한 줄**, 데이터 1개
- **열(Column)**: 표의 **세로 한 칸**, 속성 1개
- **스키마(Schema)**: 사용자 소유 객체(테이블, 시퀀스 등) 모음
- **인스턴스(Instance)**: DB 실행 환경
- **도메인(Domain)**: 컬럼에 들어갈 값의 범위/형식

---

## 3. 자료형 핵심

### 문자형

### `char(용량)`

- **무조건 고정 길이 저장**
- 남는 공간은 공백으로 채움
- 처리 속도 빠름
- 예: 성별, 주민번호, 핸드폰번호

### `varchar2(용량)`

- 입력한 만큼 가변 길이 저장
- 공간 효율 좋음
- 예: 이름, 주소

### 핵심 암기

- 한글은 보통 **3byte**
- `varchar2(5)` 는 **5byte**
- `varchar2(5 char)` 는 **5글자**
- 문자열 컬럼은 **`char` 단위 붙이는 습관**
- 문자열 값 입력 시 **작은따옴표 `' '` 사용**

예:

```sql
create table test(
	t_name varchar2(5)
);

create table test2(
	t_name varchar2(5 char)
);

insert into test values('가나다');
insert into test2 values('가나다');
```

---

### 숫자형

## `NUMBER(P, S)`

- **P(정밀도)**: 전체 유효숫자 자리수
- **S(스케일)**: 소수점 기준 자리수
    - 양수: 소수점 이하
    - 음수: 소수점 이상(정수부)

### 암기 포인트

- P보다 큰 숫자 입력 → **오류**
- S보다 많으면 → **반올림**
- `number(3)` → 전체 3자리
- `number(5,2)` → 전체 5자리 중 소수 2자리

예:

```sql
select cast(123.54 as number(3)) from dual;     -- 124
select cast(123.54 as number(3,2)) from dual;   -- 오류
select cast(123.54 as number(5,2)) from dual;   -- 123.54
select cast(123.54 as number(7,1)) from dual;   -- 123.5
select cast(123.54 as number(7,-1)) from dual;  -- 120
```

---

### `FLOAT(P)`

- `NUMBER`의 하위 타입
- **2진수 기준 정밀도**
- 실수 오차 발생 가능
- Oracle에서는 보통 **`NUMBER`만 써도 충분**

---

### 날짜형

- **DATE**: 연, 월, 일, 시, 분, 초
- **TIMESTAMP**: 연, 월, 일, 시, 분, 초, 밀리초
- 실무/학습에서는 **DATE 중심**

## 4. 제약조건 핵심

### 1) `NOT NULL`

- 값 비우면 안 됨
- **필수 입력 컬럼**
- 가장 많이 사용

예:

```sql
create table example1(
	col1 varchar2(10 char),
	col2 varchar2(10 char)
);
```

---

### 2) `UNIQUE`

- 해당 컬럼 값이 **유일해야 함**
- 중복 불가
- null은 허용 가능(단독으로는)

예:

```sql
create table example3 (
	col1 varchar2(10 char) unique,
	col2 varchar2(10 char) unique not null,
	col3 varchar2(10 char),
	constraint col3_unique unique(col3)
);
```

---

### 3) `PRIMARY KEY`

- **NOT NULL + UNIQUE**
- 중복 불가, null 불가
- 데이터를 구분하는 대표 키
- 보통 테이블마다 **1개는 두는 게 좋음**

예:

```sql
create table example4(
	col1 varchar2(10 char) primary key,
	col2 varchar2(10 char) not null,
	col3 varchar2(10 char) not null
);
```

복합키:

```sql
create table example5 (
	col1 varchar2(10 char),
	col2 varchar2(10 char),
	col3 varchar2(10 char),
	constraint PK_example5 primary key(col1, col2)
);
```

---

### 4) `FOREIGN KEY`

- 다른 테이블의 **PK / UNIQUE** 값을 참조
- 참조 대상 값이 있어야만 입력 가능
- 부모 테이블 먼저 생성해야 함
- 옵션:
    - `on delete set null`
    - `on delete cascade`

예:

```sql
create table example6(
	ex6_id varchar2(10 char) primary key
);

create table example7(
	ex7_id varchar2(10 char),
	constraint fk_ex7 foreign key(ex7_id)
	references example6(ex6_id)
	on delete cascade
);
```

---

### 5) `CHECK`

- 값의 범위/조건 제한

```sql
create table example8 (
	col1 number(3)
	constraint ex8_check check (col1 between 1 and 9)
);
```

---

### 6) `DEFAULT`

- 값 안 넣으면 자동 입력

```sql
create table example9(
	col1 number(4) default 999
);
```

---

## 5. DUAL / CAST

### `DUAL`

- Oracle 제공 **더미 테이블**
- 함수/계산 연습용

### `CAST`

- 데이터 타입 강제 변환

```sql
select cast(123.54 as number(5,2)) from dual;
```

---

## 6. DDL 핵심

### `ALTER`

구조 수정

### 컬럼 타입/크기 변경

```sql
alter table 테이블명 modify 컬럼명 데이터타입(용량);
```

### 컬럼명 변경

```sql
alter table 테이블명 rename column 기존컬럼명 to 새컬럼명;
```

### 컬럼 추가

```sql
alter table 테이블명 add 컬럼명 데이터타입 제약조건;
```

### 컬럼 삭제

```sql
alter table 테이블명 drop column 컬럼명;
```

### 주의

- 값 있는 상태에서 타입 변경하면 오류 가능
- 용량 줄일 때 기존 데이터 길이 확인
- 기존 데이터가 있으면 `not null` 컬럼 추가가 어려울 수 있음

## 7. DROP / TRUNCATE / FLASHBACK

### `truncate table`

- **데이터만 삭제**
- 구조는 남김

### `drop table`

- **테이블 자체 삭제**

### `flashback table ... to before drop`

- 휴지통에 있는 테이블 복원

### `purge recyclebin`

- 휴지통 비우기

### 완전 삭제

```sql
drop table 테이블명 cascade constraint purge;
```

---

## 8. 차이점

### `char` vs `varchar2`

- `char` = 고정길이
- `varchar2` = 가변길이

### `not null` vs `unique` vs `primary key`

- `not null` = 비워둘 수 없음
- `unique` = 중복 불가
- `primary key` = 비울 수 없고 중복도 불가

### `delete` vs `truncate` vs `drop`

- `delete` = 행 삭제
- `truncate` = 데이터만 한꺼번에 삭제
- `drop` = 테이블 구조까지 삭제

---

## 9. 외래키 연쇄 삭제 암기

- 부모 테이블 값 삭제
- 자식 테이블이 `on delete cascade` 면
- 연결된 자식 행도 같이 삭제

예:

```sql
delete from major where m_code = 1001;
```

# 암기

## 자료형

- 문자: `char`, `varchar2`
- 숫자: `number(p,s)`, `float`
- 날짜: `date`, `timestamp`

## 제약조건

- `not null` = null 금지
- `unique` = 중복 금지
- `primary key` = null 금지 + 중복 금지
- `foreign key` = 다른 테이블 참조
- `check` = 범위 제한
- `default` = 기본값

## DDL

- 생성: `create`
- 수정: `alter`
- 데이터만 비움: `truncate`
- 삭제: `drop`
- 복구: `flashback`
- 완전삭제: `purge`

## 필수 암기

- 문자열은 `'값'`
- 한글 저장 시 `varchar2(n char)` 권장
- `dual` = 연습용 테이블
- `cast` = 형변환
- `on delete cascade` = 부모 삭제 시 자식도 삭제

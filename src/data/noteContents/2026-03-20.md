```java
				}
				break;

			case 6:
				if (accumulatedLoan > 0) {
					System.out.println("돈을 갚아야합니다");
					break;
				}
				System.out.println("게임 종료!");
				break a;

			default:
				System.out.println("잘못 입력하셨습니다:");
				break;
			}

		}
	}
}
```

## 3. 절차지향 프로그래밍 / 객체지향 프로그래밍 개념

### 절차지향 프로그래밍 (Procedural Programming)

- 순서대로 잘 프로그래밍해서 결과를 잘 내자는 것을 목표로 한다.
- 변수 하나라도 덜 쓰고, 효율적으로 코드를 짰었다.
- 저사양 PC 환경이었기 때문이다.
- 조건문, 반복문 등을 잘 다룰 수 있어야 한다.

### 장점

- 처리 속도
- 실행 속도가 빠르다

---

### 객체지향 프로그래밍 (Object-Oriented Programming)

- 1960년 이후 등장
- 본격적으로 확산된 것은 1980년대 이후
- Java가 1990년대에 등장한 이후 본격적으로 사용되기 시작했다.
- 물론 그 이전에도 C로 활용은 했었다.

### 객체지향 프로그래밍의 목적

- 실생활을 표현하고 묘사하자
- 우리가 살고 있는 현실을 모델링하자

### 특징 / 장점

- 코드 보기가 더 편하다
- 코드의 재사용성
- 유지보수에 용이
- 대규모 프로젝트에 적합

### 시대 변화

- 알고리즘의 시대 → 유지보수의 시대로 변화

## 4. 객체(Object)

### 객체란?

- 실생활에 존재하는 모든 것
- 자동차, 노트북, 책, ATV, 에어컨 등등

### 객체 = 인스턴스(Instance)

- class에서 만들어진 객체를 해당 클래스의 인스턴스라고 한다.

### 예시

### 사람

- 속성: 이름, 나이, 사는 곳, ...
- 동작: 웃다, 울다, 걷다, ...

### 에어컨

- 속성: 제조사, 온도, 바람, 풍향, ...
- 동작: 켜다, 끄다

## 9. Function / Method

### Function(함수)

- Class 밖에 만든 것 → Java에서는 이런 표현을 쓰지 않는다.
- 사실상 Java에는 `function`이라는 용어가 거의 없다.

### Method(메소드)

- Class 속에서 만든 일종의 function
- Class에 종속적인 함수라고 표현할 수 있다.
- 여태까지 써온 것은 다 method였다.

예:

```java
b.go();
```

- method(멤버 메소드)는 액션이다.
- 프로그램상 필요한 기능이다.
- 객체의 동작에 해당하는 실행 블록이다.

예:

```java
public void go() {
	System.out.println("부릉부릉 ~");
}
```

## 10. 멤버 변수(Member Variable)

- 속성
- attribute

### 전역 변수와의 구분

- 어떤 사람들은 멤버변수를 전역변수라고도 한다.
- 하지만 Java에는 전역변수라는 개념은 없다.
- 전역변수는 C언어에서 유래된 개념이다.
- 함수의 외부에서 선언된 변수이다.
- 그런데 Java에는 function이라는 개념을 쓰지 않으므로, 전역변수라고 보기 어렵다.

### 정리

- 객체의 데이터가 저장되는 공간
- Class에 포함되는 변수(Variable)

## 1. 절차지향 프로그래밍

- 순서대로 처리해서 결과를 내는 방식
- 조건문, 반복문이 중요
- 장점: 처리 속도, 실행 속도가 빠름

## 2. 객체지향 프로그래밍

- 실생활을 표현하고 묘사하는 방식
- 현실을 모델링하는 것이 목적
- 장점: 재사용성, 유지보수 용이, 대규모 프로젝트에 적합

## 3. 객체(Object)

- 실생활에 존재하는 모든 것
- 클래스에서 만들어진 객체를 인스턴스라고 함

## 4. 클래스(Class)

- 객체를 만들기 위한 틀
- 붕어빵 틀 비유로 이해

## 5. 인스턴스(Instance)

- 클래스로부터 만들어진 객체
- 예: `Bus b;` 에서 `b`는 인스턴스

## 6. 멤버 변수(Member Variable)

- 객체의 속성
- attribute, field라고도 부름
- 클래스 안에 들어 있는 변수

## 7. 메소드(Method)

- 클래스 안에 정의된 기능
- 객체의 동작을 담당
- Java에서는 function보다 method라는 표현을 사용

## 8. this

- 자기 자신 객체를 가리킴

## 9. 참조형

- 기본형이 아닌 자료형은 참조형
- 객체 변수는 주소값(참조)을 공유할 수 있음
- `Phone p2 = p;` 는 복사가 아니라 같은 객체를 같이 가리키는 것

## 10. 한 줄 요약

- 절차지향 = **순서 중심**
- 객체지향 = **현실 세계 모델링 중심**

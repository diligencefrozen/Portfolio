## 2. 용어 정리

- **테이블(Table)**: 데이터를 **표 형태(엑셀 표 같은 형태)** 로 저장하는 공간
- **행(Row)**: 데이터 **가로 한 줄** / 한 개의 정보 묶음
    
    예: **`1, 초코파이, 오리온, 5000`**
    
- **열(Column)**: 데이터 **세로 한 칸의 항목**
    
    예: **`과자이름`**, **`회사`**, **`가격`**
    
- **스키마(Schema)**: 사용자 소유의 DB 객체(테이블, 시퀀스 등) 모음
- **인스턴스(Instance)**: DB가 실행될 때 동작하는 메모리와 프로세스 환경
- **도메인(Domain)**: 컬럼에 들어갈 수 있는 값의 범위, 자료형, 조건
    
    예: 가격 컬럼에는 숫자만, 이름 컬럼에는 문자만
    

---

## 3. 자료형

- 문자형: `varchar2`
- 숫자형: `number`
- 날짜형: `date`

### 주의

- `varchar2(10 char)` → **글자 수 기준**
- `number(4)` → **자리수/범위 주의**
- `date` → 날짜 + 시간 저장 가능

---

## 4. 제약조건

- `not null` : 반드시 값 입력
- `primary key` : 중복 불가 + null 불가
- `foreign key` : 다른 테이블 PK 참조

## 5. 과자 테이블과 PK 고민

과자 테이블에 넣고 싶은 정보:

- 과자 이름
- 회사
- 가격

PK를 어디에 둘지 고민:

- 과자이름: `초코파이`는 오리온/롯데 둘 다 가능 → 단일값 아님
- 회사이름: 한 회사가 여러 과자 생산 → 중복 가능
- 가격: 같은 가격 가능

### 결론

자연키가 애매하므로 **번호값(s_no)** 을 따로 만들어 PK로 사용

## 7. INSERT 기본

```sql
insert into 테이블명(컬럼명, 컬럼명, ...) values(값, 값, ...);
insert into snack(s_no, s_name, s_compa, s_price)
	values(1, '초코파이', '오리온', 5000);
```

### 컬럼 순서 바꾸기

```sql
insert into snack(s_name, s_no, s_compa, s_price)
	values('새콤달콤', 500, 2, '롯데');
```

### 주의

컬럼 순서를 바꾸면 **값 자료형도 그 순서에 맞아야 함**

위 예시는 `s_compa` 자리에 `2`, `s_price` 자리에 `'롯데'`가 들어가므로 오류

올바른 예:

```sql
insert into snack(s_name, s_no, s_compa, s_price)
	values('새콤달콤', 500, '롯데', 2000);
```

## 11. 번호 자동 증가 필요성

데이터가 많아지면 번호를 직접 기억하기 어려움

→ 자동으로 번호 생성 필요

### DB별 방식

- MySQL: `AUTO_INCREMENT`
- Oracle: `SEQUENCE`

---

## 12. Sequence

번호를 자동 생성하는 객체

### 특징

- 테이블과 무관
- `insert` 실패해도 번호 증가
- 증가한 값은 다시 내릴 수 없음

---

## 13. Sequence 생성

### 기본

```sql
create sequence 시퀀스명;
```

```sql
create sequence snack_seq;
```

### 관례

테이블명 뒤에 `_seq` 붙임

### 옵션

```sql
create sequence 시퀀스명
	increment by 1
	start with 1
	minvalue 1
	maxvalue 999
	nocycle / cycle
	nocache / cache 크기
	noorder / order;
```

### 의미

- `increment by 1` : 1씩 증가
- `start with 1` : 시작값
- `minvalue 1` : 최소값
- `maxvalue 999` : 최대값
- `nocycle / cycle` : 반복 여부
- `nocache / cache` : 미리 메모리에 생성 여부
- `noorder / order` : 요청 순서 보장 여부

## 15. Sequence 활용

```sql
drop table snack cascade constraint purge;

create table snack(
	s_no number(4) primary key,
	s_name varchar2(10 char) not null,
	s_compa varchar2(10 char) not null,
	s_price number(4) not null
);

insert int snack values(snack_seq.nextval; '초코파이', '롯데', 5000);
insert int snack values(snack_seq.nextval; '초코파이', '오리온', 5000);
insert int snack values(snack_seq.nextval; '엄마손파이', '노브랜드', 3000);
insert int snack values(snack_seq.nextval; '빼배로', '롯데', 2000);
insert int snack values(snack_seq.nextval; '포키', '롯데', 2000);
insert int snack values(snack_seq.nextval; '포키(실패)', '롯데', 99999);
select * from snack;
```

### 보완된 형태

```sql
insert into snack values(snack_seq.nextval, '초코파이', '롯데', 5000);
insert into snack values(snack_seq.nextval, '초코파이', '오리온', 5000);
insert into snack values(snack_seq.nextval, '엄마손파이', '노브랜드', 3000);
insert into snack values(snack_seq.nextval, '빼빼로', '롯데', 2000);
insert into snack values(snack_seq.nextval, '포키', '롯데', 2000);
insert into snack values(snack_seq.nextval, '포키(실패)', '롯데', 99999);
select * from snack;
```

### 핵심

- `insert int` → `insert into`
- `;` → `,`
- `99999`는 `number(4)` 범위 초과 가능
- 실패해도 sequence 값은 증가

## 16. date 컬럼 추가

```sql

drop sequence snack_seq;

create table snack(
	s_no number(4) primary key,
	s_name varchar2(10 char) not null,
	s_compa varchar2(10 char) not null,
	s_price number(4) not null,
	s_year date not null

select * from snack;

select sysdate from dual ;
```

## 23. 내장 함수

### 숫자 함수

- `ABS(숫자)` : 절대값
- `ROUND(숫자, m)` : 반올림
- `POWER(숫자, n)` : n제곱
- `TRUNC(숫자, m)` : 버림
- `MOD(m,n)` : 나머지
- `SQRT(숫자)` : 제곱근

예:

```sql
ABS(-4.5)
ROUND(5.36, 1)
POWER(2,3)
TRUNC(123.456,2)
TRUNC(123.456,-1)
TRUNC(123.456,0)
MOD(12,10)
SQRT(9.0)
```

### 문자 함수

- `CONCAT(s1, s2)` : 문자열 연결
- `SUBSTR(s1, n, k)` : 부분 문자열
- `INSTR(s1,s2,n,k)` : 위치 찾기
- `LPAD(s,n,c)` : 왼쪽 채우기
- `RPAD(s,n,c)` : 오른쪽 채우기
- `LTRIM(s1,s2)` : 왼쪽 제거
- `RTRIM(s1,s2)` : 오른쪽 제거
- `REPLACE(s1,s2,s3)` : 문자열 치환
- `LENGTH(s1)` : 글자 수
- `LENGTHB(s1)` : 바이트 수

예:

```sql
CONCAT('ABCD', 'EFG')
SUBSTR('ABCDEFG', 3, 4)
INSTR('Hello', 'l', 1, 2)
LPAD('ABC', 5, '*')
RPAD('ABC', 5, '*')
LTRIM('***ABC', '*')
RTRIM('ABC***', '*')
REPLACE('JACK and JUE', 'J', 'BL')
LENGTH('ㅋㅋㅋ')
LENGTHB('ㅋㅋㅋ')
```

### 날짜/시간 함수

- `SYSDATE` : 현재 날짜/시간
- `TO_DATE(s1, 패턴)` : 문자 → 날짜
- `TO_CHAR(date, 패턴)` : 날짜 → 문자
- `ADD_MONTHS(date, 숫자)` : 월 더하기
- `LAST_DAY(date)` : 해당 월 마지막 날

패턴:

- `YYYY`
- `YYY`, `YY`, `Y`
- `MM`
- `DAY`
- `DY`
- `HH`, `HH12`
- `AM` or `PM`
- `HH24`
- `MI`
- `SS`

예:

```sql
select to_date('2026-04-04 오전 04:00', 'YYYY-MM-DD PM HH:MI') from dual;
select to_char(sysdate, 'YYYY-MM-DD') from dual;
select add_months(sysdate, -3) from dual;
select last_day(sysdate) from dual;
```

### 보완

- `ADD_DAYS(date, 숫자)` 는 Oracle에 없음
- 날짜 ± 숫자 로 계산

예:

```sql
select sysdate + 10 from dual;
select sysdate - 7 from dual;
```

- `ADD_YEARS()` 없음 → `add_months(date, 12)` 사용

```sql
select to_char(last_day(add_months(sysdate, -3)), 'MM"월" DD"일"') from dual;
select to_char(last_day(add_months(sysdate, -3)), 'MM월 DD일') from dual;
```

---

## 25. 보완점

- Oracle은 `varchar`보다 `varchar2`
- `s.no` 보다 `s_no`
- `insert int` → `insert into`
- `drop sequence 시퀀스명.nextval` 틀림
- `drop sequence snack_seq;` 맞음
- `number(4)` 에 `99999` 넣기 어려움
- sequence는 실패해도 증가
- 컬럼 순서 바꾸면 값 자료형도 맞춰야 함

---

# 요약

- Oracle 기본 자료형: `varchar2`, `number`, `date`
- 필수값: `not null`
- 기본키: `primary key`
- 자연키 애매하면 번호 컬럼 `s_no` 사용
- 자동 번호: `sequence`
- 현재 날짜: `sysdate`
- 문자→날짜: `to_date`
- 날짜→문자: `to_char`
- 전체조회: `select *`
- 중복제거: `distinct`
- 문자열 연결: `||`
- 연습용 더미 테이블: `dual`

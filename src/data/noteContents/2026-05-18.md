# 6. Spring 등장 배경

Spring 이전에는 EJB를 많이 사용했다.

하지만 EJB는 기능이 많아질수록 복잡하고 무거웠다.

개발자들은 이 시기를 “겨울”에 비유했고,

그 겨울이 끝나고 “봄”이 왔다는 의미로 Spring이라는 이름이 붙었다.

# 7. Spring 핵심 개념

## POJO

```
POJO = Plain Old Java Object
```

특정 기술에 심하게 의존하지 않는 순수 Java 객체를 말한다.

즉,

```
Java의 기본으로 돌아가자.
```

---

## Framework

Spring은 웹 개발에 필요한 기본 뼈대를 제공한다.

```
Framework = 정해진 틀 안에서 개발하도록 도와주는 구조
```

## Library

```
Library = 필요할 때 직접 호출해서 쓰는 도구
```

| 구분 | Framework | Library |
| --- | --- | --- |
| 제어 흐름 | Framework가 주도 | 개발자가 주도 |
| 느낌 | 틀 | 도구 |
| 예시 | Spring | json-simple.jar |

# 11. ViewResolver

Controller에서

```java
return "home";
```

이라고 하면 ViewResolver가 아래 경로로 바꿔준다.

```
/WEB-INF/views/home.jsp
```

설정 예시

```xml
<beans:bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <beans:property name="prefix" value="/WEB-INF/views/" />
    <beans:property name="suffix" value=".jsp" />
</beans:bean>
```

---

# 12. Annotation

```
@ = Annotation
```

코드에 붙여서 Spring이나 개발 도구에게 특정 기능을 알려주는 표시이다.

| Annotation | 의미 |
| --- | --- |
| `@Bean` | 객체를 Bean으로 등록 |
| `@Autowired` | Bean 자동 주입 |
| `@Controller` | 화면을 반환하는 Controller |
| `@RestController` | 데이터를 반환하는 Controller |
| `@ResponseBody` | 응답 Body에 데이터 직접 반환 |
| `@Service` | 비즈니스 로직 처리 |
| `@RequestMapping` | URL과 method 연결 |
| `@RequestParam` | 요청 파라미터 받기 |

---

# 13. @RequestMapping

```java
@RequestMapping(value = "/", method = RequestMethod.GET)
```

의미

```
주소가 "/"이고
요청 방식이 GET이면
해당 method가 처리한다.
```

HTTP 요청 방식

| 방식 | 의미 |
| --- | --- |
| GET | 조회 |
| POST | 등록 |
| PUT | 전체 수정 |
| PATCH | 일부 수정 |
| DELETE | 삭제 |

# 14. servlet-context.xml 핵심

```xml
<annotation-driven />

<resources mapping="/resources/**" location="/resources/" />

<beans:bean class="org.springframework.web.servlet.view.InternalResourceViewResolver">
    <beans:property name="prefix" value="/WEB-INF/views/" />
    <beans:property name="suffix" value=".jsp" />
</beans:bean>

<context:component-scan base-package="com.beaver.may182" />
```

## 설명

```
annotation-driven = Annotation 기능 활성화
resources = CSS, JS, 이미지 같은 정적 파일 처리
ViewResolver = return 값을 JSP 경로로 변환
component-scan = Annotation 붙은 클래스를 자동 탐색
```

# 15. Spring IoC / DI

## 기존 방식

```java

```

---

## Spring 방식

```java
AbstractApplicationContext aac =
    new ClassPathXmlApplicationContext("may182beans.xml");

Dog d = aac.getBean("d", Dog.class);
```

Spring이 객체를 만들고,

개발자는 꺼내서 사용한다.

---

# 16. IoC

```
IoC = Inversion of Control = 제어의 역전
```

객체 생성과 관리의 제어권이 개발자에서 Spring으로 넘어가는 것.

```
기존: 내가 직접 객체 생성
Spring: Spring이 객체 생성, 나는 가져다 씀
```

---

# 17. DI

```
DI = Dependency Injection = 의존성 주입
```

필요한 객체나 값을 외부에서 넣어주는 방식이다.

```xml
<bean id="d2" class="com.beaver.may182.dog.Dog">
    <property name="name" value="멍멍이" />
    <property name="age" value="3" />
</bean>
```

Java에서는 가져다 쓴다.

```java
Dog d2 = aac.getBean("d2", Dog.class);
```

정리

```
IoC = 제어권을 Spring에게 넘기는 개념
DI = IoC를 구현하는 대표적인 방법
```

---

# 18. Bean 등록 방식

## 기본 Bean

```xml
<bean id="d" class="com.beaver.may182.dog.Dog"></bean>
```

## setter 주입

```xml
<bean id="d2" class="com.beaver.may182.dog.Dog">
    <property name="name" value="멍멍이"></property>
    <property name="age" value="3"></property>
</bean>
```

⇒ setter가 있어야 한다.

## 생성자 주입

```xml
<bean id="d3" class="com.beaver.may182.dog.Dog">
    <constructor-arg value="멍멍이님"></constructor-arg>
    <constructor-arg value="7"></constructor-arg>
</bean>
```

⇒ 해당 생성자가 있어야 한다.

```java

```

## close

```java

```

# 20. Dog 예제

## Dog.java

```java
public class Dog {
    private String name;
    private int age;

    public Dog() {}

    public Dog(String name, int age) {
        this.name = name;
        this.age = age;
    }

    public String getName() { return name; }
    public void setName(String name) { this.name = name; }

    public int getAge() { return age; }
    public void setAge(int age) { this.age = age; }
}
```

## may182beans.xml

```xml
<bean id="d" class="com.beaver.may182.dog.Dog"></bean>

<bean id="d2" class="com.beaver.may182.dog.Dog">
    <property name="name" value="멍멍이"></property>
    <property name="age" value="3"></property>
</bean>

<bean id="d3" class="com.beaver.may182.dog.Dog">
    <constructor-arg value="멍멍이님"></constructor-arg>
    <constructor-arg value="7"></constructor-arg>
</bean>
```

| Bean id | 방식 |
| --- | --- |
| `d` | 기본 생성자 |
| `d2` | setter 주입 |
| `d3` | 생성자 주입 |

# 24. 헷갈리는 부분

## id와 class

```xml
<bean id="d2" class="com.beaver.may182.dog.Dog">
```

| 속성 | 의미 |
| --- | --- |
| `id` | Bean 이름 |
| `class` | 실제 Java 클래스 경로 |

---

## property

```xml
<property name="name" value="멍멍이"></property>
```

내부적으로 아래 setter를 호출하는 느낌이다.

```java
setName("멍멍이");
```

---

## constructor-arg

```xml
<constructor-arg value="멍멍이님"></constructor-arg>
<constructor-arg value="7"></constructor-arg>
```

아래 코드와 비슷하다.

```java
new Dog("멍멍이님", 7);
```

---

## getBean

```java
Dog d = aac.getBean("d", Dog.class);
```

의미

```
id가 d인 Bean을 Dog 타입으로 꺼내라.
```

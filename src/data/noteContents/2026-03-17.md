## [코드 5] for-each 문 예제

```java
public class PMain3 {
	public static void test() {
	}

	public static void main(String[] args) {
		String[] ss = { "ㅋㅋ", "ㅎㅎ", "ㅇㅇ", "^__^" };

		// 요소들 전부 다 출력
		System.out.println(ss[0]);
		System.out.println(ss[1]);
		System.out.println(ss[2]);
		System.out.println(ss[3]);
		System.out.println(ss[0] + ss[1] + ss[2] + ss[3]);

		for (int i = 0; i < ss.length; i++) {
			System.out.println(ss[i]);
		}

		for (String s2 : ss) {
			// 배열 ss의 요소를 s2라는 변수에다 담음.
			System.out.println(s2);
			// 출력 (배열의 요소 순서대로)
		}

		for (int i = 0; i < ss.length; i++) {
			ss[i] = "ASDF";
			// normal for 문은 값을 변경 가능.
		}

		for (int i = 0; i < ss.length; i++) {
			System.out.println(ss[i]);
		}

		for (String s2 : ss) {
			s2 = "ZXCV";
			System.out.println(s2);
			// 원본에는 영향을 못줌.
		}

		for (String s3 : ss) {
			System.out.println(s3);
			// 여기서 ss는 복사본 개념이다
		}
	}
}
```

## [코드 6] Call by Value / Call by Reference 설명 코드

```java
// Call by Value, Call By Reference
public class PMain4 {
	public static void test(int a, int[] b, int[] c) {
		System.out.println("b(green) " + b);   // {10,20}(int[] b)의 주소값(번지값)
		System.out.println("c(green) " + c);   // {10,20}(int[] c)의 주소값(번지값)
		System.out.println(a + "a(green)");    // 10
		System.out.println(b[0] + "b(green)"); // 10
		System.out.println(c[0] + "c(green)"); // 10
		System.out.println("============");

		a = 100;        // a는 변경됨
		b[0] = 100;     // b도 변경됨 -> main 쪽에 영향을 줌!
		c = new int[] { 100, 200 };
		// c도 변경됨 -> 새로운 배열 생성

		// 값 변경 = 각각의 요소에 접근을 해야함!
		// new int[] 꼭 써줘야!
		// c = {100, 200};  ← 에러 발생
		// 배열 선언과 동시에 초기화를 할 때에만 사용할 수 있는 축약 문법임!

		System.out.println("b(green) " + b);
		System.out.println("c(green) " + c);
		System.out.println(a + "a(green)");    // 100
		System.out.println(b[0] + "b(green)"); // 100
		System.out.println(c[0] + "c(green)"); // 100
		System.out.println("============");
	}

	public static void main(String[] args) {
		int a = 10;
		int[] b = { 10, 20 };
		int[] c = { 10, 20 };

		System.out.println("b(red) " + b); // {10,20}(int[] b)의 주소값(번지값)
		System.out.println("c(red) " + c); // {10,20}(int[] c)의 주소값(번지값)
		System.out.println("======");

		test(a, b, c);

		System.out.println("======");
		System.out.println(a + "a(red)");    // 10
		System.out.println(b[0] + "b(red)"); // 100 -> 값 변경에 영향을 받음.
		System.out.println(c[0] + "c(red)"); // 10 -> 값 변경에 영향을 받지 않음.

		// test 함수의 변경된 c와는 다르기 때문이다.
		// 자바에서 파라미터를 넘기는 과정에 직접적인 참조를 넘겨주는게 아닌,
		// 주소 값을 복사해서 넘기기 때문에 이는 Call By Value다!
	}
```

```java
	public static void printSum(int[] sum) {
		System.out.println("입력한 데이터");

		}

		System.out.println();
		System.out.println("######");
	}

	public static int[] result(int[] sum) {
		int b = 0;
		// 값을 옮겨담기 위한 변수

		for (int turn = 1; turn < sum.length; turn++) {

				// num.length - 1

				// length 길이 그대로 사용하면 에러가 발생함.
				if (sum[i] > sum[i + 1]) {
					b = sum[i];
					sum[i] = sum[i + 1];
					sum[i + 1] = b;
				}

				System.out.println(sum[i] + " ");
			}

			System.out.println(sum[4]);
			System.out.println("정렬된 값:" + b);

			b = 0; // 무조건 4턴을 다 보여주는데
			// 4턴 이전에 끝났을 때 정렬된 값 초기화

			System.out.println("######");
		}

		return sum;
	}

	public static void printResult(int[] result) {
		/*
		System.out.println(sum[0]);
		System.out.println(sum[1]);
		System.out.println(sum[2]);
		System.out.println(sum[3]);
		System.out.println(sum[4]);
		*/

		System.out.println("정렬 후 값:");

		for (int i : result) {
			System.out.println(i + " ");
		}

		System.out.println();
		System.out.println("######");
	}

	public static void main(String[] args) {
		// Bubble sort : 인접한 두 항목의 값을 비교해서 기준에 만족하면 서로의 값을 교환하는 정렬 방법
		// 정수를 입력받아서 배열에 담을 것 (숫자는 다섯개, 중복 x)
		// 정렬 전의 값을 출력

		// 코딩테스트 준비 용이라면, 선택정렬, 퀵정렬, 삽입정렬, 합병정렬, ...

		System.out.println("######");
```

- 자바 프로그램 실행 시 가장 먼저 실행됨

### 같이 암기할 것

- `public` : 어디서든 접근 가능
- `static` : 객체 생성 없이 실행 가능
- `void` : 반환값 없음
- `String[] args` : 외부에서 전달되는 문자열 배열

## 37) Call by Value

- 값을 복사해서 전달하는 방식

### 자바 핵심

- 자바는 **기본적으로 Call by Value**

### 코드에서 확인되는 점

- `a = 100;` 해도 main의 `a`는 안 바뀜
- `b[0] = 100;` 하면 배열 내부 값이 바뀌어 main에도 영향이 감
- `c = new int[] {100, 200};` 는 새로운 배열을 가리키게 하는 것이므로 main의 `c`와는 별개

---

## 38) Bubble Sort

- 인접한 두 값을 비교해서 순서를 바꾸는 정렬 방법

### 핵심 구조

```java
if (sum[i] > sum[i + 1]) {
    b = sum[i];
    sum[i] = sum[i + 1];
    sum[i + 1] = b;
}
```

### 특징

- 구현은 쉬움
- 성능은 좋지 않음

---

## 39) `Arrays.sort()`

- 자바가 제공하는 정렬 메서드
- 직접 버블정렬 구현하는 것보다 편하고 효율적

예:

```java
Arrays.sort(sum);
```

## 40) 임시 변수

- 값 교환 시 잠깐 저장하는 변수

예:

```java
int b = 0;
```

### 역할

- swap(교환)할 때 한쪽 값을 잠시 보관

# 3. 암기용

### 입력

- `Scanner` : 입력 클래스
- `next()` : 문자열 입력
- `nextInt()` : 정수 입력

### 출력

- `println()` : 출력 후 줄바꿈
- `print()` : 줄바꿈 없음
- `printf()` : 형식 지정 출력

### 제어문

- `if` : 조건 분기
- `switch` : 값 분기
- `while(true)` : 무한 반복
- `break` : 탈출
- `break m` : 라벨 반복문 탈출

### 배열

- 같은 타입 묶음
- 인덱스 0부터
- `.length` : 길이
- 2차원 배열 : 배열 안에 배열

### 메서드

- 기능 단위 코드
- `void` : 반환 없음
- `return` : 값 반환
- 오버로딩 : 이름 같고 매개변수 다름
- 가변인자 : `int... nx`

### 연산자

- `=` : 대입
- `+=` : 더해서 저장
- `=` : 빼서 저장
- `==` : 같다
- `%` : 나머지

### 중요한 개념

- 자바는 Call by Value
- 배열은 참조형
- Bubble Sort는 인접 비교 정렬
- `Arrays.sort()`가 더 효율적

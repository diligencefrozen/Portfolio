# 1. SELECT 기본 구조

```sql
select [distinct] [컬럼명], [컬럼명 as 별칭 또는 컬럼명 별칭], [연산자], [내장함수], ...
from [테이블명]
where [조건식]
group by [그룹 대상]
having [그룹 결과 조건]
order by [정렬대상 asc/desc];
```

## 실행 순서 개념

```sql
from
where
group by
having
select
order by
```

즉,

- `from` : 테이블 선택
- `where` : 조건에 맞는 행 선택
- `group by` : 그룹으로 묶기
- `having` : 그룹 결과에 조건
- `select` : 출력할 컬럼 선택
- `order by` : 정렬

# 2. WHERE절 (조건식)

전체 데이터 중 **원하는 조건의 데이터만 조회**할 때 사용한다.

## 비교 연산자

```sql
=, !=, ^=, <>, >, <, >=, <=
```

`!=`, `^=`, `<>` 는 모두 **같지 않다** 의미로 사용 가능하다.

---

## 가격이 3000원인 과자의 모든 정보 조회

```sql
select * from snack where s_price = 3000;
```

## 모든 과자 이름, 가격 조회

```sql
select s_name, s_price from snack;
```

## 모든 과자 이름(가격) 조회

예: `콘칩(2000원)`

```sql
select s_name || '(' || s_price || '원)' "이름(가격)"
from snack;
```

- `||` : 문자열 연결
- 별칭에 공백이나 특수문자가 있으면 `" "` 사용

# 3. 관계 연산자

## AND

모든 조건을 만족해야 한다.

### 가격이 1000원 이상 3000원 이하인 과자의 전체정보 조회

```sql
select *
from snack
where s_price >= 1000 and s_price <= 3000;
```

---

## BETWEEN

범위를 간단하게 표현할 수 있다.

```sql
select *
from snack
where s_price between 1000 and 3000;
```

`between a and b` 는 **a 이상 b 이하**이다.

즉,

```sql
s_price >= 1000 and s_price <= 3000
```

와 같다.

### 주의

1000원 **초과** 3000원 **미만**이면 `between` 쓰면 안 된다.

```sql
select *
from snack
where s_price > 1000 and s_price < 3000;
```

---

## OR

둘 중 하나라도 참이면 된다.

### 가격이 1000원, 2000원, 3000원인 과자의 전체 정보 조회

```sql
select *
from snack
where s_price = 1000 or s_price = 2000 or s_price = 3000;
```

---

## IN

여러 개의 특정 값을 한 번에 비교할 때 사용한다.

```sql
select *
from snack
where s_price in (1000, 2000, 3000);
```

- 연속되지 않은 값 여러 개 조회할 때 편하다
- `or`를 줄여준다

---

## NOT IN

특정 값들을 제외할 때 사용한다.

```sql
select *
from snack
where s_price not in (1000, 2000, 3000);
```

아래는 문법 오류다.

```sql
select * from snack where s_price in not (1000, 2000, 3000);
```

`not in` 으로 붙여서 생각하면 편하다.

---

# 4. LIKE

문자열 패턴 검색에 사용한다.

```sql
like '패턴'
```

## `%` 의미

`%` 는 **문자가 0개 이상 아무거나 올 수 있음**을 뜻한다.

```sql
like '%ㅋ'    -- ㅋ으로 끝남
like 'ㅋ%'    -- ㅋ으로 시작
like '%ㅋ%'   -- ㅋ 포함
```

`=` 는 패턴 검색이 아니라 **문자 그대로 비교**다.

그래서 패턴 검색은 반드시 `like` 를 사용해야 한다.

---

## '해'가 들어가는 회사에서 만든 과자의 이름 조회

```sql
select s_name, s_compa
from snack
where s_compa like '%해%';
```

---

## '이'가 들어가는 과자들 평균가 조회

```sql
select round(avg(s_price), 1)
from snack
where s_name like '%이%';
```

---

## 크라운 회사의 과자거나 초코로 시작하는 과자의 이름, 제조사, 가격 조회

```sql
select s_name, s_compa, s_price
from snack
where s_compa = '크라운' or s_name like '초코%';
```

- 회사명이 정확히 `크라운`이면 `=`
- 이름이 `초코`로 시작하면 `like '초코%'`

---

## 이름에 '새콤달콤' 또는 '초코'가 포함된 과자 조회

```sql
select s_name, s_compa, s_price
from snack
where s_name like '%새콤달콤%' or s_name like '%초코%';
```

---

## `like in` 구조는 없다

```sql
select s_name, s_compa, s_price
from snack
where s_name like in ('%새콤달콤%', '%초코%'); -- 문법 오류
```

여러 패턴을 찾으려면 `or`로 연결한다.

---

# 5. REGEXP_LIKE()

정규표현식을 이용한 문자열 검색이다.

```sql
regexp_like(컬럼명, '패턴')
```

## 기본 패턴

- `^문자` : 문자로 시작
- `문자$` : 문자로 끝
- `패턴1|패턴2` : 둘 중 하나

### 과자 이름이 '새콤달콤'으로 시작하거나 '이'로 끝나는 과자 조회

```sql
select s_name, s_compa, s_price
from snack
where regexp_like(s_name, '^새콤달콤|이$');
```

# 6. 집합 연산자

집합 연산자는 `select` 결과끼리 합치거나 비교하는 연산이다.

각 `select` 문은 **컬럼 개수와 자료형이 맞아야 한다.**

---

## UNION : 합집합 (중복 제거)

```sql
select 1 id, '수학' subject, 90 score from dual
union
select 1 id, '수학' subject, 90 score from dual
union
select 1 id, '수학' subject, 90 score from dual;
```

---

## UNION ALL : 합집합 (중복 포함)

```sql
select 1 id, '수학' subject, 90 score from dual
union all
select 1 id, '수학' subject, 90 score from dual
union all
select 1 id, '수학' subject, 90 score from dual;
```

---

## INTERSECT : 교집합

```sql
select 1 id, '수학' subject, 90 score from dual
union
select 2 id, '영어' subject, 80 score from dual
intersect
select 1 id, '수학' subject, 90 score from dual;
```

두 결과에서 **공통되는 행**만 출력한다.

---

## MINUS : 차집합

```sql
select 1 id, '수학' subject, 90 score from dual
minus
select 1 id, '수학' subject, 80 score from dual
minus
select 1 id, '수학' subject, 90 score from dual;
```

첫 번째 결과에서 두 번째 결과를 뺀다.

---

# 7. GROUP BY

같은 값끼리 묶어서 집계할 때 사용한다.

```sql
select 그룹컬럼, 집계함수
from 테이블
group by 그룹컬럼;
```

## 회사별 과자 평균 가격, 최소 가격, 최고 가격

```sql
select s_compa, round(avg(s_price), 1), min(s_price), max(s_price)
from snack
group by s_compa;
```

### 핵심

`group by s_compa` 를 했으면

`select` 절에서 일반 컬럼은 `s_compa` 만 올 수 있다.

예를 들어 아래는 안 된다.

```sql
select s_compa, s_name, avg(s_price)
from snack
group by s_compa;
```

`s_name` 이 그룹 기준도 아니고 집계도 아니기 때문이다.

---

# 8. ROLLUP

소계와 총계를 같이 보고 싶을 때 사용한다.

```sql
group by rollup(컬럼명)
```

## 회사별 과자 총 가격 합계와 전체 합계

```sql
select s_compa, sum(s_price)
from snack
group by rollup(s_compa);
```

총계 행의 회사명은 `null` 로 나온다.

보기 좋게 바꾸려면:

```sql
select nvl2(s_compa, s_compa, '전체') s_compa, sum(s_price)
from snack
group by rollup(s_compa);
```

---

# 9. HAVING

`group by` 로 묶은 결과에 조건을 줄 때 사용한다.

```sql
select 그룹컬럼, 집계함수
from 테이블
group by 그룹컬럼
having 집계조건;
```

## 회사별 과자 평균 가격이 2000원 이상인 회사 조회

```sql
select s_compa, round(avg(s_price), 1)
from snack
group by s_compa
having round(avg(s_price), 1) >= 2000;
```

### WHERE와 HAVING 차이

- `where` : 원본 데이터 조건
- `having` : 그룹 결과 조건

---

# 10. ORDER BY

정렬할 때 사용한다.

```sql
order by 컬럼명 asc;
order by 컬럼명 desc;
```

기본값은 `asc` 이다.

---

## 롯데 또는 해태 회사의 과자 평균 가격이 1800원 이상일 때 평균 가격 내림차순 조회

### 내 코드

```sql
select s_compa, round(avg(s_price),1)
from snack
where s_compa like = '%롯데%' or s_compa like = '%해태%'
group by s_compa
having round(avg(s_price),1) >= 1800 desc;
```

### 정답 코드

```sql
select s_compa, round(avg(s_price), 1)
from snack
where s_compa in ('롯데', '해태')
group by s_compa
having round(avg(s_price), 1) >= 1800
order by round(avg(s_price), 1) desc;
```

### 틀린 이유

- `like =` 는 문법 오류
- 회사명이 정확하면 `in` 이 더 적절
- `desc` 는 `having` 뒤가 아니라 `order by` 에 붙는다

# 11. 정리

## WHERE vs HAVING

```sql
where  : 원본 행 조건
having : group by 결과 조건
```

예시:

```sql
select s_compa, avg(s_price)
from snack
where s_price >= 1000
group by s_compa
having avg(s_price) >= 2000;
```

---

## BETWEEN

```sql
between 1000 and 3000
```

는

```sql
>= 1000 and <= 3000
```

와 같다.

양쪽 끝값이 포함된다.

---

## IN

```sql
where s_price in (1000, 2000, 3000)
```

여러 값 비교에 사용한다.

---

## LIKE

```sql
where s_name like '%초코%'
```

패턴 검색에 사용한다.

---

## ORDER BY는 맨 끝

```sql
select ...
from ...
where ...
group by ...
having ...
order by ...;
```

### 1.

```sql

```

→ 1000, 3000 포함

### 2.


→ 패턴 검색은 `like`

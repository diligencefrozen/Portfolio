## 1. 상속 / `Object` / 오버라이딩

### 개념 정리

- 아무 클래스도 상속하지 않으면 자동으로 `Object`를 상속한다.
- 즉 `public class Product` 는 사실상 `public class Product extends Object` 와 같다.
- 자식 클래스에서 부모 메서드를 다시 정의하는 것을 **오버라이딩**이라고 한다.
- 출력 기능처럼 **공통 내용 + 추가 내용** 구조는 오버라이딩이 잘 맞는다.
- `Lap extends Com` 은 공통 필드가 `Com`에 있으니 `Mall`보다 더 적절한 구조다.

### 코드

```java
public class Mall {
String name;
int price;
public Mall(String name, int price) {
super();
this.name = name;
this.price = price;
}
public void printInfo() {
System.out.println("상품명: "+name);
System.out.println("가격: "+price);
}
}
```

## 3. 생성자 / `super()` / 기본생성자

### 개념 정리

- 클래스에 **생성자가 하나도 없을 때만** 컴파일러가 기본생성자를 자동으로 만든다.
- 자식 생성자 첫 줄에는 부모 생성자 호출이 들어간다.
- 안 쓰면 보통 `super();`가 자동으로 들어간다.
- 부모에 기본생성자가 없으면 `super();`는 못 쓰고, 맞는 생성자를 직접 호출해야 한다.
- 그런데 현재 `Product2`는 **기본생성자도 직접 있음**. 그래서 `super();` 사용 가능하다.
- `Pen()`에서 객체가 `null`이 되는 게 아니라, 필드값이 기본값(`null`, `0`)으로 들어가는 것이다.

### 코드

```java
public class Pen extends Product2 {
String colour;
public Pen() {
}
public Pen(String name, int price, String colour) {
super(name, price);
this.colour = colour;
}
public Pen(String colour) {
super();
this.colour = colour;
}
}
```

```java
public class Product2 {
String name;
int price;
public Product2() {
System.out.println("Product !");
}
public Product2(String name, int price) {
super();
this.name = name;
this.price = price;
}
public static void main(String[] args) {
Pen p = new Pen();  // null
System.out.println("---");  // ---
Pen p2 = new Pen("빨강"); // 빨강
System.out.println("---"); // ---
Pen p3 = new Pen("모나미", 300, "검정"); // 모나미, 300, 검정
System.out.println("---");  // ---
}
}
```

## 5. 패키지

### 개념 정리

- 패키지는 **클래스 이름 충돌 방지**용이다.
- 보통 `com.회사명.프로그램명.카테고리` 식으로 작성한다.
- 패키지명은 전 세계적으로 겹치지 않게 도메인 역순 스타일을 많이 쓴다.
- 객체 생성 시 풀네임으로 쓸 수 있다.
- 생략 가능한 경우:
    1. `java.lang`
    2. 같은 패키지
    3. `import`한 클래스
    

### 예시

```java
/*java.lang.*/String s1 = "z";
/*java.lang.*/System.out.println("z");
```

```java
com.jay.mar263.main.Beaver b = new com.jay.mar263.main.Beaver();
Beaver b2 = new Beaver();
```


## 6. 접근제어자

### 개념 정리

접근 범위:

```java
private < default < protected < public
```

- `public` : 어디서든 접근 가능
- `protected` : 같은 패키지 + 다른 패키지의 자식 클래스
- `default` : 같은 패키지 안에서만 가능
- `private` : 같은 클래스 안에서만 가능

### 주의

- `protected`는 그냥 무조건 다 되는 게 아니라 **상속 관계와 패키지 관계를 같이 봐야 한다.**

### 예시

```java
public int a;
protected int b;
/*default*/ int c;
private int d;
```

```java
AMain1 amain = new AMain1();
amain.a = 1;
amain.b = 2;
amain.c = 3;
amain.d = 4;
```

## 7. 캡슐화

### 개념 정리

- 캡슐화는 **멤버변수를 직접 건드리지 못하게 하고**, 메서드로 접근하게 만드는 것이다.
- 보통 `private`로 막고 `getter/setter`를 만든다.
- setter 안에서 값 보정이나 검사가 가능하다.

### 코드

```java
package com.beaver.mar265.human;
public class Human {
private String name;
private int age;
private double w;
public Human() {
super();
setName(name);
setAge(age);
//this.w = w;
}
public String getName() {
return name;
}
public void setName(String name) {
this.name = name;
}
/////////////////////////////////////
public int getAge() {
return age;
}
// setXXX : 외부의 접근이 제한된 멤버변수를 가공할 수 있음.
public void setAge(int age) {
if (age < 0) {
age *= -1;
}
this.age = age;
}
public void printInfo() {
System.out.println("이름: "+name+" 씨");
System.out.println("나이: "+"만 "+age+"세");
System.out.println("몸무게: "+w+"kg");
}
}
```

## 9. `Menu` 예제

### 코드

```java
package com.beaver.mar265.manu;
public class Menu {
// 1.멤버변수
private String name;
private int price;
// 2. 기본 생성자
public static void name() {
}
// 3. 생성자 오버로딩 - ctrl + shift + space
public Menu(String name, int price) {
super();
this.name = name;
this.price = price;
}
// 4. getter / setter - ctrl + shift + g
public String getName() {
return name;
}
public void setName(String name) {
this.name = name;
}
public int getPrice() {
return price;
}
public void setPrice(int price) {
this.price = price;
}
// 5. 나머지 메소드들...
public void printInfo() {
System.out.println(name);
System.out.println(price);
}
}
```

- `public static void name()` 은 **기본생성자 아님**, 그냥 메서드다.
- 생성자는 **클래스 이름과 같고 리턴타입이 없어야** 한다.

# 암기 정리

```
1. 아무 것도 상속하지 않으면 자동으로 Object 상속
2. 상속은 extends
3. 부모 생성자 호출은 super()
4. 생성자가 하나도 없을 때만 기본생성자 자동 생성
5. 오버라이딩 = 부모 메서드 재정의
6. 오버로딩 = 이름 같고 매개변수 다름
7. 접근제어자 범위: private < default < protected < public
8. 캡슐화 = private + getter/setter
9. protected는 패키지/상속 관계를 같이 봐야 함
10. 생성자는 클래스명과 같고 리턴타입이 없음
```

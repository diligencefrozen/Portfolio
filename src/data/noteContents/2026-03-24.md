## 핵심 정리

- `Friend` : 1~100 사이 정답 생성
- `Me` : 사용자 입력
- 범위를 벗어난 입력은 재입력
- 맞출 때까지 반복

```java
import java.util.Iterator;

public class Referee {

	public Me callMe() {
		System.out.println("Referee :나 사용자 나오세요");
		System.out.println("Me : 나 사용자 등장");
		return new Me();
	}

	// 친구에게 답 요구
	public int askFriAns(Friend f) {
		int friAns = f.writeFriAns();
		return friAns;
	}

	// 나에게 답 요구
	public int askMyAns(Me mm) {
		int myAns = mm.tellMyAns();

		if (myAns < 1 || myAns > 100) {
			System.out.println("갈 !!!!");
		}

		return (myAns >= 1 && myAns <= 100) ? myAns : askMyAns(mm);
	}

	// 판정
	public boolean judge(int answer, int myAnswer) {
		if (answer == myAnswer) {
			System.out.println(". . . !");
		} else if (answer > myAnswer) {
			System.out.println("UP !");
		} else {
			System.out.println("DOWN !");
		}

		return (answer == myAnswer);
	}

	// 시도 횟수
	public void countTry(int turn) {
		System.out.println("------");
		System.out.printf("%d번째 시도만에 정답!\n", turn);
		System.out.println("------");
	}

	public void start(Friend f) {
		Me mm = callMe();
		int answer = askFriAns(f);
		int myAnswer = 0;
		boolean endGame = false;

		for (int turn = 1; true; turn++) {
			myAnswer = askMyAns(mm);
			endGame = judge(answer,myAnswer);

			if (endGame) {
				countTry(turn);
				break;
			}
		}
	}
}
```

## Referee.java

```java
public class Referee {

	String[] rulebook = new String[] { " ", "가위", "바위", "보" };

	public void doOpening() {
		System.out.println("가위@ 바위@ 보@");
		System.out.println("경기를 시작~~ 하겠습니다.");
		System.out.println("############");
	}

	public void readRulebook() {
		System.out.println("############");
		for (int i = 1; i < rulebook.length; i++) {
			System.out.printf("%d. %s\n", i, rulebook[i]);
		}
		System.out.println("############");
	}

	// 오버로딩
	public int askAns(User u) {
		System.out.println("유저 뭐낼래? :");
		int userAns = u.fire();
		return (userAns >= 1 && userAns <= 3) ? userAns : askAns(u);
	}

	//오버로딩
	public int askAns(Other o) {
		int otherHand = o.fire();
		return otherHand;
	}

	public void printHand(User u, Other o, String[] rulebook) {
		System.out.printf("유저는 %s\n", rulebook[u.hand]);
		System.out.printf("상대방은 %s\n", rulebook[o.hand]);
	}

	public boolean judgeWin(User u, Other o) {
		int result = u.hand - o.hand;

		if (result == 0) {
			System.out.println("비겼다 !");
			u.draw++;
		} else if (result == -1 || result == 2) {
			System.out.println("유저가 졌다 !");
		} else {
			System.out.println("유저가 이겼다 !");
			u.win++;
		}

		return (result == -1 || result == 2);
	}

	public void sayUserWin(User u) {
		System.out.printf("유저는 %d번 이겼고 %d번 비겼다 \n", u.win, u.draw);
	}

	public void start(User u, Other o ) {
		doOpening();
		readRulebook();

		while (true) {
			u.hand = askAns(u);
			o.hand = askAns(o);
			printHand(u, o, rulebook);

			if (judgeWin(u, o)) {
				sayUserWin(u);
				break;
			}
		}
	}
}
```

# PART 3. .java / .class / .jar / JavaDoc / static

---

## 메모

- 게임캐릭터 하나 생성
- 속성: 닉네임 / 레벨 / 직업 / 무기
- 속성들을 출력하는 기능까지
- Main에서 객체를 만들고 닉네임 값만 넣어서 출력

### 클라이언트가 캐릭터 생성 기능을 만들어달라고 요구

- 만드는데 성공

### 건네주는 방법

- `.java` 파일 말고, `.jar` 파일로 건네줘라

### 이유

- `.java` : 소스까지 공개
- `.jar` : 소스를 공개하지 않고 + 그 기능만 전달

### 정리

- JAR(Java ARchive, 자바 아카이브) 파일
- 기계어 상태로 바꿔서 압축해 놓은 형태

---

## 자바 파일 흐름

- 한국어 -번역(개발) -> `.java`로 저장
- `.java` 파일 -번역(Compile) -> `.class` 파일 자동 생성
- `.class` 파일 - 실행

### 정리

- `.java` 파일 : 사람이 보려고 만든 것
- `.class` 파일 : 실제로 자바에서 구동하는 것

### 왜 굳이 바이트코드로 변환해서 돌릴까?

1. 바이트코드를 사용하면 실행이 빠르고
2. 소스를 노출하면 무슨 프로그램인지 알 수 있기 때문에 보안상 좋지 않음

### 결론

- 다른 사람에게 기능을 넘겨줄 때 소스를 알려주지 말고
- 그 기능만을 사용하게 하자
- 그래서 `.java` 파일 말고 `.jar` 파일로 전달

---

## JavaDoc

### 역할

- 제품 설명서
- 우리가 만든 프로그램을 보기 쉽게 HTML 형식, 웹 형식으로 문서화한 것

### 특징

- 주석은 `/** 내용 */`
- 주석 안은 크게 2가지

- 주석 시작부터 첫 번째 태그 전까지

- 줄바꿈하고 싶으면 `<p>` 사용

### 2) 태그 섹션

- 첫 글자가 `@`로 시작

- `@param` : 파라미터 설명
- `@throws` : 예외 설명

# PART 4. static 필기

## 핵심 개념

- 멤버변수 : 객체의 속성 => `객체.속성`
- `this` : 이 객체 자신
- 모든 객체가 공통으로 가지는 정보는 `static` 사용

예:

- 과자 이름 -> 객체마다 다름
- 과자 무게 -> 객체마다 다름
- 과자 가격 -> 객체마다 다름
- 과자 생산자 -> 모두 같을 수 있음

이럴 때 생산자는 `static`

---

## static 특징

- 객체가 없어도 쓸 수 있는 정보
- `클래스명.xxx` 로 사용
- 객체들이 공통으로 공유하는 속성
- 객체를 여러 개 만들어도 static 변수는 하나만 존재
- 메모리 절약 가능
- Static 영역의 데이터는 프로그램 시작부터 종료까지 메모리에 남음
- 무분별하게 많이 사용하면 메모리 부족 가능

---

## static 메소드 특징

- 객체가 없어도 사용 가능
- `클래스명.메소드명()`
- static이 아닌 멤버변수에는 접근 불가
- static 메소드에서 만든 객체는 heap 영역에 저장

---

## Static 영역

- JVM이 동작하고 클래스가 로딩될 때 생성
- static 변수, static 메소드 등을 저장
- 프로그램이 종료될 때까지 메모리에 남음
- 어디서든 사용 가능
- 너무 많이 두면 메모리 부담

# PART 5. 암기 요약

## 1) 객체지향

- 객체 = 역할 담당
- 각 객체가 자기 책임을 수행
- 예: 심판 / 친구 / 사용자

## 2) 오버로딩

- 메소드 이름은 같고
- 매개변수가 다름

## 3) 재귀호출

- 메소드가 자기 자신을 다시 호출
- 입력값 재입력 처리에 자주 사용

## 4) `.java / .class / .jar`

- `.java` : 소스코드
- `.class` : 바이트코드
- `.jar` : 배포용 파일 묶음

## 5) JavaDoc

- HTML 형식 문서 생성
- `/** */` 사용
- 대표 태그: `@author`, `@version`, `@param`, `@throws`, `@return`

## 6) static

- 클래스 소속
- 객체들이 공유
- 클래스명으로 접근 가능

## 7) static 메소드

- 객체 없이 호출 가능
- 인스턴스 변수 직접 접근 불가

## 8) static final

- 수정 불가한 공통 상수

## 핵심

- 인스턴스 변수는 객체마다 따로
- static 변수는 클래스당 하나
- static 메소드는 인스턴스 변수 직접 접근 불가
- `.java`는 소스, `.class`는 실행, `.jar`는 배포
- 오버로딩은 이름 같고 매개변수 다름

## 1. 테이블 설계의 핵심

### 잘못 만든 방식

처음 코드에서는 `vagie01`, `vagie02`, `vagie03`처럼 **데이터 한 줄마다 테이블을 따로 만드는 구조**가 들어가 있었다.

이 방식은 데이터베이스(데이터베이스) 설계로는 맞지 않는다.

이유는 다음과 같다.

- 상품 하나마다 테이블을 새로 만들어야 해서 관리가 불가능하다.
- `배추`처럼 같은 이름의 상품이 여러 마트에 존재할 수 있는데, `상품명`을 기본키(primary key, 기본키)로 두면 중복 저장이 막힌다.
- 상품 정보와 마트 정보가 한 테이블에 섞여 있어서 **정규화(normalization, 정규화)** 가 안 되어 있다.
- 조회(select, 조회), 수정(update, 수정), 삭제(delete, 삭제)가 불편해진다.

### 올바른 방식

정답 코드처럼 **마트 테이블(market)** 과 **상품 테이블(product)** 을 분리해야 한다.

- `market` : 마트 자체 정보 저장
- `product` : 상품 정보 저장
- 상품은 어느 마트 소속인지 `외래키(foreign key, 외래키)` 로 연결

즉, 관계형 데이터베이스(relational database, 관계형 데이터베이스)에서는

**중복되는 정보를 분리하고, 서로 참조하게 만드는 것**이 핵심이다.

---

## 2. CRUD(생성/조회/수정/삭제)

CRUD는 데이터베이스에서 가장 기본이 되는 네 가지 작업이다.

### C - Create(생성)

새로운 데이터를 넣는 작업이다.

```sql
insert into product values(product_seq.nextval, '배추', 3000, 10000, 1);
```

### R - Read(조회)

데이터를 읽어오는 작업이다.

```sql
select * from product;
```

### U - Update(수정)

기존 데이터를 바꾸는 작업이다.

```sql
update product
set p_price = 0
where p_name = '무';
```

### D - Delete(삭제)

데이터를 지우는 작업이다.

```sql
delete from product
where p_name = '무';
```

실무에서도 보통 **조회(Read, 조회)** 의 비중이 가장 크고,

그다음이 생성(Create, 생성), 수정(Update, 수정), 삭제(Delete, 삭제) 순으로 많이 다뤄진다.

---

## 3. 시퀀스(sequence, 순번 생성기)

오라클(Oracle)에서는 기본키(primary key, 기본키)를 자동 증가시키기 위해

시퀀스(sequence, 순번 생성기)를 자주 사용한다.

예:

```sql
create sequence market_seq;
create sequence product_seq;
```

삽입(insert, 삽입)할 때는:

```sql
market_seq.nextval
product_seq.nextval
```

을 사용해서 번호를 자동으로 부여한다.

## 4. 원본 SQL(에스큐엘)에서 틀린 부분 정리

### 1) 10% 할인 계산 오류

원본에는 이런 식이 있었다.

```sql
set p_price = p_price / 10
```

이건 **10% 할인**이 아니라 **원래 가격의 10%만 남기는 것**, 즉 **90% 할인**에 가깝다.

정확한 10% 할인은:

```sql
set p_price = p_price * 0.9
```

이다.

---

### 2) 모든 마트의 주차장 30% 줄이기

원본에는 상품 테이블(product)에 대해 무게(p_weight)를 수정하는 잘못된 코드가 있었다.

```sql
update product
set p_weight = p_weight / 30
where p_no;
```

문제점:

- 주차장은 `market` 테이블 정보인데 `product`를 수정하고 있다.
- `where p_no;` 는 문법 자체가 틀렸다.
- 30% 줄이기는 `/30` 이 아니라 `0.7` 이다.

정답:

```sql
update market
set m_parking = m_parking * 0.7;
```

---

### 3) `like`(포함 검색) 사용

자바(Java)에서 `PreparedStatement(준비된 명령문)` 를 쓸 때 `%?%` 식으로 쓰면 안 된다.

`?` 는 자바 영역이 아니라 SQL(에스큐엘)에서 값이 채워지는 자리이기 때문이다.

따라서 오라클(Oracle)에서는 이렇게 써야 한다.

```sql
where p_name like '%' || ? || '%'
```

## 5. JDBC(Java DataBase Connectivity, 자바 데이터베이스 연결성)

JDBC는 자바(Java)에서 데이터베이스(DB, 데이터베이스)를 다루기 위한 API(응용 프로그램 인터페이스)이다.

오라클(Oracle) DB와 연결하려면 보통 다음 요소가 필요하다.

- `Connection(연결)`
- `PreparedStatement(준비된 명령문)`
- `ResultSet(조회 결과 집합)`

흐름은 아래와 같다.

1. `Connection` 으로 DB 서버에 연결
2. SQL 문 작성
3. `PreparedStatement` 에 SQL 장착
4. 필요한 값 채우기
5. 실행
6. `select` 면 `ResultSet` 으로 결과 받기
7. 다 쓴 자원 닫기(close)

---

## 6. executeUpdate() 와 executeQuery() 차이

### executeUpdate()

- `insert`
- `update`
- `delete`

처럼 **데이터에 변화가 생기는 SQL** 을 실행할 때 사용한다.

반환값은 **영향받은 행(row, 행)의 수** 이다.

### executeQuery()

- `select`

처럼 **데이터를 조회하는 SQL** 을 실행할 때 사용한다.

반환값은 `ResultSet(조회 결과 집합)` 이다.

---

## 7. ResultSet(조회 결과 집합)

`select` 결과는 `ResultSet` 으로 받는다.

```java
rs = pstmt.executeQuery();
while (rs.next()) {
    System.out.println(rs.getString("p_name"));
}
```

핵심은 다음과 같다.

- `rs.next()` : 다음 행(row, 행)으로 이동
- 다음 데이터가 있으면 `true`
- 더 이상 없으면 `false`

컬럼(column, 열)을 읽을 때는 인덱스보다 **컬럼명**을 쓰는 것이 좋다.

```java
rs.getString("p_name");
rs.getInt("p_price");
```

이유는 인덱스 방식은 컬럼 순서를 착각하기 쉽기 때문이다.

---

## 8. PreparedStatement(준비된 명령문)를 쓰는 이유

`PreparedStatement` 를 쓰면 좋은 점은 다음과 같다.

- 값만 바뀌고 SQL 구조는 유지된다.
- 문자열 이어붙이기보다 안전하다.
- 문법 실수를 줄일 수 있다.
- `SQL Injection(에스큐엘 인젝션, SQL 삽입 공격)` 위험을 줄일 수 있다.

예:

```java
String sql = "insert into market values(market_seq.nextval, ?, ?, ?, ?)";
pstmt = con.prepareStatement(sql);
pstmt.setString(1, name);
pstmt.setString(2, location);
pstmt.setInt(3, land);
pstmt.setInt(4, parking);
```

---

## 9. 자원 닫기(close)와 NullPointerException(널 포인터 예외)

원본 `BeaverDBManager` 에서 `ResultSet rs` 를 무조건 닫으려고 하다가

`insert`, `update`, `delete` 때는 `rs` 가 없어서 `NullPointerException(널 포인터 예외)` 가 날 수 있다고 설명한 부분이 있었다.

방향은 맞다. 그래서 **널(null) 검사** 또는 **공통 닫기 메서드**가 중요하다.

## 10. MVC(Model View Controller, 모델/뷰/컨트롤러) + DAO(Data Access Object, 데이터 접근 객체) + DTO(Data Transfer Object, 데이터 전달 객체)

### MVC

- **Model(모델)** : 데이터와 비즈니스 로직
- **View(뷰)** : 화면 출력, 입력
- **Controller(컨트롤러)** : 흐름 제어

### DAO

DB 관련 작업만 전담하는 클래스

예:

- 예약 추가
- 매장 조회
- 예약 삭제

### DTO

데이터를 담아서 옮기는 객체

예:

- 매장 객체
- 예약 객체

즉, 프로젝트에서는

- `ConsoleScreen` → View(뷰)
- `Controller` → Controller(컨트롤러)
- `DAO` → Model(모델) 중 DB 처리 담당
- `DTO` → 데이터 전달 객체

로 역할을 분리하는 것이 좋다.

# 꼭 고쳐야 할 점

아래는 네 코드 전체에서 핵심적으로 보완해야 하는 부분이다.

## 1. DB 연결 정보가 여기저기 중복됨

`addr`, `id`, `pw` 가 여러 클래스에 반복된다.

→ `BeaverDBManager` 에서 한 번만 관리해야 한다.

## 2. `close()` 가 널(null) 안전하지 않음

`rs.close()` 전에 `rs != null` 확인이 필요하다.

## 3. `PreparedStatement` 를 안 닫는 경우가 있음

`InsertMain1` 에서 `pstmt.close()` 가 빠져 있다.

## 4. 사용하지 않는 `ResultSet` 이 많음

`UpdateMain`, `DeleteMain` 에서 `ResultSet` 이 필요 없는데 선언돼 있다.

## 5. `phone` 을 숫자형(number)으로 두면 불편함

전화번호는 계산 대상이 아니므로 `varchar2` 가 더 적절하다.

앞자리 `0`, 국가번호 `+82`, 하이픈 `-` 처리도 문자열이 유리하다.

## 6. 예약 날짜를 문자열(varchar2)로 저장하면 정렬/비교가 불편함

가능하면 `timestamp(날짜+시간)` 나 `date(날짜)` 로 저장하는 것이 좋다.

## 7. 예약 테이블에서 매장 위치를 문자열로 직접 저장함

이건 중복 데이터가 생긴다.

→ `매장 번호` 를 외래키(foreign key, 외래키)로 참조하는 방식이 더 좋다.

```java
package com.beaver.db.manager;

import java.sql.Connection;
import java.sql.DriverManager;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.SQLException;

public final class BeaverDBManager {

    private static final String ADDR = "jdbc:oracle:thin:@localhost:1521:xe";
    private static final String USER = "jeesungmain";
    private static final String PASSWORD = "123456";

    private BeaverDBManager() {
    }

    public static Connection connect() throws SQLException {
        return DriverManager.getConnection(ADDR, USER, PASSWORD);
    }

    public static void close(Connection con, PreparedStatement pstmt, ResultSet rs) {
        close(rs);
        close(pstmt);
        close(con);
    }

    public static void close(Connection con, PreparedStatement pstmt) {
        close(pstmt);
        close(con);
    }

    private static void close(AutoCloseable target) {
        if (target != null) {
            try {
                target.close();
            } catch (Exception e) {
                // 조용히 종료
            }
        }
    }
}
```

# 8. market / product 테이블 최종 정리 SQL

```sql
drop table product cascade constraints purge;
drop table market cascade constraints purge;

create table market(
    m_no number(4) primary key,
    m_name varchar2(20 char) not null,
    m_location varchar2(20 char) not null,
    m_land number(4) not null,
    m_parking number(4) not null
);

create sequence market_seq start with 1 increment by 1;

insert into market values(market_seq.nextval, '홈플러스', '인천', 1000, 500);
insert into market values(market_seq.nextval, '홈플러스', '강남', 500, 300);
insert into market values(market_seq.nextval, '이마트', '서초', 800, 800);

create table product(
    p_no number(4) primary key,
    p_name varchar2(20 char) not null,
    p_weight number(4) not null,
    p_price number(5) not null,
    p_m_no number(4) not null,
    constraint fk_product_market
        foreign key (p_m_no)
        references market(m_no)
        on delete cascade
);

create sequence product_seq start with 1 increment by 1;

insert into product values(product_seq.nextval, '배추', 3000, 10000, 1);
insert into product values(product_seq.nextval, '무', 500, 5000, 2);
insert into product values(product_seq.nextval, '배추', 2000, 5000, 3);

select * from market;
select * from product;
```

# 9. SQL(db1.sql)

**시간은 timestamp(타임스탬프)** 로 저장하고, **매장 번호를 외래키로 참조.**

```java
package com.beaver.franc.main;

import java.sql.Connection;
import java.sql.PreparedStatement;
import java.sql.ResultSet;
import java.sql.Timestamp;

import com.beaver.db.manager.BeaverDBManager;

public class DAO {

    private static final DAO dao = new DAO();

    private DAO() {
    }

    public static DAO getInstance() {
        return dao;
    }

    // 1. 예약하기
    public boolean insertReservation(String name, Timestamp date, String phone, int restaurantNo) {
        String sql = "insert into reserva values(reserva1_seq.nextval, ?, ?, ?, ?)";

        try (Connection con = BeaverDBManager.connect();
             PreparedStatement pstmt = con.prepareStatement(sql)) {

            pstmt.setString(1, name);
            pstmt.setTimestamp(2, date);
            pstmt.setString(3, phone);
            pstmt.setInt(4, restaurantNo);

            return pstmt.executeUpdate() == 1;

        } catch (Exception e) {
            e.printStackTrace();
            return false;
        }
    }

    // 2. 매장 등록
    public boolean insertRestaurant(String location, String owner, int seats) {
        String sql = "insert into res values(res_seq.nextval, ?, ?, ?)";

        try (Connection con = BeaverDBManager.connect();
             PreparedStatement pstmt = con.prepareStatement(sql)) {

            pstmt.setString(1, location);
            pstmt.setString(2, owner);
            pstmt.setInt(3, seats);

            return pstmt.executeUpdate() == 1;

        } catch (Exception e) {
            e.printStackTrace();
            return false;
        }
    }

    // 지점명으로 번호 찾기
    public Integer getRestaurantNoByLocation(String location) {
        String sql = "select r_no from res where r_loca = ?";

        try (Connection con = BeaverDBManager.connect();
             PreparedStatement pstmt = con.prepareStatement(sql)) {

            pstmt.setString(1, location);

            try (ResultSet rs = pstmt.executeQuery()) {
                if (rs.next()) {
                    return rs.getInt("r_no");
                }
            }

        } catch (Exception e) {
            e.printStackTrace();
        }
        return null;
    }

    // 3. 전체 예약 확인
    public void getAllReservations() {
        String sql = "select re.re_no, re.re_name, re.re_date, re.re_phone, r.r_loca "
                   + "from reserva re, res r "
                   + "where re.re_r_no = r.r_no "
                   + "order by re.re_no asc";

        try (Connection con = BeaverDBManager.connect();
             PreparedStatement pstmt = con.prepareStatement(sql);
             ResultSet rs = pstmt.executeQuery()) {

            boolean hasData = false;

            while (rs.next()) {
                hasData = true;
                System.out.println(
                    "예약번호: " + rs.getInt("re_no")
                    + ", 예약자명: " + rs.getString("re_name")
                    + ", 예약일시: " + rs.getTimestamp("re_date")
                    + ", 전화번호: " + rs.getString("re_phone")
                    + ", 지점명: " + rs.getString("r_loca")
                );
            }

            if (!hasData) {
                System.out.println("출력할 예약이 없습니다.");
            }

        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    // 4. 전체 매장 확인
    public void getAllRestaurants() {
        String sql = "select * from res order by r_loca asc";

        try (Connection con = BeaverDBManager.connect();
             PreparedStatement pstmt = con.prepareStatement(sql);
             ResultSet rs = pstmt.executeQuery()) {

            boolean hasData = false;

            while (rs.next()) {
                hasData = true;
                System.out.println(
                    "지점번호: " + rs.getInt("r_no")
                    + ", 지역: " + rs.getString("r_loca")
                    + ", 사장님: " + rs.getString("r_owner")
                    + ", 좌석수: " + rs.getInt("r_seats")
                );
            }

            if (!hasData) {
                System.out.println("출력할 매장이 없습니다.");
            }

        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    // 5. 최소 좌석 수 이상 매장 찾기
    public void findRestaurantsByMinSeats(int seats) {
        String sql = "select * from res where r_seats >= ? order by r_loca asc";

        try (Connection con = BeaverDBManager.connect();
             PreparedStatement pstmt = con.prepareStatement(sql)) {

            pstmt.setInt(1, seats);

            try (ResultSet rs = pstmt.executeQuery()) {
                boolean hasData = false;

                while (rs.next()) {
                    hasData = true;
                    System.out.println(
                        "지점번호: " + rs.getInt("r_no")
                        + ", 지역: " + rs.getString("r_loca")
                        + ", 사장님: " + rs.getString("r_owner")
                        + ", 좌석수: " + rs.getInt("r_seats")
                    );
                }

                if (!hasData) {
                    System.out.println("조건에 맞는 매장이 없습니다.");

        } catch (Exception e) {
            e.printStackTrace();
        }
    }

    // 6. 예약자 이름으로 예약 찾기
    public void findReservationsByName(String keyword) {
        String sql = "select re.re_no, re.re_name, re.re_date, re.re_phone, r.r_loca "
                   + "from reserva re, res r "
                   + "where re.re_r_no = r.r_no and re.re_name like '%' || ? || '%' "
                   + "order by re.re_no asc";

        try (Connection con = BeaverDBManager.connect();
             PreparedStatement pstmt = con.prepareStatement(sql)) {

            pstmt.setString(1, keyword);

            try (ResultSet rs = pstmt.executeQuery()) {
                boolean hasData = false;

                while (rs.next()) {
                    hasData = true;
                    System.out.println(
                        "예약번호: " + rs.getInt("re_no")
                        + ", 예약자명: " + rs.getString("re_name")
                        + ", 예약일시: " + rs.getTimestamp("re_date")
                        + ", 전화번호: " + rs.getString("re_phone")
                        + ", 지점명: " + rs.getString("r_loca")
                    );
                }

                if (!hasData) {
                    System.out.println("검색 결과가 없습니다.");
                }
            }

        } catch (Exception e) {
            e.printStackTrace();
        }
    }

            e.printStackTrace();
```

# 15. 암기용

## JDBC 핵심

- JDBC(자바 데이터베이스 연결성)는 자바(Java)에서 DB(데이터베이스)를 다루기 위한 API(응용 프로그램 인터페이스)이다.
- `Connection(연결)` 으로 접속하고, `PreparedStatement(준비된 명령문)` 로 SQL(에스큐엘)을 실행한다.
- 조회(select, 조회)는 `ResultSet(조회 결과 집합)` 으로 받는다.

## CRUD 핵심

- Create(생성) = `insert`
- Read(조회) = `select`
- Update(수정) = `update`
- Delete(삭제) = `delete`

## 실수 포인트

- 문자열 SQL 끝에 세미콜론(`;`) 넣지 않기
- 공백 빠뜨리지 않기
- `like` 검색은 `'%‘ || ? || '%'`
- 할인율 계산할 때 `/10` 과 `0.9` 구분하기

## 설계 핵심

- 데이터 한 건마다 테이블을 따로 만들지 않는다.
- 중복 정보는 분리하고 외래키(foreign key, 외래키)로 연결한다.
- 전화번호는 숫자보다 문자열이 적합하다.
- 날짜/시간은 문자열보다 `date` 또는 `timestamp` 가 적합하다.
